# Урок 7.2 · Память и планирование агента

**Фаза 7 — Agent Engineering** · **Результат фазы:** Собрать надёжного агента с памятью, планированием, human-in-the-loop и guardrails.
<!-- **Requires:** платный API-ключ — только для блока USE IT -->

> **MOTTO.** Надёжный агент не импровизирует на каждом шаге: он строит план и несёт состояние в памяти.

## PROBLEM

Голый цикл reason→act→observe (7.1) решает простое, но на многошаговой задаче спотыкается: забывает промежуточные результаты и теряет нить. Двух вещей не хватает — **плана** (разбить цель на шаги, как в 2.3) и **памяти** (носить состояние между шагами, как в 4.2). Соединим их в агенте.

## CONCEPT

```
цель → plan() → [шаг1, шаг2, шаг3]
                   │ каждый шаг:
                   │   recall(value)  ← читаем из памяти
                   │   tool(...)      ← действуем
                   │   remember(value) → пишем в память
                   ▼
              итог + трейс шагов
```

Память превращает цепочку шагов в связный процесс: шаг N видит результат шага N−1. Это мост от «промпт-цепочки» (2.3) к автономному агенту.

## BUILD IT

Планировщик + рабочая память поверх цикла из 7.1: [`code/planner_agent.py`](../code/planner_agent.py).

- `Memory` — `remember/recall` (рабочая память агента);
- `plan(goal)` — декомпозиция цели в шаги;
- `run(goal)` — исполнить план, неся состояние через память; возвращает `(итог, память, трейс)`.

```bash
python code/planner_agent.py
pytest code -q
```

Демо считает `(2+3)*4-1 = 19`: видно, как каждый шаг читает прошлое значение из памяти и пишет новое; в памяти остаются все промежуточные шаги.

## USE IT

Многошаговая задача в реальном инструменте (мульти-провайдер): план строит LLM, память — платформенная или своя.

- **Claude Agent SDK** — agent loop + memory tool (`/memories`) / managed memory stores; план — через промптинг или субагентов.
- **OpenAI Agents SDK** — sessions хранят историю; планирование через инструкции.
- Свой вариант: `MemoryStore` из урока 4.2 как долговременная память + планировщик-LLM.

⚠️ Память, в которую пишет агент, — поверхность для prompt-инъекций: reference-данные держи read-only (урок 6.5).
