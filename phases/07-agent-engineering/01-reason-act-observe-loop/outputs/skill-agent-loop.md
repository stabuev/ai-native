# Артефакт: agent-loop (skill)

Переиспользуемый каркас агентного цикла reason → act → observe с встроенными guardrails. Подставляешь свои инструменты и policy — получаешь рабочего агента. Основа для всей Фазы 7 и для Capstone.

## Контракт

- **Tool** — обычная функция `name(*args) -> observation`. Регистрируется в словаре `tools`.
- **Policy** — `policy(goal, history) -> Action`. Решает, что делать дальше.
- **Action** — `Action("tool", tool, args)` или `Action("final", answer=...)`.
- **run_agent(goal, tools, policy, max_steps)** — крутит цикл; возвращает `(answer, history)` или поднимает `AgentError`.

## Guardrails (всегда включены)

- **Лимит шагов** — защита от петель: нет финала за `max_steps` → `AgentError`.
- **Проверка инструмента** — вызов незарегистрированного инструмента → `AgentError`.
- (дальше по фазе) human-in-the-loop перед «опасными» инструментами — урок 7.3.

## Как переиспользовать

1. Опиши инструменты и положи в `tools`.
2. Для прототипа возьми детерминированную policy; для прода — LLM-policy через function calling (форма `Action` та же).
3. Логируй `history` — это память и трейс для отладки.
4. Роль policy отдавай модели подешевле (Haiku/Flash) — см. Фазу 9; на сложные решения эскалируй на Opus.
