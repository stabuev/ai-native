## Детекторы

```python
from agent_failures import run_guarded, detect_loop, goal_relatedness

res = run_guarded(policy, goal="сделать отчёт",
                  max_steps=10, repeat=3, drift_threshold=0.3)
# res["status"]: done | loop_detected | goal_drift | step_limit
```

## Плейбук: симптом → причина → приём

| Симптом | Вероятная причина | Приём |
|---|---|---|
| Повторяет одно действие | петля (нет прогресса) | `detect_loop` → стоп/смена стратегии; добавить состояние в память (7.2) |
| Ушёл не туда | дрейф цели | сверять действия с целью (`goal_relatedness`); напоминать цель |
| Не останавливается | нет условия выхода | лимит шагов/времени/бюджета (7.1) |
| Зовёт несуществующий инструмент | плохие схемы | валидация tool_call (6.1), чёткие описания |
| Уверенно врёт в шаге | галлюцинация | проверки результата (5.5), цитаты (1.4) |
| Опасное действие | нет заслона | guardrails + human-in-the-loop (7.3) |

## Приёмы устойчивости

- **Лимиты** на шаги/время/токены — обязательный потолок.
- **Детекция повторов и дрейфа** — раннее срабатывание предохранителя.
- **Self-reflection (Reflexion)** — после неудачи отрефлексировать и перепланировать, а не повторять.
- **Трейс и наблюдаемость** (Фаза 10) — чтобы видеть, где именно сломалось.
