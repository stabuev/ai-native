# Урок 9.3 · Паттерны экономии

**Фаза 9 — Экономика и оркестрация моделей (FinOps)** · **Результат фазы:** Спроектировать маршрутизацию моделей и посчитать бюджет «до/после».
<!-- **Requires:** платный API-ключ — только для блока USE IT -->

> **MOTTO.** Сначала дёшево, эскалация по необходимости: каскад + кэш + batch дают 70–90% экономии.

## PROBLEM

Маршрут по сложности (9.2) хорош, но классификатор может ошибиться. Надёжнее — **каскад**: пробуем дешёвую модель, проверяем качество, и только если не дотянула — эскалируем на сильную. Плюс инфраструктурные скидки: prompt caching и batch API. Вместе — кратная экономия без потери качества.

## CONCEPT

```
запрос → cheap → quality >= gate? ──да──► ответ (заплатили мало)
                     │ нет
                     ▼
                  mid → quality >= gate? ──да──► ответ
                     │ нет
                     ▼
                  top → ответ (платим за топ только когда нужно)
   + prompt caching (−90% на повторяемый вход)  + batch (−50%)
```

Идея каскада из **FrugalGPT**: дешёвая модель часто справляется; дорогую зовём только на сложном. Скидки стэкаются.

## BUILD IT

Роутер + каскад с quality-gate: [`code/cost_router.py`](../code/cost_router.py).

- `cascade(request, answer_fn, quality_fn, gate)` — идёт по лестнице моделей, пока качество ≥ gate; считает суммарную стоимость;
- `always_top_cost()` — сколько стоило бы «всё через топ» (для сравнения экономии).

```bash
python code/cost_router.py
pytest code -q
```

Тест на смешанной нагрузке: каскад дешевле, чем «всегда топ» — простое решается дёшево, на сложное эскалируем. Качество измеряет `quality_fn` (стаб; в проде — LLM-judge из урока 2.5).

## USE IT

Инфраструктурные паттерны экономии (мульти-провайдер):

- **Prompt caching** — повторяемый префикс (system, документы) кэшируется: cache read ≈ 10% цены входа. Окупается с 1–2 попаданий.
- **Batch API** — асинхронные задачи (классификация, eval, генерация) −50%; стэкается с кэшем → эффективно <5% базовой цены.
- **Каскад + роутер** — quality-gate решает, нужна ли эскалация (как наш `cascade`).
- **Quality-gate** — дешёвой моделью-судьёй (урок 2.5), чтобы не эскалировать зря.
