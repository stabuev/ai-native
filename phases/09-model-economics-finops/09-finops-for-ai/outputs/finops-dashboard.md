# Артефакт: бюджет-дашборд + расчёт ROI

Шаблон FinOps-контура для ИИ. Код учёта — [`code/finops.py`](../code/finops.py).

## Что тегировать в каждом запросе

```
{team, feature, model, cache_hit, input_tokens, output_tokens, cost}
```

Без тегов нет attribution — невозможно понять, куда уходят деньги.

## Разрезы дашборда

| Разрез | Метрика | Зачем |
|---|---|---|
| По команде/фиче | сумма $, доля | кому отнести расход |
| По модели | сумма $, доля вызовов | где гонять дешевле (роутинг 9.2) |
| Cache-hit rate | % попаданий в кэш | потенциал prompt caching (9.3) |
| Cost per outcome | $ / результат | главная метрика зрелости (не cost per inference) |
| Тренд/форкаст | $/день, прогноз | расходы ИИ нелинейны — нужен форкаст |

## Лимиты и алерты

- Потолок бюджета на команду/фичу → алерт при приближении (`over_budget`).
- Алерт на скачок (новая фича/модель может удвоить расход за ночь).

## Расчёт ROI «до/после»

```python
from finops import roi
# было: процесс стоил 100; стало с ИИ: 20; доп. выгода (скорость/качество): 50
print(roi(before_cost=100, after_cost=20, value_gained=50))   # 6.5
```

ROI = (экономия + выгода) / затраты. Это финальная метрика capstone (Фаза 12).

## Правила

- **Измеряй до оптимизации** — иначе не докажешь эффект.
- **Тегируй на входе**, агрегируй на выходе (`Ledger.by_tag`).
- **Cost per outcome > cost per inference** — считай стоимость результата, а не вызова.
- Связывай с маршрутизацией (9.2), каскадом и кэшем (9.3): дашборд показывает, сработали ли они.
