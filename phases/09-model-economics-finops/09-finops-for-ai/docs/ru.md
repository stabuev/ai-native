> **MOTTO.** Нельзя оптимизировать то, что не измеряешь: тегируй запросы, считай по разрезам, меряй ROI.

## PROBLEM

Маршрутизация и каскад (9.2–9.3) экономят, но без учёта непонятно, сработало ли и где деньги. Нужен **FinOps-контур**: каждый запрос с тегами (команда, модель, cache-hit, стоимость), агрегация по разрезам, бюджеты с алертами и расчёт ROI автоматизации.

## CONCEPT

```
запрос → лог с тегами {team, model, cache_hit, cost}
              │ агрегация
   по команде / по модели / cache-hit rate
              │
   бюджет + алерт при превышении
   ROI = (экономия + выгода) / затраты   ← «до/после»
```

FinOps для ИИ — это видимость (куда уходят деньги), ответственность (кому отнести расход) и оптимизация (что улучшить). Метрика зрелости — не «cost per inference», а **cost per outcome** (стоимость результата).

## BUILD IT

Тегирование, агрегация, бюджет и ROI: [`code/finops.py`](../code/finops.py).

- `Ledger.record(cost, **tags)` / `by_tag(key)` — лог и агрегация по разрезам;
- `cache_hit_rate()`, `over_budget(limit)` — мониторинг;
- `roi(before, after, value)` — окупаемость автоматизации.

```bash
python code/finops.py
pytest code -q
```

Демо: расходы по командам и моделям, cache-hit rate, флаг превышения бюджета и ROI = 6.5 (было 100 → стало 20 + выгода 50). Это «бюджет-дашборд» в миниатюре.

## USE IT

Лимиты, алерты, мониторинг токенов (мульти-провайдер):

- **Тегируй каждый запрос** — команда/фича/модель/cache-hit; без тегов нет attribution.
- **Лимиты и алерты** — потолок бюджета на команду/фичу, уведомление при приближении.
- **Дашборд** — расходы по разрезам + cache-hit rate + cost per outcome; недельный/месячный форкаст (расходы ИИ нелинейны).
- **ROI «до/после»** — стоимость процесса до автоматизации vs после + полученная выгода (capstone, Фаза 12).
