> **MOTTO.** Платишь за токены, а не за запросы; стоимость надо уметь считать до отправки.

## PROBLEM

«Сколько это стоит в проде на 100k запросов?» — без калькулятора это гадание. Цена зависит от модели, числа входных/выходных токенов и режима (кэш, batch). Не считая стоимость заранее, легко получить счёт в 10× из-за одной неоптимальной модели или невключённого кэша.

## CONCEPT

```
стоимость = (вход × цена_вход + выход × цена_выход) / 1M
   модификаторы:
   - prompt caching: кэшированный вход ≈ 10% цены (−90%)
   - batch API: −50% на всё
   (скидки стэкаются → эффективно <5% от базовой цены)
```

Выход обычно в разы дороже входа. Разброс между моделями — на порядки (Flash vs Opus), поэтому выбор модели — главный рычаг стоимости.

## BUILD IT

Калькулятор стоимости по токенам с кэшем и batch: [`code/cost_calculator.py`](../code/cost_calculator.py).

- `PRICES` — карта цен 2026 (вход/выход на 1M);
- `cost(model, n_in, n_out, cached_in, batch)` — стоимость вызова;
- `compare(models, n_in, n_out)` — сравнить модели по цене.

```bash
python code/cost_calculator.py
pytest code -q
```

Демо: тот же запрос на Opus в ~11× дороже, чем на Flash; кэш и batch заметно режут счёт. Это вход в маршрутизацию (9.2–9.3).
