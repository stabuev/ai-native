# Урок 0.4 · Соглашения курса

**Фаза 0 — Setup & Tooling** · **Результат фазы:** Настроить окружение, ключи и агента; осознанно выбрать модель под задачу.

**Последний урок Фазы 0 — про то, как устроен сам курс.** Дальше будет 59 уроков с общим педагогическим контрактом, но не обязательно с одинаковым оглавлением. Структурную заполненность можно проверить автоматически; необходимость каждого блока — только по тому, помогает ли он студенту достичь результата.

> **MOTTO.** Общий контракт — не одинаковое оглавление, а одинаковая требовательность к результату.

## PROBLEM

Без соглашений курс превращается в свалку, но жёсткий шаблон создаёт другую проблему: автор добавляет код, квиз, артефакт и шесть секций даже там, где они ничего не учат. Нужен **общий контракт результата** и гибкая структура, в которой каждый элемент проходит проверку на целесообразность.

## CONCEPT

### Интуиция

Соглашение — это **контракт**, а не бюрократия. Договорились раз, что урок должен привести к наблюдаемому действию, проверке и переносу. А вот форма следует из outcome: вычислению нужен код, решению — рубрика, процедуре — сценарий. Автоматика может найти незаполненный стаб, но не может решить, нужен ли конкретному уроку `BUILD IT` или отдельный артефакт.

### Как это работает

У каждого урока есть обязательный материал и опциональные носители практики:

```
phases/<PP>-<phase-slug>/<LL>-<lesson-slug>/
├── docs/ru.md   обязательный связный материал
├── code/        опционально: исполняемая практика + тесты
├── outputs/     опционально: переиспользуемый артефакт
└── quiz.json    опционально: диагностический квиз
```

`PP` — номер фазы, а `LL` — порядковый номер урока внутри неё. Поэтому урок 7.4
лежит в `phases/07-agent-engineering/04-where-agents-break/`: цифры отвечают на разные
вопросы и не дублируют друг друга.

**Definition of Done состоит из двух слоёв.** Структурно обязателен заполненный `docs/ru.md` без следов стаба. Педагогически урок готов, когда студент без скрытых предпосылок выполняет заявленное действие, проверяет результат и переносит его в новую задачу. `MOTTO`, `PROBLEM`, `CONCEPT`, `BUILD IT`, `USE IT`, `SHIP IT`, ошибки, квиз и материалы можно оставить, объединить или опустить. Полный внутренний контракт курса лежит в [`LESSON_CONTRACT.md`](../../../../LESSON_CONTRACT.md).

**Код, тест, квиз и артефакт — не догма.** Добавляем их только там, где они доказывают или переносят outcome. Если код есть, он запускается офлайн на CPU без платных ключей, а тест проверяет содержательное поведение.

Где код содержательный, **BUILD IT оформляется как упражнение**: спека + тесты как ТЗ, эталонное решение под катом, и три пути пройти его — собрать самому, подсмотреть эталон или делегировать ИИ с обязательной самопроверкой.

> **Врезка: AI Fluency («свободное владение ИИ») и рамка 4D — как делегировать упражнение.** Четыре компетенции помогают не превращать делегирование в кнопку «сделай за меня»: **Delegation** — решить, какую часть отдать ИИ, а какую оставить себе; **Description** — описать задачу, контекст и критерии приёмки; **Discernment** — проверить результат тестами и смыслом, заметить ограничения; **Diligence** — сохранить ответственность за итог, защитить данные и честно обозначить использование ИИ. Это не четыре последовательных шага, а четыре контрольных вопроса, которые сопровождают работу. Полная рамка — в [глоссарии](../../../../glossary/terms.md) и первоисточнике [AI Fluency Framework](https://www-cdn.anthropic.com/b383cf6baddbfc72fdf8b0ed533a518e2872d531.pdf).

## РАЗБОР ПО ШАГАМ

Посмотрим, какую часть готовности можно проверить кодом. `scaffold_lesson()` создаёт только обязательный `docs/ru.md` с минимальным паспортом, а `validate_lesson()` ищет незаполненный стаб.

**Шаг 1 — свежий стаб.** Паспорт ещё содержит плейсхолдеры:

```
validate_lesson(новый_стаб) → ["в docs/ru.md остались маркеры незаполненного стаба"]
```

Валидатор не требует секции и дополнительные файлы: он не знает, какие из них нужны outcome.

**Шаг 2 — заполнили связный материал** без стандартных секций и артефакта:

```
validate_lesson(урок) → []      # пустой список = готов по структуре
```

Структурно готово: материал заполнен. Теперь нужен содержательный аудит: соответствует ли выбранная структура outcome и не добавлено ли что-нибудь только ради шаблона.

## BUILD IT

Scaffolder и валидатор урока с нуля, без зависимостей (читай и запусти): [`code/conventions.py`](../code/conventions.py).

- `scaffold_lesson(root, phase_dir, lesson_dir, id, title)` — создаёт минимальный `docs/ru.md` с паспортом;
- `validate_lesson(path)` — проверяет наличие заголовка и отсутствие маркеров стаба. Секции, код, квиз и артефакт намеренно не требует.

```bash
python code/conventions.py
pytest code -q
```

Демо создаёт временный урок и показывает, что незаполненный паспорт не проходит проверку.

## USE IT

Те же соглашения уже автоматизированы скиллом курса — `/lesson-author <id>` (см. [`.claude/skills/lesson-author`](../../../../.claude/skills/lesson-author/SKILL.md)):

```
/lesson-author 2.3     # восстанавливает outcome и соседей,
                       # выбирает целесообразные блоки и проверки
```

Цикл сдачи урока: определить наблюдаемый результат → `scaffold` → наполнить → пройти педагогическое ревью → получить зелёный `validate_lesson` и соразмерные тесты → отметить `[x]` в `PROGRESS.md` → коммит **после всей фазы**.

## SHIP IT

**Артефакт:** Заготовка репозитория курса → [`outputs/course-kit/`](../outputs/course-kit/)

Мини-кит для старта своего курса: `conventions.py` создаёт минимальный материал и ловит незаполненный стаб, а `CONVENTIONS.md` объясняет выбор структуры по outcome.

## ЧАСТЫЕ ОШИБКИ

- **Считать шесть секций обязательными.** Если блок не влияет на outcome, ошибку, проверку или перенос, его нужно объединить или убрать.
- **Считать «код+тест» обязательными.** Они нужны для исполняемого механизма, но вредны как ритуал в уроке про решение или объяснение.
- **Считать зелёный валидатор доказательством обучения.** Он видит заполненность текста, но не видит скрытые пререквизиты, копирование примера или отсутствие переноса.
- **Проверять структурную комплектацию глазами.** «Вроде всё на месте» не масштабируется на 59 уроков. Гоняй `validate_lesson` (хоть в CI) — он вернёт конкретный список нехватки.
- **Произвольная структура без причины.** Гибкость не отменяет контракт: каждое решение «оставить / объединить / опустить» должно поддерживать результат.
- **Создавать пустой артефакт ради Ship It.** Если отдельный файл не будет переиспользован, достаточно доказанного действия и понятного переноса.
