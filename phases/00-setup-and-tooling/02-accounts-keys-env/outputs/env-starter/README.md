# env-starter

Минимальный набор для безопасной локальной настройки одного API-провайдера.

## Что скопировать в проект

- `.env.example` — имена поддерживаемых переменных без секретов;
- `.gitignore` — исключает локальный `.env`, `.venv` и кэш Python;
- `check_env.py` — проверяет наличие и грубый формат ключа без сети.

## Как использовать

1. Скопируй все три файла в корень Python-проекта.
2. Создай и активируй `.venv`.
3. Создай локальный файл: `cp .env.example .env`.
4. Выбери одного провайдера и заполни только его строку в `.env`.
5. Запусти `python check_env.py`.

Пустые строки не мешают проверке: достаточно одного настроенного провайдера. Скрипт не
печатает значения и не обращается к API. Зелёный результат означает только, что строка
присутствует и похожа на ключ; действительность ключа проверяется первым API-запросом.

Перед добавлением настоящего ключа проверь, что `.env` игнорируется:

```bash
git check-ignore .env
git status --short
```

Если секрет уже попал в коммит, немедленно отзови его у провайдера и выпусти новый.
