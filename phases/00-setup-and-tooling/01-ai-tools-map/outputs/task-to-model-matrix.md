## Самопроверка

- [ ] Описаны три реальные или предложенные в уроке задачи, а не абстрактные «тексты» и «код».
- [ ] У каждой задачи есть проверяемый критерий успеха.
- [ ] Указаны только ограничения, которые действительно повлияли на выбор.
- [ ] Модель, API и готовый продукт не смешаны.
- [ ] Проведён один сравнительный прогон на одинаковых примерах.
- [ ] Наблюдения отделены от предположений.
- [ ] У каждой задачи есть понятное условие эскалации.
- [ ] В публичные сервисы не отправлены конфиденциальные данные.

## Как переиспользовать

Обновляй дату и повторяй прогон после смены модели, продукта, данных или критерия успеха.
В уроках 0.2 и 0.3 сверяйся с полем «Способ работы и кандидат»: если там выбран API
или агент, продолжай с тем же рабочим сценарием; если выбран другой способ, помечай API
или агента как инфраструктуру курса и не переписывай решение задним числом. В Фазе 9
эта карта станет входом для маршрутизации и контроля расходов.
