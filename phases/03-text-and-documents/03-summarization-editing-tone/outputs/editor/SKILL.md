# editor — конвейер «черновик → редактура → тон»

Обрабатывает текст стадиями, каждую можно настроить и оценить отдельно.

## Как пользоваться

1. **Черновик/резюме** — сжать до главного (extractive или промптом «сожми в N пунктов»).
2. **Редактура** — убрать воду, филлеры, повторы; короче и яснее.
3. **Тон** — привести под аудиторию (деловой / нейтральный / дружелюбный).

Офлайн-движок для прототипа:

```bash
python -c "import editing_pipeline as e; \
print(e.run_pipeline('привет очень всем', [e.tighten, lambda t: e.set_tone(t,'formal')]))"
```

В проде каждую стадию заменяет вызов модели (`stage(instruction, text)`), а `run_pipeline` собирает их в цепочку.

## Файлы

- `editing_pipeline.py` — `summarize_extractive`, `tighten`, `set_tone`, `run_pipeline`.

## Правила

- Стадии независимы — меняй и оценивай по одной (eval-harness, урок 2.5).
- Для длинных документов сначала chunking + map-reduce (урок 3.2), потом редактура.
