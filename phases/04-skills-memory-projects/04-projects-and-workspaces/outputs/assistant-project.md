# Артефакт: настроенный ассистент с базой знаний

Рецепт сборки рабочего проекта (workspace) под конкретную роль. Код-прототип — [`code/project.py`](../code/project.py).

## Конфигурация проекта

```
name:           support-assistant
system_prompt:  Ты ассистент поддержки. Отвечай кратко, по делу,
                только на основе базы знаний. Не знаешь — скажи «не знаю».
knowledge:      [политика возврата, гарантия, доставка, FAQ, прайс]
```

## Что куда класть

| Элемент | Назначение | Правило |
|---|---|---|
| **system prompt** | роль, тон, правила, границы | коротко и чётко; не дублировать знания |
| **knowledge** | факты, документы, прайсы | справочные данные, не правила поведения |
| **instructions** | формат ответа, что нельзя | поведение → сюда, не в knowledge |

## Как собрать (через `project.py`)

```python
from project import Project
proj = Project("support", SYSTEM_PROMPT, knowledge=docs)
prompt = proj.build_context("сколько дней на возврат?", k=3)  # system + RAG + запрос
# → отправить prompt в модель
```

## Платформы

- **Claude Projects** — knowledge base + project instructions (RAG авто).
- **Custom GPTs** — instructions + knowledge + capabilities.
- **Gemini Gems** — инструкции + файлы знаний.

## Правила

- Знания — справка, **правила поведения — в инструкции** (частая ошибка: смешать).
- В контекст подмешивай **только релевантное** (retrieve top-k), а не весь корпус (бюджет — урок 4.4).
- Требуй ответ строго по базе знаний и проси «не знаю» при пробелах (урок 1.4).
- Качество ответов меряй eval-harness (урок 2.5).
