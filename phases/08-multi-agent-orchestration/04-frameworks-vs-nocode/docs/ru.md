**Requires (только для USE IT):** аккаунт/инстанс инструмента (LangGraph/n8n/…). BUILD IT работает офлайн, без зависимостей и без сети.

**Это финал Фазы 8 — и развилка выбора.** Свой оркестратор (8.1) понятен, но в проде берут готовое: код-фреймворк, no-code или gateway-платформу. Чтобы выбрать осознанно, а не «по хайпу», надо увидеть, что у всех внутри **одно и то же** — граф процесса.

> **MOTTO.** Под капотом фреймворков и no-code — один и тот же граф: узлы, рёбра, состояние.

## PROBLEM

Перед тобой задача-процесс — несколько шагов с ветвлениями и, может, циклами. Вариантов реализации море: **код-фреймворки** (LangGraph, CrewAI, AutoGen), **no-code/workflow** (n8n), **gateway-платформы** (OpenClaw/Hermes из 4.5), или свой код. Легко взять не тот **класс** инструмента: затащить тяжёлый фреймворк туда, где хватило бы n8n, или наоборот — упереться в потолок no-code на сложной агентной логике.

Ключ к выбору — понять, что под капотом у всех **граф**: узлы-шаги, рёбра-переходы, общее состояние. Соберём этот движок сами (как мини-LangGraph) — и станет видно, что инструменты отличаются не сутью, а удобством и потолком.

## CONCEPT

### Интуиция

Любой процесс — это **блок-схема, ставшая исполняемой**. Кружки (узлы) — шаги, стрелки (рёбра) — переходы между ними, в том числе развилки («если X — туда, иначе сюда») и циклы («повторяй, пока не готово»). По схеме течёт **состояние** — общие данные, которые каждый шаг читает и обновляет.

LangGraph, n8n-флоу и твой оркестратор из 8.1 — это всё один и тот же движок блок-схемы. Разница не в сути, а в обёртке: код-фреймворк даёт максимум контроля и циклы, no-code — визуальный редактор и готовые интеграции. Поняв граф, ты выбираешь инструмент под задачу, а не под моду.

### Как это работает

```
граф процесса = узлы (шаги) + рёбра (переходы, в т.ч. условные/циклы) + state
   └─ это и есть «движок» LangGraph, n8n-флоу, своего оркестратора

выбор класса инструмента:
  from-scratch / SDK   контроль, нет лок-ина, надо писать всё
  код-фреймворк        LangGraph (граф/циклы), CrewAI (роли), AutoGen (диалоги)
  no-code / workflow   n8n (визуально, 400+ интеграций, AI-agent-ноды)
  gateway-платформа    OpenClaw / Hermes (персональный агент, каналы) — 4.5
```

- `add_node(name, fn)` — шаг: функция над состоянием (`state -> state`).
- `add_edge(src, dst)` — безусловный переход.
- `add_conditional(src, router)` — развилка: `router(state)` возвращает имя следующего узла (так делаются ветвления **и** циклы — ребро назад в тот же узел).
- `run(state, max_steps)` — гнать состояние от входа до `END`; `max_steps` — guard от бесконечного цикла (ровно как в 7.4).

## РАЗБОР ПО ШАГАМ

Соберём граф с **циклом**: узел `inc` увеличивает `n`, условие гонит обратно в `inc`, пока `n < 3`, иначе в `done`.

```
вход: inc;  state = {n: 0}

шаг 1: node=inc → n=1; router: 1<3 → "inc"        path=[inc]
шаг 2: node=inc → n=2; router: 2<3 → "inc"        path=[inc,inc]
шаг 3: node=inc → n=3; router: 3<3 ложь → "done"  path=[inc,inc,inc]
шаг 4: node=done → status="готово"; edge done→END  path=[inc,inc,inc,done]
node=END → стоп

итог: state = {n:3, status:"готово"};  путь = [inc,inc,inc,done]
```

Три механики в одном прогоне. **Узел** меняет состояние (`inc` растит `n`). **Условное ребро** делает и ветвление, и **цикл** — `router` вернул имя того же узла `inc`, и граф пошёл по кругу, пока условие не сменило маршрут. **Guard**: будь условие всегда `"inc"` (бесконечный цикл), `max_steps` оборвал бы прогон — без него граф зависнет (7.4). Это и есть ядро LangGraph; он добавляет поверх персистентность, инструменты и параллелизм, но движок — тот же.

## BUILD IT

**Задание: собери мини-граф процесса** (как LangGraph) — узлы, рёбра, условные ветвления и циклы поверх состояния. Только стандартная библиотека, без сети.

> **Перед запуском.** Работай в своей папке курса (`ai-native/8.4-graph/`), а файлы урока клади в подпапку `code/` (по соглашению курса). Нужен только **Python 3** (для теста ещё `pytest`).

Создай файл `mini_graph.py`: константу `END = "__end__"` и класс `Graph`:

- **`add_node(name, fn)`** — узел-функция `state -> state`.
- **`set_entry(name)`** — стартовый узел.
- **`add_edge(src, dst)`** — безусловный переход.
- **`add_conditional(src, router)`** — `router(state)` возвращает имя следующего узла.
- **`run(state, max_steps=100)`** — от `entry`: на каждом шаге выполнить узел (`state = fn(state)`), затем выбрать следующий — `router` (если есть условное ребро), иначе `edge`, иначе `END`. Остановиться на `END` или по `max_steps`. Вернуть `(state, path)`.

**Готово, когда** все тесты в `test_mini_graph.py` зелёные — они проверяют: линейный граф (`a→b`); условное ветвление (`big`/`small` по `state`); цикл с выходом по условию (`inc` ×3); срабатывание `max_steps` на бесконечном цикле.

```bash
pytest code -q           # красное → реализуй Graph → зелёное
python code/mini_graph.py # демо: цикл inc до n=3, потом done
```

**Подсказка.** В `run` держи текущий `node` и список `path`. Порядок выбора следующего узла: сначала `self.cond.get(node)`, потом `self.edges.get(node)`, иначе `END`. `max_steps` — это просто верхняя граница цикла `for`.

## USE IT

Тот же граф — в готовых инструментах (мульти-инструмент):

- **LangGraph** — по доке, «low-level orchestration framework and runtime» для долгоживущих стейтовых агентов: граф узлов/рёбер над общим состоянием, циклы и условные переходы, **durable execution** (checkpointing — агент возобновляется после сбоя), human-in-the-loop, стриминг. Максимум контроля; в проде у Klarna, Uber, J.P. Morgan.
- **CrewAI** — оркестрация по ролям (researcher/analyst/writer), быстрый старт, но более «opinionated».
- **n8n** — визуальный workflow с AI-Agent-нодами (на базе LangChain), 400+ интеграций, MCP, human-in-the-loop; смешивает детерминированные шаги и ИИ.
- **OpenClaw / Hermes** — gateway-платформа: персональный агент с каналами и памятью (4.5).

**Правило выбора:** детерминированная автоматизация с парой ИИ-шагов → **n8n**; сложная агентная логика с циклами и состоянием → **LangGraph/код**; команда по ролям → **CrewAI**; персональный агент с каналами → **OpenClaw/Hermes**.

## ЧАСТЫЕ ОШИБКИ

- **Выбирать инструмент по хайпу, а не по задаче.** Под капотом у всех граф. Тяжёлый фреймворк на простую автоматизацию — оверкилл; no-code на сложную агентную логику — упрёшься в потолок.
- **Тащить фреймворк туда, где хватает n8n.** Детерминированный пайплайн с парой ИИ-шагов визуально собирается в no-code за час. Код-фреймворк тут — лишняя сложность.
- **No-code на сложной агентной логике с циклами.** Когда нужны состояние, циклы, ветвления и контроль — визуальный редактор мешает. Это территория LangGraph/кода.
- **Граф без лимита шагов.** Условные рёбра позволяют циклы — а цикл без выхода зависает. `max_steps` обязателен, как guardrail из 7.4.
- **Считать фреймворк «магией».** LangGraph/n8n — это узлы + рёбра + состояние, ровно как наш `Graph`. Понимаешь движок — отладишь зависание; не понимаешь — будешь гадать.
