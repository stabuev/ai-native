# Урок 8.3 · Координация и контекст

**Фаза 8 — Мульти-агенты и оркестрация** · **Результат фазы:** Построить мульти-агентный процесс с оркестратором и передачей контекста.
<!-- **Requires:** платный API-ключ — только для блока USE IT -->

> **MOTTO.** Несколько агентов над одним состоянием — это про передачу контекста и предотвращение конфликтов.

## PROBLEM

Когда субагенты (8.1) пишут в общий результат, возникает то, чего нет у одиночки: **конфликты** (двое переписали один ключ по-разному) и **потеря контекста** (агент не видит, что сделал другой). Без координации мульти-агентный процесс выдаёт несогласованный результат. Соберём общий контекст с детекцией конфликтов.

## CONCEPT

```
        SharedContext (blackboard)
   agent1 ─ write(facts, ...) ┐
   agent2 ─ read(facts) ──────┤  состояние видно всем
   agent3 ─ write(facts, ...) ┘  ← конфликт, если другой агент и другое значение
```

Координация = общее состояние + правило записи. Anthropic сводит результаты субагентов через **shared memory store** (а не длинные чат-ответы) и реконсиляцию у lead-агента.

## BUILD IT

Общий контекст с передачей состояния и детекцией конфликтов: [`code/coordination.py`](../code/coordination.py).

- `SharedContext.write(key, value, agent)` / `read(key)` — общее состояние с автором;
- детекция конфликта: другой агент перезаписал ключ другим значением → запись в `conflicts`;
- `run_process(steps)` — прогон агентов над общим контекстом.

```bash
python code/coordination.py
pytest code -q
```

Тесты: consumer видит контекст producer'а; конфликт фиксируется при кросс-агентной перезаписи; тот же агент или то же значение — не конфликт.

## USE IT

Прогон мульти-агентного процесса (мульти-провайдер): оркестратор (8.1) + discovery (8.2) + общий контекст.

- **Shared memory store** — субагенты пишут краткие выжимки в общее хранилище, lead сводит (как у Anthropic), а не пересылают длинные ответы.
- **Изолированные контексты** субагентов (Claude Code) + явная передача нужного через контекст — меньше «context rot» (урок 4.4).
- **Реконсиляция у lead** — единая точка сборки и разрешения конфликтов.
