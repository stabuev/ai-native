# Артефакт: мульти-агентный сценарий

Шаблон процесса «оркестратор + субагенты» под задачу. Код — [`code/orchestrator.py`](../code/orchestrator.py).

## Сценарий: подготовка аналитической записки

```
Lead (оркестратор): декомпозирует запрос, делегирует, сводит результат
  ├─ researcher  → собрать факты/данные (свой контекст, инструменты RAG/SQL)
  ├─ analyst     → посчитать метрики и аномалии (Фаза 5)
  ├─ writer      → собрать черновик по фактам + цифрам
  └─ reviewer    → проверить факты, тон, согласованность чисел
Lead: финальная сборка + цитаты
```

## Роли субагентов (карта)

| Роль | Вход | Выход | Инструменты | Модель |
|---|---|---|---|---|
| researcher | тема | факты | поиск, RAG | Haiku (дёшево) |
| analyst | данные | метрики, аномалии | SQL, расчёты | Sonnet |
| writer | факты+метрики | черновик | — | Sonnet |
| reviewer | черновик | замечания | проверка | Opus (сложное) |

## Правила

- **Один субагент — одна роль**; изолированный контекст (меньше шума).
- **Делегируй по сложности**: дешёвая модель на разведку, топ — на сложное (Фаза 9).
- **Параллель** для независимых ролей; последовательно — где есть зависимость.
- **Lead сводит** результаты и разрешает конфликты (урок 8.3).
- Мульти-агент дороже в 4–15× — применяй, когда одиночка не тянет.
