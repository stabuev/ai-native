# Артефакт: презентация защиты + чек-лист компетенций

Структура защиты Capstone и самопроверка освоенного. Метрики — [`code/metrics.py`](../code/metrics.py).

## Структура презентации (7 слайдов)

1. **Проблема** — процесс, его боль, baseline (сколько стоит/занимает сейчас).
2. **Решение** — архитектура из кирпичей курса (агент + MCP + cost-aware + память + мониторинг).
3. **Демо** — живой прогон end-to-end на реальном кейсе.
4. **Метрики «до/после»** — время ↓, стоимость ↓, качество ↑ (с %).
5. **ROI** — (экономия + выгода) / затраты на решение.
6. **Риски и ограничения** — приватность, галлюцинации, предвзятость (Фаза 11) и меры.
7. **Что дальше** — масштабирование, что доработать.

## Таблица эффекта (пример)

| Метрика | До | После | Δ |
|---|---|---|---|
| Время/единица | 10 мин | 3 мин | −70% |
| Стоимость/единица | $0.50 | $0.10 | −80% |
| Качество | 70% | 88% | +25.7% |
| **ROI** | | | **5.5** |

```python
from metrics import before_after, roi
before_after({"время": 10}, {"время": 3}, {"время": True})   # → −70%
roi(before_cost=1000, after_cost=200, value_gained=300)       # → 5.5
```

## Чек-лист компетенций курса

- [ ] LLM изнутри: токены, контекст, сэмплинг, галлюцинации (Ф1)
- [ ] Воспроизводимые промпты + eval-harness (Ф2)
- [ ] Резюмирование/редактура и RAG над корпусом (Ф3)
- [ ] Скиллы, память, проекты, контекст-инжиниринг (Ф4)
- [ ] Данные: NL→SQL, BI-агент, валидация/аномалии (Ф5)
- [ ] Tool use и свой MCP-сервер с доступом (Ф6)
- [ ] Агент: цикл, память, guardrails, отладка поломок (Ф7)
- [ ] Мульти-агентная оркестрация и передача контекста (Ф8)
- [ ] FinOps: роутинг, каскад, кэш/batch, ROI (Ф9)
- [ ] Production: интеграция, мониторинг, телеметрия (Ф10)
- [ ] Этика: приватность, фактчекинг, риски, политика (Ф11)
- [ ] Capstone: спека → сборка → защита с метриками (Ф12)

## Правила честной защиты

- **Реальные замеры**, не «кажется лучше» (eval 2.5, валидация 5.5, телеметрия 10.3).
- **Учти стоимость решения** в ROI, а не только экономию.
- **Проговори риски и ограничения** (Фаза 11) — это часть зрелости, а не слабость.
