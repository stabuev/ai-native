> **MOTTO.** Capstone — это не новый кубик, а сборка изученного в один работающий процесс.

## PROBLEM

Все кирпичи курса собраны (промпты, RAG, агенты, MCP, роутинг, мониторинг). Задача Capstone — соединить их в **один end-to-end процесс**, решающий реальную задачу из спеки (12.1), а не очередное демо. Главное — рабочая интеграция, а не отдельные фрагменты.

## CONCEPT

```
вход → [роутинг по сложности (Ф9)] → выбор модели
     → [guardrail (Ф7)] → пропустить опасное?
     → [инструмент через MCP (Ф6)] → действие
     → [память (Ф4/7)] → состояние процесса
     → [телеметрия токенов/стоимости (Ф10)] → учёт
     → результат
```

Capstone = граф из кирпичей: агентный цикл + cost-aware маршрутизация + память + инструменты + наблюдаемость. Каждый блок уже собран в своей фазе — здесь они работают вместе.

## BUILD IT

End-to-end процесс из кирпичей курса: [`code/solution.py`](../code/solution.py).

- `classify(task)` — роутинг по сложности (Фаза 9);
- `Capstone.run(task)` — guardrail → инструмент → память → телеметрия; возвращает результат + модель + стоимость;
- `total_cost()` — суммарные расходы процесса.

```bash
python code/solution.py
pytest code -q
```

Демо: простая задача → Haiku (дёшево), сложная → Opus; обе записаны в память и телеметрию. Это скелет реального процесса.

## USE IT

Интеграция в рабочий инструмент (мульти-провайдер): замени стабы на реальные блоки курса.

- **Агент** — цикл reason→act→observe (Фаза 7) на Claude Agent SDK / OpenAI Agents SDK.
- **Инструменты/данные** — через MCP-сервер (Фаза 6), доступ к БД/файлам с контролем (6.5, 11.1).
- **Cost-aware** — роутер/каскад (Фаза 9), кэш и batch.
- **Память** — модуль из 4.2; **наблюдаемость** — логи/телеметрия (Фаза 10).
- Встрой за фича-флагом + fallback (10.1) в реальный инструмент команды.
