# Урок 12.1 · Выбор процесса и спецификация

**Фаза 12 — Capstone** · **Результат фазы:** Автоматизировать реальный процесс end-to-end и защитить решение с метриками эффекта.

> **MOTTO.** Capstone начинается не с кода, а со спецификации: цель, измеримые метрики, ограничения, бюджет.

## PROBLEM

«Автоматизирую что-нибудь с ИИ» — путь в никуда: без чёткой цели и **измеримых** метрик нечего защищать и не докажешь эффект «до/после». Нужно выбрать реальный процесс и зафиксировать спецификацию так, чтобы успех был проверяемым.

## CONCEPT

```
CapstoneSpec
  goal        — что автоматизируем (один конкретный процесс)
  metrics[]   — измеримые: baseline («до») + target (цель)  ← без этого нет защиты
  constraints — приватность, human-in-the-loop, безопасность
  budget_usd  — потолок расходов (FinOps, Фаза 9)
        │ validate_spec → готова ли к сборке
```

Хороший выбор процесса: частый, болезненный, с понятной метрикой эффекта (время/стоимость/качество), не критичный для жизни на старте.

## BUILD IT

Спецификация + валидатор измеримости: [`code/spec.py`](../code/spec.py).

- `CapstoneSpec` / `Metric` — структура спеки;
- `is_measurable(metric)` — есть ли baseline и target;
- `validate_spec(spec)` — чего не хватает (цель, измеримые метрики, бюджет).

```bash
python code/spec.py
pytest code -q
```

Валидатор не пропустит спеку без измеримых метрик или бюджета — иначе защиту (12.3) не на чем строить.

## USE IT

Согласование с реальной задачей: возьми **свой** процесс (разбор тикетов, подготовка отчётов, ответы клиентам), зафиксируй baseline руками (сколько времени/денег сейчас), поставь target. Опирайся на:

- **Define success criteria** (Anthropic) — критерии успеха до разработки.
- **Empirical evaluations** — как измерять качество.
- Согласуй ограничения (приватность 11.1, human-in-the-loop 7.3) с реальными процессами.

## SHIP IT

**Артефакт:** Проектная спецификация → [`outputs/capstone-spec.md`](../outputs/capstone-spec.md)

Заполненная спека твоего процесса (цель · метрики baseline/target · ограничения · бюджет). Это вход в сборку решения (12.2) и основа защиты (12.3).
