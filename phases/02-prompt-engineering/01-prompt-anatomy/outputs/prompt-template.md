# Артефакт: карточка проектирования промпта

Карточка помогает превратить расплывчатый запрос в проверяемый рабочий контракт. Это не
анкета с обязательными полями: оставляй только те решения, которые влияют на пригодность
результата.

## 1. Рабочая ситуация

**Какое решение или действие должен поддержать ответ?**

> …

**Кто и как будет использовать результат?**

> …

**Какова цена существенной ошибки?**

> …

## 3. Проектирование контракта

Заполни только применимые строки.

| Возможная часть | Решение для этой задачи | Зачем оно нужно |
|---|---|---|
| Наблюдаемая задача | … | … |
| Вход и граница источника | … | … |
| Существенные ограничения | … | … |
| Критерий успеха | … | … |
| Форма результата | … | … |
| Роль | … / не нужна | … |
| Примеры | … / отложены до 2.2 / не нужны | … |

Проверка целесообразности каждой строки:

> Если удалить эту инструкцию, какой существенный риск вернётся?

Если убедительного ответа нет, сократи промпт.

## 4. Итоговый промпт

Заголовки не обязательны. Запиши контракт так, чтобы его было удобно читать и
переиспользовать.

```text
…
```

## 6. Сравнение версий

Для сравнения используй один инструмент, одну модель или один видимый режим, одинаковый
вход и две новые беседы. Намеренно меняй только текст промпта. Если точная версия модели
скрыта, так и запиши.
