# Артефакт: набор few-shot и CoT заготовок

Готовые шаблоны под типовые задачи. Сборщики — в [`code/few_shot.py`](../code/few_shot.py).

## Few-shot: классификация

```
Классифицируй тональность отзыва: pos / neg / neutral.

Вход: обожаю этот продукт
Выход: pos

Вход: ужасная поддержка
Выход: neg

Вход: {новый отзыв}
Выход:
```

## Few-shot: извлечение в формат

```
Извлеки сущности в виде «поле: значение».

Вход: Иван Петров, заказ #123, срочно
Выход: имя: Иван Петров; заказ: 123; приоритет: high

Вход: {новый текст}
Выход:
```

## Chain-of-thought: рассуждение

```
Реши задачу. Думай по шагам, затем дай финальный ответ после «Ответ:».

Задача: {условие}
```

## CoT few-shot (примеры с рассуждением)

```
Вопрос: В корзине 3 яблока, добавили 2×2. Сколько стало?
Рассуждение: 2×2=4; 3+4=7.
Ответ: 7

Вопрос: {новый вопрос}
```

## Правила

- **Few-shot** — когда важен формат/класс: 2–5 примеров, покрывающих разные случаи (и граничные).
- **CoT** — когда задача многошаговая (арифметика, логика): проси «думай по шагам».
- Примеры должны быть **согласованы по формату** — модель копирует их разметку.
- Не раздувай: лишние примеры жгут токены (бюджет — урок 1.1) и не всегда помогают.
- Эффект **измеряй** на наборе кейсов (eval-harness, урок 2.5), а не на одном примере.
