## PROBLEM

На задачах с нестандартным форматом или многошаговым рассуждением «просто спросить» сбоит: модель угадывает разметку, путает классы, перескакивает к неверному ответу. Дообучать модель ради этого дорого и медленно. Нужны приёмы, которые поднимают качество **здесь и сейчас, прямо в промпте** — и их два: few-shot и chain-of-thought.

## CONCEPT

### Интуиция

Принцип — **«покажи, а не объясняй»**. Описать желаемый формат словами трудно и неоднозначно; пара примеров «вход → выход» передаёт его точнее любого описания. А для задач, где надо посчитать или связать несколько фактов, помогает дать модели **подумать вслух**: промежуточные шаги резко снижают шанс прыгнуть к неверному ответу.

### Как это работает

```
zero-shot : задача → ответ                          (модель угадывает формат)
few-shot  : пример, пример, пример → задача → ответ  (учится по образцам)
CoT       : задача → «думай по шагам» → шаги → ответ (разворачивает рассуждение)
```

- **Few-shot — это in-context learning.** Модель «подхватывает» паттерн из 2–5 примеров прямо в промпте, **без обновления весов** — именно это показали Brown et al. (2020) на GPT-3: большая модель учится задаче по нескольким примерам во время вывода. Лучше всего работает там, где важны формат и семантика метки: классификация, разметка, извлечение, единый стиль ответа.
- **CoT — это рассуждение по шагам.** Если попросить модель сначала расписать промежуточные шаги, а потом дать ответ, точность на арифметике, логике и многошаговых задачах заметно растёт (Wei et al., 2022). Reasoning-модели из урока 1.2 делают это автоматически (скрытые thinking-токены), но и обычную модель можно «развязать» фразой «думай по шагам».
- **Когда что.** Few-shot — когда нужен формат/семантика; CoT — когда нужно рассуждение. Их можно сочетать (few-shot CoT — примеры, где показано само рассуждение).

Важно: примеры должны быть **качественными и разнообразными** (модель копирует и ошибки тоже), а их число — в меру: каждый пример занимает место в контекстном окне (урок 1.2) и стоит токенов.

## РАЗБОР ПО ШАГАМ

Покажем численно, **зачем** примеры, на игрушечной классификации тональности. Обучающие примеры (вход → метка):

```
"обожаю прекрасный фильм"      → pos
"чудесный замечательный день"  → pos
"ненавижу отвратительную еду"  → neg
"ужасный кошмарный сервис"     → neg
```

Тест: `"обожаю прекрасный сериал" → pos`, `"ужасный кошмарный продукт" → neg`.

**Zero-shot** (без примеров) — модель «угадывает», в нашей игрушке это дефолтная метка `neg`. Тогда из двух тестовых она угадает только второй:

```
accuracy(zero-shot) = 1 / 2 = 0.5
```

**Few-shot** — берём метку самого похожего примера (похожесть по доле общих слов, Жаккар):

- `"обожаю прекрасный сериал"` ближе всего к `"обожаю прекрасный фильм"` (2 общих слова из 4 → 0.5) → метка **pos** ✓;
- `"ужасный кошмарный продукт"` ближе к `"ужасный кошмарный сервис"` (0.5) → метка **neg** ✓.

```
accuracy(few-shot) = 2 / 2 = 1.0
```

Те же данные, разница 0.5 → 1.0 — только за счёт того, что в промпт положили примеры. Это и есть in-context learning в миниатюре (на реальной модели вместо Жаккара — её собственное понимание).

## BUILD IT

**Задание: собери few-shot/CoT-промпты и измерь ценность примеров** — стандартная библиотека, без сети.

> **Перед запуском.** Работай в своей папке курса (`ai-native/2.2-few-shot-cot/`), а файлы урока клади в подпапку `code/` (по соглашению курса). Нужен только **Python 3**, всё офлайн (для теста ещё `pytest`).

Создай `few_shot.py` и реализуй:

- **`format_few_shot(instruction, examples, query)`** — собери промпт: инструкция, затем каждый пример как `Вход: {inp}\nВыход: {out}`, и в конце `Вход: {query}\nВыход:` (хвост оставь открытым — модель его продолжит). `examples` — список пар `(вход, выход)`.
- **`format_cot(instruction, query)`** — верни промпт, который просит рассуждать пошагово перед ответом (например «Думай по шагам, затем дай ответ»).
- **`_similarity(a, b)`** — похожесть строк по Жаккару: доля общих слов = `|пересечение| / |объединение|` множеств слов.
- **`knn_predict(query, examples)`** — few-shot аналог: верни метку примера с максимальной `_similarity` к `query` (на пустом списке — `None`).
- **`zero_shot_predict(query, default)`** — без примеров верни `default`.
- **`accuracy(predict_fn, dataset)`** — доля верных предсказаний на наборе пар `(вход, метка)`.

**Готово, когда** тест `test_few_shot.py` зелёный — он проверяет, что few-shot промпт содержит примеры и запрос и кончается на `Выход:`, что CoT просит шаги, что `knn_predict` берёт самый похожий пример, и что few-shot бьёт zero-shot (1.0 против 0.5).

```bash
pytest code -q             # красное → реализуй функции → зелёное
python code/few_shot.py    # демо: few-shot промпт + accuracy few-shot vs zero-shot
```

**Подсказка.** `_similarity`: переведи строки в `set(a.split())` и считай `len(sa & sb) / len(sa | sb)`. `knn_predict`: `max(examples, key=lambda e: _similarity(query, e[0]))[1]`.

## USE IT

Сравни качество с/без примеров на реальной модели (мульти-провайдер): прогони один кейс в трёх режимах — zero-shot, few-shot, CoT — и сравни ответы.

```python
zero = "Классифицируй тональность: 'ужасный сервис'"
few  = format_few_shot("Классифицируй тональность.", EXAMPLES, "ужасный сервис")
cot  = format_cot("Классифицируй тональность.", "ужасный сервис")
for prompt in (zero, few, cot):
    Anthropic().messages.create(model="claude-haiku-4-5", max_tokens=100,
        messages=[{"role": "user", "content": prompt}])
    OpenAI().responses.create(model="gpt-5.x", input=prompt)
    genai.Client().models.generate_content(model="gemini-2.x-flash", contents=prompt)
```

На простой задаче разницу видно на глаз; **системно** мерить долю верных на наборе кейсов будем в уроке 2.5 (eval-harness).

## ЧАСТЫЕ ОШИБКИ

- **Few-shot с плохими примерами.** Модель копирует и формат, и ошибки. Примеры должны быть аккуратными, однозначными и покрывать разные классы.
- **Заваливать промпт десятками примеров.** Каждый пример занимает контекст (урок 1.2) и стоит токенов. Обычно хватает 2–5 хорошо подобранных.
- **CoT там, где нужен только формат.** Для простой классификации рассуждение вслух лишь раздувает ответ; CoT — для многошаговых задач (счёт, логика).
- **Сравнивать режимы на одном примере и делать выводы.** Один кейс — это анекдот, а не метрика. Меряй долю верных на наборе (урок 2.5).

## Дополнительное чтение

- [Brown et al., 2020 — Language Models are Few-Shot Learners (GPT-3)](https://arxiv.org/abs/2005.14165) — откуда взялся few-shot (in-context learning).
- [Wei et al., 2022 — Chain-of-Thought Prompting](https://arxiv.org/abs/2201.11903) — первоисточник CoT.
- [anthropics/prompt-eng-interactive-tutorial](https://github.com/anthropics/prompt-eng-interactive-tutorial) — практические главы про примеры и пошаговое мышление.
- [Wang & Zhou, 2024 (NeurIPS) — Chain-of-Thought Reasoning Without Prompting](https://arxiv.org/abs/2402.10200) — CoT-decoding: цепочки рассуждений уже «зашиты» в модель, их достаёт перебор top-k веток, а не промпт.
- [DeepSeek-R1, 2025 (Nature) — Incentivizing Reasoning via RL](https://arxiv.org/abs/2501.12948) — эра reasoning-моделей: CoT, выученный через RL, «aha moment».
- [Anthropic, 2025 — Reasoning Models Don't Always Say What They Think](https://arxiv.org/abs/2505.05410) — важный нюанс: озвученная цепочка ≠ реальная причина ответа (faithfulness).
- [Cameron R. Wolfe — Demystifying Reasoning Models (2025)](https://cameronrwolfe.substack.com/p/demystifying-reasoning-models) — практический мост CoT → o1/R1 и test-time compute.
- [Wang et al., 2022 — Self-Consistency Improves CoT](https://arxiv.org/abs/2203.11171) — по-прежнему стандартный приём: сэмплируй несколько цепочек, бери консенсус.
