# Артефакт: чек-лист доверия к ответу

Быстрая проверка перед тем, как поверить ответу модели и тем более встроить его в работу. Механика проверки — в [`code/hallucination_probes.py`](../code/hallucination_probes.py).

## Когда повышенный риск галлюцинации

- [ ] Вопрос про **конкретные факты/числа/даты/цитаты** (имена, статистика, ссылки).
- [ ] Возможна **ложная предпосылка** в самом вопросе.
- [ ] Тема **за пределами знаний** модели (свежие события, будущее, выдуманные сущности).
- [ ] Ответ звучит уверенно, но **источников нет**.
- [ ] Высокая цена ошибки (юридич./финанс./медицина).

## Что сделать с промптом

- [ ] Явно **разреши «не знаю»**: «если не уверен — так и скажи, не выдумывай».
- [ ] Требуй **цитаты/источники** для каждого фактического утверждения.
- [ ] Для длинных документов — сначала **дословные цитаты**, потом вывод (grounding).
- [ ] Ограничь **только предоставленными данными** («не используй общие знания»).
- [ ] Снизь **temperature** для фактических задач (там, где параметр доступен).

## Как проверить ответ

- [ ] **Перекрёстно**: тот же вопрос на другой модели/прогоне — расхождения = красный флаг.
- [ ] **Сверка с источником**: факты подтверждаются цитатой из текста/ссылкой.
- [ ] **Граница**: на неотвечаемое модель воздержалась, ложную предпосылку оспорила.
- [ ] **Человек на критичном**: высокие ставки → ручная валидация обязательна.
