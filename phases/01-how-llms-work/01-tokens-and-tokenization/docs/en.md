# Урок 1.1 · Токены и токенизация

**Фаза 1 — Как работают LLM** · **Результат фазы:** Объяснить токены, контекст и inference; собрать токен-бюджетер; отличать надёжный ответ от галлюцинации.

> **MOTTO.** Модель видит не слова и не символы, а токены — и платишь ты тоже за токены.

## PROBLEM

«Сколько это будет стоить и влезет ли в контекст?» — на оба вопроса нельзя ответить, считая слова или символы. Модель работает с токенами, а их число не равно ни тому, ни другому: `tokenization` может быть 1 токеном, а `ทดสอบ` — десятком. Пока не понимаешь, откуда берутся токены, бюджет контекста и счёт за API остаются магией.

## CONCEPT

Токенизация разбивает текст на куски по выученной статистике, а не по словарю языка. Самый наглядный механизм — **BPE (byte-pair encoding)**: начинаем с отдельных символов и итеративно сливаем самую частую соседнюю пару в один токен.

```
"low lower lowest"
 l o w </w>  l o w e r </w>  l o w e s t </w>
        │ частая пара (l,o) → слияние
 lo w </w>   lo w e r </w>   lo w e s t </w>
        │ (lo,w) → слияние
 low </w>    low e r </w>    low e s t </w>   ...
```

Частые куски («low», «er») получают свой токен, редкие распадаются на мелкие части. Поэтому частотный текст «дешевле» по токенам, чем редкий.

## BUILD IT

Реализован мини-BPE с нуля, без зависимостей: [`code/tokenizer.py`](../code/tokenizer.py).

- `train_bpe(corpus, num_merges)` — учит merge-правила по частоте пар;
- `encode(text, merges)` / `decode(tokens)` — кодирование и обратный разбор.

Запуск демо и тестов:

```bash
python code/tokenizer.py
pytest code -q
```

Демо показывает ключевую интуицию: чем больше выучено merge-правил, тем меньше токенов на тот же текст, а `decode(encode(x)) == x`.

## USE IT

То же самое — через продакшн-токенизатор. <!-- **Requires:** опциональная библиотека, без сети -->

```bash
pip install tiktoken
```

```python
import tiktoken
enc = tiktoken.get_encoding("o200k_base")
text = "tokenization is not word counting"
print(len(enc.encode(text)), "токенов")     # реальный счётчик
```

Сравни: наш учебный токенизатор и продакшн дают разное число токенов, но принцип один — слияние частых пар. Для оценки бюджета всегда считай реальным токенайзером модели.
