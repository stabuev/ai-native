# Урок 5.4 · BI-агенты

**Фаза 5 — Данные и аналитика** · **Результат фазы:** Анализировать данные с ИИ, делать NL→SQL и автоотчёты с валидацией результатов.
<!-- **Requires:** платный API-ключ — только для блока USE IT -->

> **MOTTO.** BI-агент не отвечает одним запросом, а ведёт расследование: обзор → разбивка → причина.

## PROBLEM

На вопрос «почему упала выручка?» один SQL не ответит — нужно **расследование**: посмотреть общий тренд, разбить по измерениям, найти сегмент-виновник, копнуть глубже. BI-агент планирует и исполняет такую цепочку шагов сам. Соберём планировщик, чтобы понять, как агент выбирает следующий шаг.

## CONCEPT

```
вопрос → policy выбирает шаг → исполнить → наблюдение → следующий шаг
   overview → breakdown(по измерению) → top_segment(доля) → [drill-down]
```

Это агентный цикл (reason→act→observe из Фазы 7), но над аналитикой: каждый шаг сужает поиск к причине. Policy здесь детерминированная; в проде шаги выбирает LLM.

## USE IT

Готовые BI-агенты (обзор и практика):

- **Databricks AI/BI Genie** — NL-вопросы к данным в Unity Catalog: возвращает SQL, таблицу и график, объясняет ход; есть Managed MCP.
- **ThoughtSpot Spotter** — агент-аналитик поверх семантической модели (токены вместо прямой генерации SQL), human-in-the-loop коучинг.
- **Hex** — Notebook Agent: NL→SQL/Python в ноутбуке с диффом и ревью ячеек.

Общее: семантический слой (5.3) + планирование шагов + объяснимость и governance. ⚠️ Агент ускоряет, но выводы проверяет человек (5.5).
