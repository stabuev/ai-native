# Артефакт: шаблон аналитического запроса

Структурированный запрос к ИИ-аналитику — чтобы ответы были воспроизводимы и проверяемы. Механика плана — [`code/analyze.py`](../code/analyze.py).

## Шаблон

```
# Данные
{что за таблица, какие колонки и типы, источник, период}

# Вопрос
{бизнес-вопрос одним предложением}

# Метрика
{что считаем: SUM(amount) = выручка, COUNT(*) = заказы, AVG = средний чек}

# Разрез
{по чему группируем: регион / месяц / продукт; и фильтры}

# Формат ответа
{таблица топ-N / график / число + вывод одним абзацем}

# Проверка
{покажи код/SQL; проверь, что строки не потеряны и суммы сходятся}
```

## Пример

```
# Данные: sales(region, product, amount) за Q2
# Вопрос: какие регионы дают основную выручку?
# Метрика: выручка = SUM(amount)
# Разрез: по region, топ-5
# Формат: таблица регион→выручка + доля от总
# Проверка: сумма по регионам = общей выручке
```

## Правила

- **Метрику определяй явно** — иначе ИИ выберет свою (семантический слой, урок 5.3).
- **Проси показать код/SQL** — это аудируемость (Code Interpreter, урок 5.2).
- **Всегда добавляй «Проверку»** — суммы, пропуски, аномалии (урок 5.5).
- Считай токены на больших данных (урок 1.1) и сэмплируй, если таблица огромна.
