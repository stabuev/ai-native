# Артефакт: план интеграции ИИ для команды

Где и как встроить ИИ в существующий процесс безопасно. Код-обёртка — [`code/integration.py`](../code/integration.py).

## Шаги плана

1. **Выбрать точку встраивания** — где ИИ даёт ценность: черновик ответа, классификация тикета, саммари, извлечение.
2. **Определить fallback** — детерминированный запасной путь (шаблон, правило, ручной режим).
3. **Фича-флаг** — включение по сегментам/проценту трафика; мгновенный откат.
4. **Доступ к данным** — ИИ видит только то, что разрешено пользователю (как Copilot/Graph).
5. **Мониторинг** — error_rate, latency, токены/стоимость (уроки 10.2–10.3).
6. **Rollout** — пилот → расширение → дефолт; на каждом этапе сверка метрик.

## Шаблон встраивания (код)

```python
from integration import AIFeature
feature = AIFeature(fn=call_model, fallback=old_logic, enabled=flags.ai_summary)
result = feature.run(record)        # упадёт ИИ — вернётся old_logic, процесс жив
```

## Чек-лист готовности к интеграции

- [ ] Есть fallback на случай сбоя/выключения ИИ.
- [ ] Фича за флагом, откат за секунды.
- [ ] Доступ к данным ограничен правами пользователя.
- [ ] Логирование и алерты настроены (10.2).
- [ ] Токены/стоимость считаются на процесс (10.3, FinOps 9.4).
- [ ] Качество измеряется (eval, урок 2.5).

## Платформы

- **M365 Copilot** / **Gemini for Workspace** — готовое встраивание в офисные приложения.
- **Свой инструмент** — через API с обёрткой `AIFeature` + мониторинг.
