# lesson-author — автор уроков курса AI Native

Скилл помогает писать уроки единообразно по каркасу курса (см. `CLAUDE.md` и `README.md`).

## Аргумент

ID урока, напр. `2.3`. Найди его в `README.md` (название, Build It / Use It / Ship It, папку).

## Шаги

1. **Определи папку** из колонки «Папка» в README. Создай `code/`, `docs/`, `outputs/` если их нет.
2. **Напиши `docs/ru.md`** по шаблону ниже — все 6 шагов, на русском.
3. **Build It → `code/`.** Реализуй механизм урока с нуля, без внешних фреймворков. Должно запускаться на CPU офлайн.
4. **Тест → `code/test_<name>.py`.** Минимум один проходящий юнит-тест на ключевое поведение.
5. **Use It.** Покажи то же через реальный инструмент/API. Платный ключ → пометь `**Requires:**` в шапке урока.
6. **Ship It → `outputs/`.** Положи артефакт, указанный в колонке Ship It (prompt/skill/agent/MCP), и опиши переиспользование.
7. **Проверь DoD** (см. CLAUDE.md) и прогони `pytest` по папке урока.

## Шаблон docs/ru.md

```markdown
# Урок <id> · <название>

**Фаза <N> — <фаза>** · **Результат фазы:** <outcome>
<!-- **Requires:** платный API-ключ <провайдер> — только если применимо -->

> **MOTTO.** <идея в одну строку>

## PROBLEM
<конкретная боль>

## CONCEPT
<интуиция, диаграмма ASCII/mermaid>

## BUILD IT
<что собираем с нуля; ссылка на code/>

## USE IT
<то же через реальный инструмент/API>

## SHIP IT
**Артефакт:** <из колонки Ship It> → `outputs/`
<как переиспользовать>

---
**Часы:** ~<n> · **DoD:** код запускается, тест зелёный, ru.md заполнен.
```

## Правила

- Никаких заглушек — только рабочий код.
- Объяснения по-русски, код по-английски, минимум зависимостей.
- Не меняй зафиксированные слаги папок.
- Образец: `phases/01-how-llms-work/01-tokens-and-tokenization/`.
