# find-your-level — входная диагностика и маршрут

Помоги студенту выбрать глубину прохождения курса. Это не квалификационный экзамен:
не присваивай единый ярлык «новичок/инженер» и не объявляй тему освоенной только по
самооценке.

## Что измеряем

Диагностика состоит из двух разных источников:

1. **Самооценка по наблюдаемым действиям** — 12 вопросов, по одному на фазы 0–11.
2. **Сценарная форма A** — 13 коротких задач по результатам каждого урока контрольных
   фаз 1, 4 и 8. Она использует ту же карту компетенций, что последующая
   `/check-understanding`, но другие ситуации.

Capstone (фаза 12) проходят все. Самооценка помогает сократить повторение, но только
сценарное доказательство позволяет перейти сразу к контрольной попытке фазы.

## Как проводить

### Шаг 1. Самооценка

Возьми `SELF_ASSESSMENT` из `find_your_level.py`. Задавай вопросы небольшими порциями,
показывай варианты `0..3` полностью. Не заменяй поведенческие формулировки названиями
уровней и объясняй незнакомое слово прямо в вопросе.

Сохрани 12 выбранных индексов в исходном порядке.

### Шаг 2. Входные сценарии

Возьми `BASELINE_BANKS[1]`, затем `[4]` и `[8]`. Задай все 13 сценариев, не показывая
поле `correct`. После ответа дай короткое объяснение; по ошибке используй
`misconception`, чтобы назвать конкретную неверную модель, а не личное качество
студента.

Сохрани ответы отдельно по фазам:

```python
baseline_answers = {
    1: [3, 0, 1, 2],
    4: [0, 2, 1, 3, 3],
    8: [1, 0, 3, 2],
}
```

### Шаг 3. Маршрут

Передай ответы в детерминированный офлайн-движок:

```bash
python -c "import find_your_level as f, json; \
self_answers=[1,1,2,1,2,1,1,0,1,0,0,1]; \
baseline={p:[q['correct'] for q in b] for p,b in f.BASELINE_BANKS.items()}; \
print(json.dumps(f.route(self_answers, baseline), ensure_ascii=False, indent=2))"
```

Покажи:

- профиль и пояснение, что это маршрут, а не квалификация;
- план по 13 фазам и причину каждой рекомендации;
- расхождения самооценки и сценариев из `calibration`;
- пробелы формы A из `baseline[*].gaps`;
- ориентировочные часы как диапазон планирования, а не обещание.

## Значение действий

- `focus` — пройти фазу полностью.
- `refresh` — быстро восстановить модель, затем выполнить практику и проверку; примерно
  половина базового времени.
- `challenge` — начать с выходной проверки и артефакта фазы; примерно четверть времени.
  Ошибка возвращает соответствующий урок в маршрут.

Действия `skip` нет. Для фаз без сценарной формы высокая самооценка даёт не больше чем
`refresh`. Capstone всегда `focus`.

## Что сохранить

Предложи сохранить маршрут и поле `baseline_competencies` в
`outputs/personal-route.md` или рядом в JSON. Эти значения понадобятся
`/check-understanding`, чтобы показать изменение по тем же компетенциям.

## Файл

- `find_your_level.py` — вопросы, входная форма A, `self_profile()`,
  `grade_baseline()` и `route()`; работает офлайн без зависимостей.
