# CLAUDE.md — AI Native Course

Это репозиторий курса **AI Native: From Scratch to Agentic Systems**. Здесь ты помогаешь *разрабатывать* курс: писать уроки, код и артефакты в едином формате.

## Что это за курс

Технический, self-paced, **без видео**. Аудитория умеет немного программировать. Полная программа — в `README.md` (13 фаз, 59 уроков). Каждая фаза надстраивается над предыдущей.

## Главный принцип: Build It / Use It / Ship It

Каждый урок сначала строит механизм **с нуля** (Build It, без фреймворков), затем запускает то же через production-инструмент (Use It), и отдаёт **артефакт** (Ship It: prompt · skill · agent · MCP). Студент понимает инструмент, потому что собрал его уменьшенную версию.

**Build It — не догма.** Код и тест добавляем там, где сборка механизма реально что-то объясняет. Если урок концептуальный/обзорный (setup, карта инструментов и т.п.) или механизм тривиален либо дублирует более поздний урок — допустим урок **без `code/`**, либо ужатый до честного полезного куска. Заполненный `docs/ru.md` и артефакт (Ship It) обязательны всегда.

**Build It — упражнение, а не листинг.** Где есть содержательный код, оформляй BUILD IT как задачу: спека (сигнатуры функций) + тесты как ТЗ + «попробуй сам», а **эталонное решение студент видит под катом**. Технически: поставь маркер `<!-- exercise -->` в `docs/ru.md` — сайт тогда раскроет тесты (ТЗ) и **свернёт** код-решение в блок «Показать решение». Решение всегда в одном клике (модель поощряет, а не заставляет). Тривиальные/концептуальные уроки маркер не ставят — у них read-and-run.

**Задачу ставь подробно** — чтобы студенту было ясно, что сделать: «Задание» одной строкой, какой файл создать, **контракт каждой функции** (вход → что делает → выход/инвариант), критерий «Готово, когда тесты зелёные», команды запуска (red→green), короткая подсказка и ссылка `[эталонное решение](#lesson-files)`. Образец — урок 1.1.

**Три пути пройти упражнение.** Любое exercise-упражнение можно (1) собрать самому, (2) подсмотреть эталон, (3) **делегировать ИИ** — написать промпт и самому проверить результат. Это центральная AI-native идея и практика AI Fluency 4D (Delegation · Description · Discernment · Diligence); делегирование — отдельный навык, не шорткат, тесты — критерий приёмки на всех путях. Для exercise-уроков сайт **сам** показывает блок «Три пути» и кнопку «Скопировать промпт для ИИ» (собирает спеку из BUILD IT + тесты) — в `docs/ru.md` это писать не нужно, достаточно маркера `<!-- exercise -->`.

## Структура урока (строго)

```
phases/<NN>-<phase-slug>/<NN>-<lesson-slug>/
├── code/        # Build It (с нуля) + Use It (через инструмент) + тесты
├── docs/ru.md   # разбор по 6 шагам
└── outputs/     # артефакт урока
```

Слаги фаз и уроков уже зафиксированы в README (колонка «Папка»). Не переименовывай. `code/` может отсутствовать у концептуальных уроков — тогда остаются `docs/` и `outputs/`.

## Цикл из 6 шагов в docs/ru.md

1. **MOTTO** — идея урока в одну строку.
2. **PROBLEM** — конкретная боль.
3. **CONCEPT** — интуиция и диаграммы (ASCII/mermaid).
4. **BUILD IT** — реализация с нуля в `code/`, без внешних фреймворков, запускается на CPU офлайн.
5. **USE IT** — то же самое через реальный инструмент/API. Если нужен платный ключ — пометь `**Requires:**` в начале урока.
6. **SHIP IT** — положи артефакт в `outputs/` и опиши, как его переиспользовать.

## Definition of Done для урока

- `docs/ru.md` заполнен (6 шагов; у уроков без кода шаг BUILD IT может быть обзорным или опущенным).
- В `outputs/` лежит обещанный артефакт (см. колонку Ship It в README).
- **Если в уроке есть Build It-код:** в `code/` рабочая реализация и, **где это осмысленно**, юнит-тест `test_*.py`, который проходит; прогон `pytest phases/<...>/code` зелёный. Тривиальный код (данные, простая формула) можно оставить без теста.
- Код (если есть) запускается на CPU без платных ключей (исключение помечается `**Requires:**`).

## Как писать новый урок

1. Найди урок в README по id (напр. 2.3) и его папку.
2. Вызови скилл: `/lesson-author 2.3` (см. `.claude/skills/lesson-author`).
3. Где урок требует механизма — заполни Build It реальным кодом (и тестом, где он осмыслен); концептуальные уроки могут обойтись без `code/`. Use It — примером через инструмент, Ship It — артефактом.
4. Прогони тесты; при выполненном DoD автообнови `PROGRESS.md` (`[ ]`→`[x]`) и добавь `✅ **Урок готов**` в строку DoD урока.
5. Коммить **после завершения всей фазы** (не каждый урок): `git commit -m "Phase <N> (<slug>): lessons complete"`. Пуш — за пользователем.
