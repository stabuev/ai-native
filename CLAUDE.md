# CLAUDE.md — AI Native Course

Это репозиторий курса **AI Native: From Scratch to Agentic Systems**. Здесь ты помогаешь *разрабатывать* курс: писать уроки, код и артефакты в едином формате.

Педагогический источник правды — [`LESSON_CONTRACT.md`](LESSON_CONTRACT.md). Шаблон и
автоматические проверки помогают соблюдать форму, но не заменяют контракт результата.

## Что это за курс

Технический, self-paced, **без видео**. Аудитория умеет немного программировать. Полная программа — в `README.md` (13 фаз, 59 уроков). Каждая фаза надстраивается над предыдущей.

## Главный принцип: Build It / Use It / Ship It

Build It / Use It / Ship It — конструктор учебного перехода, а не обязательное оглавление.
Используй только те элементы, которые поддерживают outcome конкретного урока. Заголовки
можно переименовывать, объединять и опускать; решение проверяется по целесообразности в
[`LESSON_CONTRACT.md`](LESSON_CONTRACT.md).

**Build It — не догма.** Код и тест добавляем там, где сборка механизма реально что-то
объясняет. **Use It** нужен, только если перенос в реальный инструмент отличается от
основной практики. **Ship It** нужен, только если отдельный переиспользуемый артефакт
естественно следует из результата. Обязателен заполненный `docs/ru.md`; `code/`,
`outputs/`, квиз и материалы — по необходимости.

**Build It — упражнение, а не листинг.** Где есть содержательный код, оформляй BUILD IT как задачу: спека (сигнатуры функций) + тесты как ТЗ + «попробуй сам», а **эталонное решение студент видит под катом**. Технически: поставь маркер `<!-- exercise -->` в `docs/ru.md` — сайт тогда раскроет тесты (ТЗ) и **свернёт** код-решение в блок «Показать решение». Решение всегда в одном клике (модель поощряет, а не заставляет). Тривиальные/концептуальные уроки маркер не ставят — у них read-and-run.

**Задачу ставь подробно** — чтобы студенту было ясно, что сделать: «Задание» одной строкой, какой файл создать, **контракт каждой функции** (вход → что делает → выход/инвариант), критерий «Готово, когда тесты зелёные», команды запуска (red→green), короткая подсказка и ссылка `[эталонное решение](#lesson-files)`. Образец — урок 1.1.

**Три пути пройти упражнение.** Любое exercise-упражнение можно (1) собрать самому, (2) подсмотреть эталон, (3) **делегировать ИИ** — написать промпт и самому проверить результат. Это центральная AI-native идея и практика AI Fluency 4D (Delegation · Description · Discernment · Diligence); делегирование — отдельный навык, не шорткат, тесты — критерий приёмки на всех путях. Для exercise-уроков сайт **сам** показывает блок «Три пути» и кнопку «Скопировать промпт для ИИ» (собирает спеку из BUILD IT + тесты) — в `docs/ru.md` это писать не нужно, достаточно маркера `<!-- exercise -->`.

## Физическая структура

```
phases/<PP>-<phase-slug>/<LL>-<lesson-slug>/
├── docs/ru.md   # обязательный связный материал урока
├── code/        # опционально: исполняемая практика и тесты
├── outputs/     # опционально: переиспользуемый артефакт
└── quiz.json    # опционально: квиз, если он уместен как проверка
```

`PP` — номер фазы, `LL` — порядковый номер урока внутри фазы; оба записываются двумя
цифрами. Для id `4.3` ожидается путь `phases/04-<phase>/03-<lesson>/`.

Слаги и порядковые номера зафиксированы в README (колонка «Папка»). Не меняй их без
согласованного изменения программы и всех ссылок.
Опциональный элемент не создавай ради пустой директории или единообразия.

## Конструктор урока

Возможные блоки: `MOTTO`, `PROBLEM`, `CONCEPT`, `BUILD IT`, `USE IT`, `SHIP IT`, ошибки,
проверка себя и материалы. Для каждого реши: оставить, объединить с соседним фрагментом
или опустить. Не оставляй секцию, которая повторяет сказанное, вводит лишний механизм или
существует только ради шаблона.

Независимо от заголовков студент должен увидеть связный переход от исходной ситуации к
новому действию, способу проверки и применению за пределами примера.

## Definition of Done для урока

Полный DoD задан в [`LESSON_CONTRACT.md`](LESSON_CONTRACT.md). Коротко, готовый урок:

- обещает наблюдаемое действие студента и занимает оправданное место в маршруте;
- не содержит скрытых пререквизитов;
- ведёт от проблемы и механизма к самостоятельному действию, реалистичной ошибке,
  проверке и переносу;
- использует только целесообразные секции, практику и форму проверки;
- даёт переносимый результат; отдельный артефакт создаётся только когда он полезен;
- проходит структурные и технические проверки, соразмерные результату урока.

`validate_lesson()` проверяет только наличие заполненного материала без следов стаба. Он
намеренно не требует конкретные секции, код, квиз или артефакт: это решает аудит.

## Как писать новый урок

1. Найди урок в README по id (напр. 2.3) и его папку.
2. Вызови скилл: `/lesson-author 2.3` (см. `.claude/skills/lesson-author`).
3. Выбери целесообразные элементы конструктора. Код, отдельный Use It, артефакт, квиз и
   материалы добавляй только если они поддерживают результат урока.
4. Прогони тесты; при выполненном DoD автообнови `PROGRESS.md` (`[ ]`→`[x]`) и добавь `✅ **Урок готов**` в строку DoD урока.
5. Коммить **после завершения всей фазы** (не каждый урок): `git commit -m "Phase <N> (<slug>): lessons complete"`. Пуш — за пользователем.
