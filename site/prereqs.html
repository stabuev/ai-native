Требования и формат

Курс технический и self-paced (без видео): материалы текстовые и кодовые, акцент на самостоятельную сборку и автопроверку.

Входные требования

  • Python на уровне написания скриптов (функции, файлы, пакеты, venv).
  • Базовая работа в терминале и Linux (команды, переменные окружения).
  • git — базовое понимание (что такое коммит) пригодится, но не обязательно: курс проходится на сайте, репозиторий клонировать не нужно.
  • Школьная математика; векторы и вероятности — плюс, но не обязательно.
  • Аккаунты и API-ключи провайдеров нужны только для шага Use It (заводятся в Фазе 0); весь Build It работает офлайн и бесплатно.
  • Не требуется: GPU, опыт ML, продвинутая линейная алгебра.

Как устроен урок: Build It · Use It · Ship It

Build It

Механизм с нуля, без фреймворков и без реального LLM — учебная мини-версия. Офлайн, без GPU и ключей. С проверочным тестом.

Use It

Тот же механизм через настоящую модель/API — мульти-провайдер. Вот здесь и нужен ключ (шаг помечен Requires).

Ship It

Артефакт урока в outputs/: prompt · skill · agent · MCP.

Каждый урок проходит 6 шагов: MOTTO → PROBLEM → CONCEPT → BUILD IT → USE IT → SHIP IT и заканчивается блоком «Материалы» с проверенными ссылками.

Оценивание (без преподавателя)

  • Код-артефакты — самопроверка юнит-тестами (pytest).
  • Теория — квиз /check-understanding после ключевых фаз (1, 4, 8).
  • Каждая фаза закрывается рефлексией: сверка с результатом фазы.
  • Финал — защита Capstone: метрики «до/после» (время, качество, стоимость) и ROI.
