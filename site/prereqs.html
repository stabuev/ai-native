Требования и формат

Курс технический и self-paced (без видео): материалы текстовые и кодовые, акцент на самостоятельную сборку и автопроверку.

Входные требования

  • Python на уровне написания скриптов (функции, файлы, пакеты, venv).
  • Базовая работа в терминале и Linux (команды, переменные окружения).
  • git — нужен базовый локальный цикл init / status / add / commit. Репозиторий курса клонировать не нужно: в Фазе 0 ты с нуля создашь отдельную личную папку.
  • Школьная математика; векторы и вероятности — плюс, но не обязательно.
  • Доступ хотя бы к одному поддерживаемому агенту понадобится в Фазе 0. API-ключ опционален: без него обязательная локальная настройка всё равно выполняется, а живой запрос пропускается.
  • Не требуется: GPU, опыт ML, продвинутая линейная алгебра.

Конструктор урока: Build It · Use It · Ship It

Build It

Механизм с нуля, если сборка делает его наблюдаемым или входит в outcome.

Use It

Реальный инструмент/API, если перенос даёт новое действие, а не повторяет пример.

Ship It

Отдельный артефакт, если его будут переиспользовать за пределами урока.

Это возможные элементы, а не обязательные секции. Структура, практика, квиз, материалы и артефакт выбираются по результату конкретного урока.

Оценивание (без преподавателя)

  • Код-артефакты — самопроверка юнит-тестами (pytest).
  • Теория — квиз /check-understanding после ключевых фаз (1, 4, 8).
  • Каждая фаза закрывается рефлексией: сверка с результатом фазы.
  • Финал — защита Capstone: метрики «до/после» (время, качество, стоимость) и ROI.
