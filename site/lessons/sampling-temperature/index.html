Урок 1.3 · Сэмплирование, температура и границы воспроизводимости

Фаза 1 — Как работают LLM · Результат фазы: Объяснить токены, контекст и inference; собрать токен-бюджетер; проверять, достаточно ли у ответа доказательств для использования.

Requires (только для необязательного USE IT): API-ключ одного провайдера и модель, которая принимает хотя бы один параметр сэмплирования. Основной урок, BUILD IT и артефакт можно пройти полностью офлайн.

Результат урока. После урока ты сможешь по логитам предсказать, как temperature, top_k и top_p изменят распределение и набор кандидатов; реализовать и проверить выбор одного токена; провести контролируемое сравнение настроек и объяснить, почему воспроизводимость локального алгоритма не гарантирует одинаковые ответы удалённого API.

В уроке 1.2 ты собрал авторегрессивный цикл: модель получала видимый контекст, возвращала распределение вероятностей, а функция greedy_next_token всегда брала самый вероятный токен. Теперь мы меняем только одно звено этого цикла — правило выбора следующего токена. Контекст, повторение шагов и причины остановки остаются прежними.

Откуда берутся разные продолжения

Представь, что на очередном шаге модель оценила четыре возможных токена так:

ТокенВероятность
данные0.64
результаты0.24
цифры0.09
банан0.03

Сумма вероятностей равна 1.00: это полный набор вариантов для одного учебного шага.

Сначала сделай два прогноза:

  1. Какой токен будет выбран при greedy?
  2. Может ли при другом правиле быть выбран токен результаты, хотя он не лидер?

При greedy ответ на первый вопрос всегда один: данные, потому что 0.64 — максимум. Но распределение содержит больше информации, чем только победителя. Можно выбирать токен случайно так, чтобы данные выпадали чаще, результаты — реже, а банан — совсем редко.

Правило, которое превращает оценки модели в выбранный следующий токен, называют стратегией декодирования. Здесь «декодирование» не означает обратную токенизацию: речь именно о выборе продолжения на каждом шаге генерации.

Greedy и взвешенное сэмплирование

Есть два базовых варианта выбора.

  • Greedy — взять позицию с самым большим значением. Операцию «найти позицию максимума» часто называют argmax. Для таблицы выше argmax — позиция токена данные.
  • Сэмплирование — разыграть случайный выбор с указанными весами. Токен с вероятностью 0.64 не обязан выпасть всегда, но на длинной серии одинаковых шагов будет выпадать заметно чаще токена с вероятностью 0.03.

Сэмплирование можно представить как отрезок от 0 до 1:

0.00 ├──────────── данные ────────────┤ 0.64
0.64 ├──── результаты ────┤ 0.88
0.88 ├── цифры ──┤ 0.97
0.97 ├ банан ┤ 1.00

Генератор случайных чисел выдаёт число r в диапазоне от 0 включительно до 1 не включительно. Если r = 0.80, оно попадает в интервал результаты; если r = 0.02 — в интервал данные.

В Python random.Random(...) — псевдослучайный генератор: его последовательность полностью определяется начальным числом, которое называют seed. Одинаковые seed и одинаковая последовательность вызовов дают одинаковую последовательность бросков.

rng = random.Random(7)       # создать один генератор
first = rng.random()         # первый бросок
second = rng.random()        # второй бросок того же генератора

Важно не перепутать это с постоянным перезапуском генератора:

random.Random(7).random()    # каждый раз снова первый бросок
random.Random(7).random()    # снова первый бросок, а не второй

Seed делает локальную псевдослучайную последовательность повторяемой. Он не превращает один вариант в «правильный» и сам по себе ничего не обещает о поведении внешнего сервиса.

От оценок модели к вероятностям

В 1.2 распределения были уже готовы. Реальная модель перед выбором токена обычно выдаёт для всего словаря числовые оценки — логиты. Логит может быть положительным, нулевым или отрицательным; сам по себе он ещё не является вероятностью.

Например, для четырёх позиций модель могла выдать:

logits = [3, 2, 1, 0]

Чтобы получить положительные числа с суммой 1, применяют функцию softmax:

            exp(logit_i / T)
p_i =  ─────────────────────────
        Σ_j exp(logit_j / T)

Здесь:

  • i — позиция одного кандидата;
  • Σ означает «сложить значения для всех кандидатов»;
  • exp(x) — экспонента: она превращает любое число в положительный вес и усиливает разницу между большими и маленькими оценками;
  • деление каждого веса на общую сумму называется нормализацией — после неё вероятности складываются в 1;
  • T — температура.

В коде перед exp из всех логитов вычитают максимальный логит. Это не меняет итоговые доли: все веса уменьшаются в одинаковое относительное число раз. Зато вычисления не переполняются на больших значениях.

Что именно меняет temperature

Temperature меняет разницу между логитами перед softmax.

  • T = 1 — используем исходные различия.
  • 0 < T < 1 — делим на маленькое число; различия увеличиваются, распределение становится острее, лидер получает больше массы.
  • T > 1 — делим на большое число; различия уменьшаются, распределение становится ровнее, у остальных кандидатов растут шансы.

Для логитов [3, 2, 1, 0] получаются такие распределения:

ТемператураВероятности после softmaxЧто видно
0.5[0.865, 0.117, 0.016, 0.002]лидер почти забрал всю массу
1.0[0.644, 0.237, 0.087, 0.032]исходный баланс
2.0[0.455, 0.276, 0.167, 0.102]распределение стало ровнее

До следующего абзаца попробуй ответить: при какой строке четвёртый токен имеет наибольший шанс выпасть? Ответ — при T = 2.0: его шанс вырос с 0.2% при T = 0.5 до 10.2%.

Особый случай temperature = 0

Подставить T = 0 в формулу нельзя: деление на ноль не определено. Поэтому в нашем учебном сэмплере temperature = 0 — это отдельное соглашение:

temperature = 0 → не вызывать softmax(T) и случайный генератор → вернуть argmax

Так локальная функция становится детерминированной: одинаковые логиты приводят к одной и той же позиции. Это свойство нашего явно заданного алгоритма, а не универсальная гарантия любого API с параметром, который тоже называется temperature.

Как top-k и top-p обрезают хвост

Temperature перераспределяет вероятность, но не удаляет кандидатов. Top-k и top-p решают другую задачу: исключают часть маловероятного хвоста перед броском.

После удаления кандидатов оставшиеся вероятности снова делят на их сумму. Это называется перенормировкой: итоговый набор снова должен иметь сумму 1.

Top-k: фиксированное число кандидатов

top_k = 2 означает «оставить две самые вероятные позиции»:

[0.644, 0.237, 0.087, 0.032]
          top-k = 2
                ↓
[0.731, 0.269, 0.000, 0.000]

Первые два значения поделили на их сумму 0.881. Теперь третий и четвёртый токены не могут выпасть вообще.

Top-p: достаточная суммарная масса

top_p = 0.8 означает «сортировать кандидатов от самого вероятного и оставить минимальный набор, который набрал не меньше 0.8 общей массы».

Для уверенного распределения:

[0.64, 0.24, 0.09, 0.03]
  0.64        < 0.8  → нужен ещё один токен
  0.64 + 0.24 = 0.88 ≥ 0.8 → остановиться

Остаются два кандидата. Но для более ровного распределения [0.35, 0.30, 0.20, 0.15] первые два дают только 0.65, поэтому при том же top_p = 0.8 понадобятся уже три кандидата: 0.35 + 0.30 + 0.20 = 0.85.

В этом различие:

  • top-k всегда задаёт фиксированное количество;
  • top-p задаёт фиксированную долю вероятностной массы, поэтому количество меняется от шага к шагу.

Полный учебный конвейер одного шага

В BUILD IT мы явно зафиксируем такой порядок:

непустые logits
      │
      ├─ temperature = 0 ───────────────────────────► argmax
      │
      └─ temperature > 0
             │
             ▼
       softmax(logits / T)
             │
             ▼
       top-k и перенормировка
             │
             ▼
       top-p и перенормировка
             │
             ▼
       один бросок rng
             │
             ▼
       индекс выбранного токена

Этот порядок — точный контракт нашего кода. Провайдер может использовать другую реализацию, скрытые ограничения или вообще не предоставлять эти параметры. Поэтому учебный сэмплер объясняет причинный механизм, но не притворяется копией закрытого сервиса.

Детерминизм и воспроизводимость — не одно обещание

Детерминированный алгоритм при одинаковом входе всегда возвращает одинаковый результат. Воспроизводимый эксперимент удаётся повторить при зафиксированных условиях. Эти понятия связаны, но условия воспроизводимости шире одного параметра.

СитуацияЧто можно утверждать
Наш sample(..., temperature=0)Возвращает argmax и не использует RNG
Наш sample(...) с одним seedПовторяет последовательность, если совпали код, входы и порядок вызовов RNG
Запрос к внешнему APIПовторяемость нужно измерять; параметр 0 или seed не является гарантией сервиса

У внешнего API могут измениться точный model ID, версия модели, серверная реализация, значения параметров по умолчанию и другие скрытые условия. Поэтому для честного сравнения записывают не только prompt, но и модель, дату, доступные параметры, число повторов и наблюдаемый результат.

Чего эти параметры не делают

  • Низкая температура не делает модель умнее.
  • temperature = 0 не превращает правдоподобное утверждение в проверенный факт.
  • Seed не исправляет ошибочный prompt и не валидирует ответ.
  • Большее разнообразие не означает автоматически лучшее качество.
  • Одинаковый текст в двух запусках ещё не доказывает, что API гарантирует детерминизм.

Главная практическая граница урока: параметры выбора меняют какой вероятный вариант будет продолжен, но не проверяют истинен ли этот вариант. Проверка доверия к содержанию начнётся в уроке 1.4.

BUILD IT

Задание: собери офлайн-сэмплер одного токена на стандартных модулях math и random.

Работай в личной папке course-work/1.3/. В блоке «Исходники урока» скопируй туда только code/test_sampling.py; опубликованный code/sampling.py пока не копируй. Первый запуск должен завершиться ошибкой импорта — это ожидаемая красная точка.

Создай course-work/1.3/sampling.py и реализуй функции.

1. softmax(logits, temperature=1.0)

  • принимает непустую последовательность логитов и положительную температуру;
  • перед exp вычитает максимальный логит;
  • возвращает список неотрицательных вероятностей с суммой 1;
  • при пустых логитах или temperature <= 0 выбрасывает ValueError.

2. top_k_filter(probs, k)

  • принимает непустую последовательность неотрицательных весов;
  • при k is None или k >= len(probs) только нормализует копию;
  • иначе оставляет ровно k самых больших позиций, обнуляет остальные и перенормирует;
  • при k < 1, нецелом k или нулевой сумме весов выбрасывает ValueError.

3. top_p_filter(probs, p)

  • при p is None или p == 1 только нормализует копию;
  • при 0 < p < 1 оставляет минимальное вероятностное ядро и перенормирует;
  • при значении вне интервала (0, 1], отрицательных весах или нулевой сумме выбрасывает ValueError.

4. sample(logits, temperature=1.0, top_k=None, top_p=None, rng=None)

  • при отрицательной температуре выбрасывает ValueError;
  • при temperature == 0 сразу возвращает argmax и не вызывает RNG;
  • при положительной температуре выполняет softmax → top-k → top-p → бросок;
  • использует переданный объект rng с методом .random(), а без него — модуль random;
  • возвращает индекс выбранной позиции.

Перед реализацией письменно предскажи:

  • какие позиции останутся после top_k=2 для логитов [3, 2, 1, 0];
  • сколько позиций оставит top_p=0.8 для распределений [0.64, 0.24, 0.09, 0.03] и [0.35, 0.30, 0.20, 0.15];
  • будет ли один генератор Random(7) выдавать один и тот же бросок на каждом вызове.

Готово, когда тесты зелёные, а ты можешь объяснить, почему они проверяют не только запуск:

  • softmax нормализует значения и реагирует на температуру;
  • top-k сохраняет точные позиции, а top-p меняет размер ядра вместе с формой распределения;
  • temperature=0 обходит случайный выбор;
  • отфильтрованный токен никогда не выбирается;
  • два отдельных генератора с одним seed воспроизводят одинаковые последовательности;
  • ошибочные границы параметров завершаются явно.

Запусти цикл red → green из корня личного проекта:

python -m pytest course-work/1.3/test_sampling.py -q
python course-work/1.3/sampling.py

Подсказка. Сначала реализуй softmax, затем общую перенормировку, после неё два фильтра и только потом случайный выбор. В sample накапливай вероятности слева направо и возвращай первую позицию, где накопленная сумма стала не меньше броска.

Если делегируешь код ИИ, сохрани четыре решения 4D за собой:

  • Delegation: отдай реализацию функций, но не первоначальные прогнозы и не решение о приёмке;
  • Description: передай сигнатуры, поведение на границах и тесты как исполнимый контракт;
  • Discernment: сравни распределения и последовательность с прогнозом, проверь, что seed не создаётся заново внутри каждого шага;
  • Diligence: сам запусти тесты, прочитай обработку ошибок и объясни, почему temperature=0 находится вне формулы softmax.

USE IT: необязательный эксперимент с API

Выполняй этот блок, только если у тебя уже есть настроенный в 0.2 API и точная модель действительно принимает хотя бы один параметр сэмплирования. Не выбирай устаревшую модель только ради прохождения урока: локальный эксперимент уже закрывает обязательный результат.

  1. Открой официальную документацию точного model ID и запиши дату проверки.
  2. Зафиксируй, какие параметры модель принимает и какие значения использует по умолчанию. Не переноси возможности другой модели того же провайдера.
  3. Выбери prompt с несколькими допустимыми ответами, например «предложи один короткий заголовок для заметки о командном ретро». Фактический вопрос здесь неудобен: он смешает разнообразие с истинностью.
  4. Выполни не меньше пяти повторов с базовыми настройками.
  5. Измени только один поддерживаемый параметр и выполни ещё не меньше пяти повторов. Prompt, модель и остальные настройки оставь прежними.
  6. Сравни наблюдаемое разнообразие. Не объявляй конфигурацию воспроизводимой только потому, что пять ответов случайно совпали.

Если модель не предоставляет sampling-параметры, это не ошибка студента и не пробел урока. Запиши «параметр недоступен для выбранного model ID» и используй офлайн-ветку.

Реалистичные ошибки и диагностика

  • «temperature=0 можно подставить в softmax». Нет: в формуле получится деление на ноль. В нашем коде это отдельная ветка argmax.
  • «Одинаковый seed означает одинаковый токен на каждом шаге». Один генератор выдаёт последовательность разных бросков; одинаковый seed позволяет повторить всю последовательность при одинаковом порядке вызовов.
  • «Top-p оставляет одинаковое число токенов». Это свойство top-k. Размер top-p-ядра зависит от формы текущего распределения.
  • «Низкая температура делает факты надёжными». Она сужает случайность выбора, но не добавляет источники и не проверяет утверждения.
  • «Параметр есть в SDK, значит его принимает любая модель». Поддержка определяется точным model ID и текущим контрактом API.
  • «Я изменил temperature и top-p и увидел эффект каждого». При двух изменениях причина неразличима. Для учебного эксперимента меняй одну переменную.

  • OpenAI — Models — открой карточку точного model ID и проверь поддерживаемые параметры: возможности соседней модели нельзя переносить автоматически.
  • Gemini — Text generation — найди generation_config, затем сверь доступность параметра для выбранной модели и используемого API.

2. Чтобы закрепить механизм — выбери один материал

  • Hugging Face — How to generate text — практический разбор greedy, top-k, top-p и temperature с кодом; сопоставь порядок операций со своим sampling.py.
  • Maxime Labonne — Decoding Strategies in LLMs — более подробное продолжение с реализациями стратегий декодирования; открывай, если одного учебного шага уже недостаточно.

3. Для глубокого погружения

  • Holtzman et al., 2019 — Nucleus Sampling — первоисточник top-p; начни с abstract и иллюстрации вырождения текста при максимизации вероятности.
  • Thinking Machines — Defeating Nondeterminism in LLM Inference — техническое объяснение, почему даже temperature=0 не всегда обеспечивает одинаковый удалённый результат; для завершения урока детали вычислительных ядер не нужны.

import random

import pytest

from sampling import sample, softmax, top_k_filter, top_p_filter


LOGITS = [3.0, 2.0, 1.0, 0.0]


class FixedRng:
    def __init__(self, draw):
        self.draw = draw
        self.calls = 0

    def random(self):
        self.calls += 1
        return self.draw


class ForbiddenRng:
    def random(self):
        raise AssertionError("temperature=0 must not call the random generator")


def test_softmax_matches_worked_example_and_sums_to_one():
    probs = softmax(LOGITS, 1.0)

    assert probs == pytest.approx(
        [0.643914, 0.236883, 0.087144, 0.032059],
        abs=1e-6,
    )
    assert sum(probs) == pytest.approx(1.0)


def test_temperature_changes_distribution_not_ranking():
    cold = softmax(LOGITS, 0.5)
    hot = softmax(LOGITS, 2.0)

    assert cold[0] > hot[0]
    assert cold[-1] < hot[-1]
    assert max(range(len(cold)), key=cold.__getitem__) == 0
    assert max(range(len(hot)), key=hot.__getitem__) == 0


def test_softmax_is_stable_for_large_logits():
    probs = softmax([1001.0, 1000.0])

    assert probs == pytest.approx([0.731059, 0.268941], abs=1e-6)
    assert sum(probs) == pytest.approx(1.0)


def test_top_k_keeps_exact_positions_and_renormalizes():
    filtered = top_k_filter([0.64, 0.24, 0.09, 0.03], 2)

    assert filtered == pytest.approx([0.64 / 0.88, 0.24 / 0.88, 0.0, 0.0])
    assert sum(filtered) == pytest.approx(1.0)


def test_top_p_adapts_nucleus_size_to_distribution_shape():
    confident = top_p_filter([0.64, 0.24, 0.09, 0.03], 0.8)
    flat = top_p_filter([0.35, 0.30, 0.20, 0.15], 0.8)

    assert confident == pytest.approx([0.64 / 0.88, 0.24 / 0.88, 0.0, 0.0])
    assert flat == pytest.approx([0.35 / 0.85, 0.30 / 0.85, 0.20 / 0.85, 0.0])
    assert sum(value > 0 for value in confident) == 2
    assert sum(value > 0 for value in flat) == 3


def test_temperature_zero_returns_argmax_without_random_draw():
    assert sample(LOGITS, temperature=0, rng=ForbiddenRng()) == 0


def test_top_k_one_forces_argmax_even_with_sampling():
    rng = FixedRng(0.999)

    assert sample(LOGITS, temperature=1.0, top_k=1, rng=rng) == 0
    assert rng.calls == 1


def test_fixed_draw_selects_by_cumulative_probability():
    rng = FixedRng(0.8)

    assert sample(LOGITS, temperature=1.0, top_k=2, rng=rng) == 1
    assert rng.calls == 1


def test_filtered_positions_are_never_sampled():
    chosen = {
        sample(LOGITS, temperature=2.0, top_k=2, rng=random.Random(seed))
        for seed in range(50)
    }

    assert chosen <= {0, 1}
    assert chosen == {0, 1}


def test_same_seed_reproduces_a_sequence_when_generator_is_reused():
    first_rng = random.Random(7)
    second_rng = random.Random(7)

    first = [sample([0.0, 0.0, 0.0, 0.0], rng=first_rng) for _ in range(12)]
    second = [sample([0.0, 0.0, 0.0, 0.0], rng=second_rng) for _ in range(12)]

    assert first == second
    assert len(set(first)) > 1

"""Учебный сэмплер одного токена для урока 1.3.

Код делает явным офлайн-конвейер:
logits -> softmax(temperature) -> top-k -> top-p -> weighted sample.

Это контракт учебного алгоритма, а не копия внутренней реализации конкретного API.
"""

import math
import random


def _argmax(values):
    """Вернуть позицию первого максимума в непустой последовательности."""
    if not values:
        raise ValueError("values must not be empty")
    return max(range(len(values)), key=lambda index: values[index])


def softmax(logits, temperature=1.0):
    """Преобразовать непустые logits в вероятности при temperature > 0."""
    if not logits:
        raise ValueError("logits must not be empty")
    if temperature <= 0:
        raise ValueError("softmax temperature must be greater than zero")

    maximum = max(logits)
    weights = [math.exp((logit - maximum) / temperature) for logit in logits]
    total = sum(weights)
    return [weight / total for weight in weights]


def _renormalize(probs):
    """Проверить веса и вернуть их копию с суммой 1."""
    if not probs:
        raise ValueError("probabilities must not be empty")
    if any(probability < 0 for probability in probs):
        raise ValueError("probabilities must be non-negative")

    total = sum(probs)
    if total <= 0:
        raise ValueError("probabilities must have a positive sum")
    return [probability / total for probability in probs]


def top_k_filter(probs, k):
    """Оставить k наибольших позиций и вернуть нормализованную копию."""
    normalized = _renormalize(probs)
    if k is None:
        return normalized
    if isinstance(k, bool) or not isinstance(k, int) or k < 1:
        raise ValueError("k must be a positive integer or None")
    if k >= len(normalized):
        return normalized

    order = sorted(
        range(len(normalized)),
        key=lambda index: normalized[index],
        reverse=True,
    )
    keep = set(order[:k])
    filtered = [
        probability if index in keep else 0.0
        for index, probability in enumerate(normalized)
    ]
    return _renormalize(filtered)


def top_p_filter(probs, p):
    """Оставить минимальное ядро с исходной суммарной массой не меньше p."""
    normalized = _renormalize(probs)
    if p is None:
        return normalized
    if isinstance(p, bool) or not isinstance(p, (int, float)) or not 0 < p <= 1:
        raise ValueError("p must be in the interval (0, 1] or None")
    if p == 1:
        return normalized

    order = sorted(
        range(len(normalized)),
        key=lambda index: normalized[index],
        reverse=True,
    )
    keep = set()
    cumulative = 0.0
    for index in order:
        keep.add(index)
        cumulative += normalized[index]
        if cumulative >= p:
            break

    filtered = [
        probability if index in keep else 0.0
        for index, probability in enumerate(normalized)
    ]
    return _renormalize(filtered)


def sample(logits, temperature=1.0, top_k=None, top_p=None, rng=None):
    """Вернуть индекс, выбранный по явно заданному учебному конвейеру."""
    if not logits:
        raise ValueError("logits must not be empty")
    if temperature < 0:
        raise ValueError("temperature must not be negative")
    if temperature == 0:
        return _argmax(logits)

    probs = softmax(logits, temperature)
    probs = top_k_filter(probs, top_k)
    probs = top_p_filter(probs, top_p)

    generator = rng if rng is not None else random
    draw = generator.random()
    cumulative = 0.0
    for index, probability in enumerate(probs):
        cumulative += probability
        if draw <= cumulative:
            return index

    # Защита от накопленной ошибки float: возвращаем последнюю ненулевую позицию.
    return max(index for index, probability in enumerate(probs) if probability > 0)


if __name__ == "__main__":
    example_logits = [3.0, 2.0, 1.0, 0.0]

    print("temperature distributions")
    for value in (0.5, 1.0, 2.0):
        distribution = softmax(example_logits, value)
        print(f"T={value}: {[round(probability, 3) for probability in distribution]}")

    base = softmax(example_logits)
    print("top_k=2:", [round(value, 3) for value in top_k_filter(base, 2)])
    print("top_p=.8:", [round(value, 3) for value in top_p_filter(base, 0.8)])
    print("T=0 index:", sample(example_logits, temperature=0))

    first_rng = random.Random(7)
    second_rng = random.Random(7)
    first_sequence = [sample(example_logits, rng=first_rng) for _ in range(8)]
    second_sequence = [sample(example_logits, rng=second_rng) for _ in range(8)]
    print("seeded sequence A:", first_sequence)
    print("seeded sequence B:", second_sequence)
