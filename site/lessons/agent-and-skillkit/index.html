Урок 0.3 · Агент и SkillKit

Фаза 0 — Setup & Tooling · Результат фазы: Настроить окружение, ключи и агента; осознанно выбрать модель под задачу.

Этот урок ставит твоего рабочего агента и учит главному формату курса — скиллам. Модель в веб-чате забывает контекст и не знает твоих приёмов. Агент живёт в репозитории, видит файлы и запускает команды, а скиллы дают ему переиспользуемые навыки. К концу урока у тебя установлен агент, подключены два курсовых скилла и есть персональный маршрут по курсу — что проходить подробно, а что пролистать.

MOTTO. Агент + скиллы превращают модель из «чата» в рабочий инструмент с памятью навыков.

PROBLEM

Голая модель в чате забывает контекст и каждый раз требует объяснять ей одни и те же приёмы заново. Агент (Claude Code / Cursor / Codex) уже лучше — он видит твои файлы и запускает команды, — но без накопленных навыков ты всё равно повторяешься. И отдельная боль: курс на ~228 часов нельзя проходить вслепую, иначе утонешь в том, что и так знаешь, и пролетишь важное.

CONCEPT

Интуиция

Агент — это модель, которой дали руки: она читает твой код, правит файлы и запускает команды прямо в проекте. Официально Claude Code описан так: «an agentic coding tool that reads your codebase, edits files, runs commands, and integrates with your development tools».

Скилл — переиспользуемый навык агента: папка с файлом SKILL.md, где написано, что делать и когда это применять. В отличие от обычного промпта (разовая инструкция в одном разговоре), скилл подключается сам, когда задача под него подходит, и не нужно объяснять приём заново. По определению Anthropic: «Skills are reusable, filesystem-based resources… Skills load on-demand and eliminate the need to repeatedly provide the same guidance across multiple conversations».

Как это работает

У скилла есть SKILL.md с YAML-frontmatter (name + description), а агент решает, подключать ли скилл, по его description — поэтому описание говорит и что скилл делает, и когда его применять.

агент (Claude Code / Cursor / Codex)
   └── .claude/skills/<name>/SKILL.md       ← проектные скиллы (или ~/.claude/skills — личные)
          ---
          name: find-your-level
          description: что делает + КОГДА применять  ← по этому агент и решает
          ---
          <инструкция: шаги, примеры>

Чтобы много скиллов не забивали контекст, работает progressive disclosure — подгрузка по уровням (из доков Agent Skills):

  • Уровень 1 — метаданные (name/description): загружены всегда, ~100 токенов на скилл. Агент лишь знает, что скилл есть и когда он нужен.
  • Уровень 2 — тело SKILL.md: подгружается, только когда задача совпала с описанием.
  • Уровень 3 — ресурсы и скрипты: читаются по мере надобности, не занимая контекст заранее.

Итог: можно держать десятки скиллов почти без платы за контекст — подробности входят в окно только когда реально нужны.

РАЗБОР ПО ШАГАМ

Посмотрим, как из входной диагностики получается персональный маршрут. Она разделяет самооценку и наблюдаемое доказательство.

Сначала студент выбирает для каждой фазы не ярлык «знаю/не знаю», а описание действия: например, «могу повторить настройку по инструкции» или «самостоятельно диагностирую ошибки настройки». Это даёт профиль по фазам, а не один суммарный уровень. Сильный опыт в Python больше не скрывает пробел в механике языковых моделей.

Затем идут 13 коротких рабочих ситуаций по контрольным фазам 1, 4 и 8 — по одной на каждый урок. Например: длинный диалог перестал учитывать раннее правило; что произошло и как пересобрать контекст? Такой вопрос различает понимание механизма и правдоподобное «повысить температуру», а не проверяет память на определение.

Маршрут использует три действия:

  • focus — пройти фазу полностью;
  • refresh — быстро восстановить модель и обязательно сделать практику;
  • challenge — начать с выходной проверки и артефакта фазы.

Автоматического skip нет. Самооценка может дать только refresh; challenge появляется, когда решены все входные сценарии фазы. Если итоговая проверка или артефакт не готовы, соответствующий урок возвращается в маршрут. Capstone всегда остаётся focus.

BUILD IT

Под капотом курсовых онбординг-скиллов — два офлайн-движка без зависимостей:

  • code/find_your_level.py — 12 поведенческих самооценок,

входная форма A по фазам 1, 4 и 8 и route(), который строит маршрут без пропуска фаз.

  • code/check_understanding.py — выходная форма B: новые

сценарии, два на каждый урок фаз 1, 4 и 8; run_quiz() показывает результат по урокам, диагностирует ошибку и предлагает открытую повторную попытку.

python code/find_your_level.py
python code/check_understanding.py
pytest code -q

Обе формы используют одинаковые идентификаторы компетенций, но разные ситуации. Поэтому после фазы можно сравнить вход и выход по конкретному умению — например, управлению контекстом, — а не по несопоставимым общим баллам.

USE IT

  1. Поставь агента. Claude Code ставится одной командой (официальный способ):
curl -fsSL https://claude.ai/install.sh | bash    # macOS/Linux/WSL; или: brew install --cask claude-code
cd ai-native && claude                            # запуск в своей папке курса

(Cursor и Codex — альтернативы; формат скиллов у них общий.)

  1. Подключи курсовые скиллы. Репозиторий ты не клонируешь, поэтому собери скилл из файлов на этой странице. В своей ai-native/ создай папку .claude/skills/find-your-level/ и положи в неё два файла — SKILL.md и find_your_level.py — взяв их внизу страницы, в «Исходниках урока», кнопкой «Копировать». То же для check-understanding. Агент подхватит их сам по description. (Личные скиллы, общие для всех проектов, кладут в ~/.claude/skills/.) Не хочешь возиться с файлами — движок уже у тебя из Build It: запусти python code/find_your_level.py и увидишь пример маршрута.
  1. Запусти онбординг прямо в агенте:
/find-your-level          → профиль по фазам + входные сценарии → маршрут с причинами
/check-understanding 1    → после Фазы 1 → новые сценарии, прогресс и точечная пересдача
Врезка (2026): категории кодовых агентов. Claude Code — терминал и глубокий контекст; Cursor — IDE и инлайн-редактирование; Codex — облачный фоновый воркер; OpenClaw / Hermes — локальные персональные платформы (подробно в уроке 4.5). У каждого свой файл-инструкция проекта: CLAUDE.md (Claude Code), AGENTS.md (Codex), .cursorrules (Cursor) — но формат скиллов SKILL.md у них общий.

Безопасность. Ставь скиллы только из доверенных источников: скилл несёт инструкции и код, и вредоносный может направить агента не туда. Доки прямо предупреждают: «Use Skills only from trusted sources».

ЧАСТЫЕ ОШИБКИ

  • Путать промпт и скилл. Промпт — разовая инструкция в одном разговоре; скилл — переиспользуемый навык, который подключается сам по описанию. Повторяешь одно и то же из чата в чат — заведи скилл.
  • Слабый description. Агент решает подключить скилл именно по description. Если там не сказано, когда применять, скилл просто не сработает.
  • Скилл из недоверенного источника. Скилл = инструкции + код; чужой непроверенный скилл — как запуск чужого софта. Бери только своё или от Anthropic.
  • Считать самооценку доказательством. «Я это делал» помогает выбрать глубину, но не разрешает пропустить результат урока. Для ускорения нужен сценарий, практика или артефакт.
  • Сводить профиль к одному уровню. Опыт почти всегда неравномерен: сильная разработка не доказывает понимание доверия к ответам, а опыт с чатами — готовность строить агента.

PASS_THRESHOLD = 0.8
LESSON_THRESHOLD = 0.5


def _validate_answers(answers, bank) -> list[int]:
    answers = list(answers)
    if len(answers) != len(bank):
        raise ValueError(f"нужно ответов: {len(bank)}, получено: {len(answers)}")
    for choice, item in zip(answers, bank):
        if not isinstance(choice, int) or not 0 <= choice < len(item["options"]):
            raise ValueError("каждый ответ должен быть целым индексом существующего варианта")
    return answers


def grade(answers, bank) -> dict:
    """Оценивает фазу по общему баллу и минимальному покрытию каждого урока."""
    if not bank:
        raise ValueError("банк вопросов не может быть пустым")
    answers = _validate_answers(answers, bank)

    lesson_stats: dict[str, dict[str, int]] = {}
    competency_stats: dict[str, dict[str, int]] = {}
    feedback = []
    for choice, item in zip(answers, bank):
        is_correct = choice == item["correct"]
        lesson = lesson_stats.setdefault(item["lesson"], {"correct": 0, "total": 0})
        lesson["total"] += 1
        lesson["correct"] += int(is_correct)
        competency = competency_stats.setdefault(item["competency"], {"correct": 0, "total": 0})
        competency["total"] += 1
        competency["correct"] += int(is_correct)
        if not is_correct:
            feedback.append({
                "question_id": item["id"],
                "lesson": item["lesson"],
                "competency": item["competency"],
                "selected": item["options"][choice],
                "correct_answer": item["options"][item["correct"]],
                "diagnosis": item["misconception"],
                "explanation": item["explanation"],
            })

    correct = sum(stats["correct"] for stats in lesson_stats.values())
    total = len(bank)
    score = correct / total
    by_lesson = {
        lesson: {
            **stats,
            "score": round(stats["correct"] / stats["total"], 2),
        }
        for lesson, stats in lesson_stats.items()
    }
    competency_scores = {
        competency: round(stats["correct"] / stats["total"], 2)
        for competency, stats in competency_stats.items()
    }
    coverage_ok = all(stats["score"] >= LESSON_THRESHOLD for stats in by_lesson.values())
    passed = score >= PASS_THRESHOLD and coverage_ok
    review_lessons = [lesson for lesson, stats in by_lesson.items() if stats["score"] < 1.0]
    retry_competencies = [name for name, value in competency_scores.items() if value < 1.0]

    return {
        "correct": correct,
        "total": total,
        "score": round(score, 2),
        "coverage_ok": coverage_ok,
        "verdict": "освоено" if passed else "нужна практика",
        "by_lesson": by_lesson,
        "competency_scores": competency_scores,
        "review_lessons": review_lessons,
        "feedback": feedback,
        "retry_tasks": [
            {"competency": name, **RETRY_TASKS[name]}
            for name in retry_competencies
        ],
        "assessment_note": "Квиз подтверждает понимание сценариев, но не заменяет артефакт и практику фазы.",
    }


def compare_with_baseline(result: dict, baseline: dict[str, float]) -> list[dict]:

    progress = []
    for competency, exit_score in result["competency_scores"].items():
        if competency not in baseline:
            continue
        baseline_score = baseline[competency]
        if not isinstance(baseline_score, (int, float)) or not 0 <= baseline_score <= 1:
            raise ValueError("входной балл компетенции должен быть числом от 0 до 1")
        progress.append({
            "competency": competency,
            "baseline": round(float(baseline_score), 2),
            "exit": exit_score,
            "delta": round(exit_score - float(baseline_score), 2),
        })
    return progress


def run_quiz(phase: int, answers, baseline: dict[str, float] | None = None) -> dict:

    result = grade(answers, BANKS[phase])
    result["phase"] = phase
    if baseline is not None:
        result["progress"] = compare_with_baseline(result, baseline)
    return result

def _validate_choices(answers, size: int, option_count: int = 4) -> list[int]:
    answers = list(answers)
    if len(answers) != size:
        raise ValueError(f"нужно ответов: {size}, получено: {len(answers)}")
    if any(not isinstance(choice, int) or not 0 <= choice < option_count for choice in answers):
        raise ValueError(f"каждый ответ должен быть целым индексом от 0 до {option_count - 1}")
    return answers


def self_profile(answers) -> dict[int, int]:
    """Возвращает самооценку 0..3 отдельно по каждой фазе 0–11."""
    answers = _validate_choices(answers, len(SELF_ASSESSMENT))
    return {
        item["phase"]: item["options"][choice][1]
        for item, choice in zip(SELF_ASSESSMENT, answers)
    }


def grade_baseline(phase: int, answers) -> dict:
    """Оценивает входную сценарную форму A для одной контрольной фазы."""
    if phase not in BASELINE_BANKS:
        raise KeyError(f"нет входного банка для фазы {phase}")
    bank = BASELINE_BANKS[phase]
    answers = _validate_choices(answers, len(bank))
    correct = sum(choice == item["correct"] for choice, item in zip(answers, bank))
    competency_scores = {
        item["competency"]: float(choice == item["correct"])
        for choice, item in zip(answers, bank)
    }
    gaps = [
        {
            "lesson": item["lesson"],
            "competency": item["competency"],
            "diagnosis": item["misconception"],
        }
        for choice, item in zip(answers, bank)
        if choice != item["correct"]
    ]
    score = correct / len(bank)
    return {
        "phase": phase,
        "correct": correct,
        "total": len(bank),
        "score": round(score, 2),
        "strong_evidence": correct == len(bank),
        "competency_scores": competency_scores,
        "gaps": gaps,
    }


def _profile_label(ratings: dict[int, int]) -> str:
    experienced = sum(value >= 2 for value in ratings.values())
    if experienced <= 3:
        return "Нужна опора в большинстве тем"
    if experienced <= 8:
        return "Опыт неравномерный — ускоряйся точечно"
    return "Опыт широкий — подтверждай его практикой"


def route(self_answers, baseline_answers: dict[int, list[int]] | None = None) -> dict:
    """Строит маршрут по фазам без автоматического ``skip``.

    ``baseline_answers`` — ответы формы A для фаз 1, 4 и 8. Только сценарное
    доказательство может перевести контрольную фазу в ``challenge``. Для остальных
    фаз уверенная самооценка даёт лишь ``refresh``.
    """
    ratings = self_profile(self_answers)
    baseline_answers = baseline_answers or {}
    baseline_results = {
        phase: grade_baseline(phase, answers)
        for phase, answers in baseline_answers.items()
    }

    plan = []
    total_hours = 0.0
    for phase, name, base_hours in PHASES:
        if phase == 12:
            action = "focus"
            reason = "Capstone выполняют все: он проверяет перенос на реальный процесс."
        elif phase in BASELINE_BANKS:
            evidence = baseline_results.get(phase)
            if evidence is None:
                action = "focus"
                reason = "Нет сценарного доказательства по этой фазе."
            elif evidence["strong_evidence"]:
                action = "challenge"
                reason = "Все входные сценарии решены; начни с итоговой проверки и артефакта фазы."
            elif evidence["score"] >= 0.5:
                action = "refresh"
                reason = "Часть механики уже освоена, но есть диагностированные пробелы."
            else:
                action = "focus"
                reason = "Сценарии показали пробелы в большинстве результатов фазы."
        else:
            rating = ratings[phase]
            if rating >= 2:
                action = "refresh"
                reason = "Есть заявленный практический опыт, но он ещё не подтверждён проверкой."
            else:
                action = "focus"
                reason = "Практического опыта пока недостаточно для сокращённого маршрута."

        hours = round(base_hours * _FACTOR[action], 1)
        total_hours += hours
        plan.append({
            "phase": phase,
            "name": name,
            "action": action,
            "hours": hours,
            "reason": reason,
        })

    calibration = []
    for phase, evidence in baseline_results.items():
        rating = ratings[phase]
        if rating >= 2 and evidence["score"] < 0.5:
            calibration.append({
                "phase": phase,
                "note": "Самооценка выше результата сценариев — проходи фазу полностью.",
            })
        elif rating <= 1 and evidence["strong_evidence"]:
            calibration.append({
                "phase": phase,
                "note": "Сценарии сильнее самооценки — можно начать с итоговой проверки.",
            })

    return {
        "profile": _profile_label(ratings),
        "profile_note": "Это описание маршрута, а не квалификация или допуск к работе.",
        "total_hours": round(total_hours, 1),
        "self_ratings": ratings,
        "baseline": baseline_results,
        "baseline_competencies": {
            competency: value
            for result in baseline_results.values()
            for competency, value in result["competency_scores"].items()
        },
        "calibration": calibration,
        "plan": plan,
    }

# find-your-level — входная диагностика и маршрут

Помоги студенту выбрать глубину прохождения курса. Это не квалификационный экзамен:
не присваивай единый ярлык «новичок/инженер» и не объявляй тему освоенной только по
самооценке.

## Что измеряем

Диагностика состоит из двух разных источников:

1. **Самооценка по наблюдаемым действиям** — 12 вопросов, по одному на фазы 0–11.
2. **Сценарная форма A** — 13 коротких задач по результатам каждого урока контрольных
   фаз 1, 4 и 8. Она использует ту же карту компетенций, что последующая
   `/check-understanding`, но другие ситуации.

Capstone (фаза 12) проходят все. Самооценка помогает сократить повторение, но только
сценарное доказательство позволяет перейти сразу к контрольной попытке фазы.

## Как проводить

### Шаг 1. Самооценка

Возьми `SELF_ASSESSMENT` из `find_your_level.py`. Задавай вопросы небольшими порциями,
показывай варианты `0..3` полностью. Не заменяй поведенческие формулировки названиями
уровней и объясняй незнакомое слово прямо в вопросе.

Сохрани 12 выбранных индексов в исходном порядке.

### Шаг 2. Входные сценарии

Возьми `BASELINE_BANKS[1]`, затем `[4]` и `[8]`. Задай все 13 сценариев, не показывая
поле `correct`. После ответа дай короткое объяснение; по ошибке используй
`misconception`, чтобы назвать конкретную неверную модель, а не личное качество
студента.

Сохрани ответы отдельно по фазам:

```python
baseline_answers = {
    1: [3, 0, 1, 2],
    4: [0, 2, 1, 3, 3],
    8: [1, 0, 3, 2],
}
```

### Шаг 3. Маршрут

Передай ответы в детерминированный офлайн-движок:

```bash

self_answers=[1,1,2,1,2,1,1,0,1,0,0,1]; \
```

Покажи:

- профиль и пояснение, что это маршрут, а не квалификация;
- план по 13 фазам и причину каждой рекомендации;
- расхождения самооценки и сценариев из `calibration`;
- пробелы формы A из `baseline[*].gaps`;
- ориентировочные часы как диапазон планирования, а не обещание.

## Значение действий

- `focus` — пройти фазу полностью.
- `refresh` — быстро восстановить модель, затем выполнить практику и проверку; примерно
  половина базового времени.
- `challenge` — начать с выходной проверки и артефакта фазы; примерно четверть времени.
  Ошибка возвращает соответствующий урок в маршрут.

Действия `skip` нет. Для фаз без сценарной формы высокая самооценка даёт не больше чем
`refresh`. Capstone всегда `focus`.

# check-understanding — выходная проверка фазы

Проверь, может ли студент применить механизм фазы в новой ситуации. Не представляй
квиз как доказательство профессиональной готовности: он дополняет, но не заменяет
практику и артефакт фазы.

## Аргумент

Номер контрольной фазы: `1`, `4` или `8`. В `BANKS[phase]` лежит форма B:

- фаза 1 — 8 сценариев, по два на каждый урок;
- фаза 4 — 10 сценариев, по два на каждый урок;
- фаза 8 — 8 сценариев, по два на каждый урок.

Правильные позиции сбалансированы, а неверные варианты представляют правдоподобные
ошибки. Не показывай `correct`, `explanation` и `misconception` до ответа.

## Как проводить

1. Задавай вопросы из `BANKS[phase]` по одному или небольшими порциями. Для каждого
   показывай четыре варианта с индексами `0..3`.
2. Собери ответы в исходном порядке и запусти движок:

```bash
python -c "import check_understanding as c, json; \
p=1; answers=[2,0,3,1,1,3,0,2]; \
print(json.dumps(c.run_quiz(p, answers), ensure_ascii=False, indent=2))"
```

3. Покажи общий результат и `by_lesson`. Для каждой ошибки перескажи `diagnosis` и
   `explanation`: студент должен увидеть, какую модель он перепутал.
4. Для каждой затронутой компетенции задай открытое задание из `retry_tasks`. Прими
   повторную попытку только если ответ удовлетворяет всем её `criteria`; не подсказывай
   готовую формулировку заранее.
5. После повторения уроков разреши пройти форму B снова с переставленным порядком
   вопросов и вариантов.

## Правило результата

`освоено` означает одновременно:

- не менее 80% правильных ответов в целом;
- не менее половины заданий каждого урока решено верно.

Даже при вердикте `освоено` ошибки остаются в `review_lessons` и получают открытую
повторную попытку. Итоговый артефакт фазы проверяется отдельно по своей рубрике.

## Сравнение со входом

Если в персональном маршруте сохранено поле `baseline_competencies`, передай его как
`baseline`:

```python
result = run_quiz(phase, answers, baseline=baseline_competencies)
```

Поле `progress` покажет входной балл, выходной балл и изменение по каждой общей
компетенции. Не суммируй это в один «уровень»: студент может вырасти в одной области и
сохранить пробел в другой.
