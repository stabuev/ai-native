Урок 0.3 · Агент и SkillKit

Фаза 0 — Setup & Tooling · Результат фазы: Настроить окружение, ключи и агента; осознанно выбрать модель под задачу.

Этот урок ставит твоего рабочего агента и учит главному формату курса — скиллам. Модель в веб-чате забывает контекст и не знает твоих приёмов. Агент живёт в репозитории, видит файлы и запускает команды, а скиллы дают ему переиспользуемые навыки. К концу урока у тебя установлен агент, подключены два курсовых скилла и есть персональный маршрут по курсу — что проходить подробно, а что пролистать.

MOTTO. Агент + скиллы превращают модель из «чата» в рабочий инструмент с памятью навыков.

PROBLEM

Голая модель в чате забывает контекст и каждый раз требует объяснять ей одни и те же приёмы заново. Агент (Claude Code / Cursor / Codex) уже лучше — он видит твои файлы и запускает команды, — но без накопленных навыков ты всё равно повторяешься. И отдельная боль: курс на ~228 часов нельзя проходить вслепую, иначе утонешь в том, что и так знаешь, и пролетишь важное.

CONCEPT

Интуиция

Агент — это модель, которой дали руки: она читает твой код, правит файлы и запускает команды прямо в проекте. Официально Claude Code описан так: «an agentic coding tool that reads your codebase, edits files, runs commands, and integrates with your development tools».

Скилл — переиспользуемый навык агента: папка с файлом SKILL.md, где написано, что делать и когда это применять. В отличие от обычного промпта (разовая инструкция в одном разговоре), скилл подключается сам, когда задача под него подходит, и не нужно объяснять приём заново. По определению Anthropic: «Skills are reusable, filesystem-based resources… Skills load on-demand and eliminate the need to repeatedly provide the same guidance across multiple conversations».

Как это работает

У скилла есть SKILL.md с YAML-frontmatter (name + description), а агент решает, подключать ли скилл, по его description — поэтому описание говорит и что скилл делает, и когда его применять.

агент (Claude Code / Cursor / Codex)
   └── .claude/skills/<name>/SKILL.md       ← проектные скиллы (или ~/.claude/skills — личные)
          ---
          name: find-your-level
          description: что делает + КОГДА применять  ← по этому агент и решает
          ---
          <инструкция: шаги, примеры>

Чтобы много скиллов не забивали контекст, работает progressive disclosure — подгрузка по уровням (из доков Agent Skills):

  • Уровень 1 — метаданные (name/description): загружены всегда, ~100 токенов на скилл. Агент лишь знает, что скилл есть и когда он нужен.
  • Уровень 2 — тело SKILL.md: подгружается, только когда задача совпала с описанием.
  • Уровень 3 — ресурсы и скрипты: читаются по мере надобности, не занимая контекст заранее.

Итог: можно держать десятки скиллов почти без платы за контекст — подробности входят в окно только когда реально нужны.

РАЗБОР ПО ШАГАМ

Посмотрим, как из ответов на входной квиз получается персональный маршрут. Возьмём два крайних профиля и пройдём логику руками.

Новичок отвечает на все 10 вопросов по нулям. score суммирует веса → 0 баллов. level: меньше 10 → «Новичок». route: новичку каждая фаза идёт focus (множитель 1.0), поэтому часы = сумма базовых = 228 ч — полный курс.

Инженер отвечает по максимуму → score = 30 → level (больше 20) = «Инженер». Теперь действия по фазам разные: setup (фаза 0) — skip (0 ч), базовые фазы 1–2 — skim (половина часов), остальное — focus, а Capstone всегда focus. Итог: 228 − 8 (skip setup) − 6 (фаза 1: 12→6) − 8 (фаза 2: 16→8) = 206 ч.

Один и тот же движок по баллу выдаёт и уровень, и план с часами. Ровно это упаковано в скилл /find-your-level.

BUILD IT

Под капотом курсовых онбординг-скиллов — два офлайн-движка без зависимостей (читай, запусти, поправь под себя):

  • code/find_your_level.py — входной квиз: score(answers) суммирует веса, level(total) даёт уровень (Новичок/Практик/Инженер), route(total) строит план по 13 фазам с действием (focus/skim/skip) и часами.
  • code/check_understanding.py — квиз по фазе: run_quiz(phase, answers) считает результат, выдаёт вердикт и список уроков на повтор (банки готовы для контрольных фаз 1, 4, 8).
python code/find_your_level.py     # три профиля: новичок ~228 ч, практик ~218 ч, инженер ~206 ч
python code/check_understanding.py
pytest code -q

Это и есть «персональный маршрут» в коде; ниже мы запустим то же самое как скилл прямо в агенте.

USE IT

  1. Поставь агента. Claude Code ставится одной командой (официальный способ):
curl -fsSL https://claude.ai/install.sh | bash    # macOS/Linux/WSL; или: brew install --cask claude-code
cd ai-native && claude                            # запуск в своей папке курса

(Cursor и Codex — альтернативы; формат скиллов у них общий.)

  1. Подключи курсовые скиллы. Репозиторий ты не клонируешь, поэтому собери скилл из файлов на этой странице. В своей ai-native/ создай папку .claude/skills/find-your-level/ и положи в неё два файла — SKILL.md и find_your_level.py — взяв их внизу страницы, в «Исходниках урока», кнопкой «Копировать». То же для check-understanding. Агент подхватит их сам по description. (Личные скиллы, общие для всех проектов, кладут в ~/.claude/skills/.) Не хочешь возиться с файлами — движок уже у тебя из Build It: запусти python code/find_your_level.py и увидишь пример маршрута.
  1. Запусти онбординг прямо в агенте:
/find-your-level          → ответь на 10 вопросов → получишь маршрут с часами
/check-understanding 1     → после Фазы 1 → узнаешь, что повторить
Врезка (2026): категории кодовых агентов. Claude Code — терминал и глубокий контекст; Cursor — IDE и инлайн-редактирование; Codex — облачный фоновый воркер; OpenClaw / Hermes — локальные персональные платформы (подробно в уроке 4.5). У каждого свой файл-инструкция проекта: CLAUDE.md (Claude Code), AGENTS.md (Codex), .cursorrules (Cursor) — но формат скиллов SKILL.md у них общий.

Безопасность. Ставь скиллы только из доверенных источников: скилл несёт инструкции и код, и вредоносный может направить агента не туда. Доки прямо предупреждают: «Use Skills only from trusted sources».

ЧАСТЫЕ ОШИБКИ

  • Путать промпт и скилл. Промпт — разовая инструкция в одном разговоре; скилл — переиспользуемый навык, который подключается сам по описанию. Повторяешь одно и то же из чата в чат — заведи скилл.
  • Слабый description. Агент решает подключить скилл именно по description. Если там не сказано, когда применять, скилл просто не сработает.
  • Скилл из недоверенного источника. Скилл = инструкции + код; чужой непроверенный скилл — как запуск чужого софта. Бери только своё или от Anthropic.
  • Проходить курс вслепую. Без /find-your-level легко утонуть в знакомом и пролететь важное. Сначала маршрут, потом учёба.

PASS_THRESHOLD = 0.8


def grade(answers, bank) -> dict:
    """answers: список выбранных индексов. Возвращает результат и список повтора."""
    assert len(answers) == len(bank), "нужен ответ на каждый вопрос"
    correct = sum(a == q["correct"] for a, q in zip(answers, bank))
    review = sorted({q["lesson"] for a, q in zip(answers, bank) if a != q["correct"]})
    s = correct / len(bank)
    return {
        "correct": correct, "total": len(bank), "score": round(s, 2),
        "verdict": "сдано" if s >= PASS_THRESHOLD else "повторить",
        "review_lessons": review,
    }


def run_quiz(phase: int, answers) -> dict:
    if phase not in BANKS:
        raise KeyError(f"нет банка вопросов для фазы {phase}")
    return grade(answers, BANKS[phase])


if __name__ == "__main__":
    bank = BANKS[1]

code/find_your_level.py
"""Движок входного квиза /find-your-level — Build It для урока 0.3.

Без зависимостей и офлайн. 10 вопросов с весами (0..3 за вопрос), по сумме
баллов определяет уровень студента и строит персональный маршрут по курсу с
оценкой часов. Тот же движок упакован в скилл (см. outputs/ и .claude/skills/).
"""
from __future__ import annotations

# Каждый вопрос: текст + варианты (label, score 0..3).
QUESTIONS = [
    {"q": "Python",            "options": [("не писал", 0), ("скрипты по примеру", 1), ("свои функции/модули", 2), ("пакеты, тесты, типы", 3)]},
    {"q": "Терминал/Linux",    "options": [("боюсь", 0), ("базовые команды", 1), ("env, pipes, процессы", 2), ("свободно", 3)]},
    {"q": "git",               "options": [("нет", 0), ("commit/push", 1), ("ветки/merge", 2), ("rebase/CI", 3)]},
    {"q": "Вызов LLM API",     "options": [("никогда", 0), ("через UI", 1), ("пару раз кодом", 2), ("в проде", 3)]},
    {"q": "Промптинг",         "options": [("интуитивно", 0), ("знаю few-shot", 1), ("CoT, форматы", 2), ("делаю eval", 3)]},
    {"q": "RAG/эмбеддинги",    "options": [("что это", 0), ("слышал", 1), ("пробовал", 2), ("строил", 3)]},
    {"q": "Агенты/tool use",   "options": [("нет", 0), ("читал", 1), ("запускал", 2), ("писал свои", 3)]},
    {"q": "MCP",               "options": [("впервые слышу", 0), ("знаю идею", 1), ("подключал", 2), ("писал сервер", 3)]},
    {"q": "SQL/аналитика",     "options": [("нет", 0), ("простые SELECT", 1), ("джоины/окна", 2), ("свободно", 3)]},
    {"q": "Стоимость/FinOps",  "options": [("не думал", 0), ("примерно", 1), ("считаю токены", 2), ("оптимизирую роутингом", 3)]},
]

# (id, название, базовые часы) — сумма = 228 (см. README / ROADMAP, дорожная карта).
PHASES = [
    (0, "Setup & Tooling", 8), (1, "Как работают LLM", 12), (2, "Промпт-инжиниринг", 16),
    (3, "Текст и документы", 18), (4, "Скиллы, память, проекты", 20), (5, "Данные и аналитика", 18),
    (6, "Инструменты и MCP", 18), (7, "Agent Engineering", 26), (8, "Мульти-агенты", 18),
    (9, "FinOps", 14), (10, "Production", 16), (11, "Этика и governance", 14), (12, "Capstone", 30),
]

_FACTOR = {"focus": 1.0, "skim": 0.5, "skip": 0.0}


def score(answers) -> int:
    """Сумма баллов по выбранным индексам (0..3 за вопрос). Диапазон 0..30."""
    assert len(answers) == len(QUESTIONS), "нужен ответ на каждый вопрос"
    return sum(QUESTIONS[i]["options"][choice][1] for i, choice in enumerate(answers))


def level(total: int) -> str:
    if total < 10:
        return "Новичок"
    if total <= 20:
        return "Практик"
    return "Инженер"


def _action(lvl: str, pid: int) -> str:
    """Что делать с фазой при данном уровне. Capstone всегда focus."""
    if pid == 12:
        return "focus"
    if lvl == "Новичок":
        return "focus"
    if lvl == "Практик":
        return "skim" if pid in (0, 1) else "focus"   # уже знает setup и базу LLM
    # Инженер
    if pid == 0:
        return "skip"
    return "skim" if pid in (1, 2) else "focus"


def route(total: int) -> dict:
    """Персональный маршрут: действие и часы по каждой фазе + итог по часам."""
    lvl = level(total)
    plan, hours = [], 0.0
    for pid, name, base in PHASES:
        action = _action(lvl, pid)
        h = base * _FACTOR[action]
        hours += h
        plan.append({"phase": pid, "name": name, "action": action, "hours": h})
    return {"level": lvl, "score": total, "total_hours": hours, "plan": plan}


if __name__ == "__main__":
    for answers, who in [([0] * 10, "новичок"), ([2] * 10, "практик"), ([3] * 10, "инженер")]:
        r = route(score(answers))
        print(f"{who}: {r['level']}, балл {r['score']}, ~{r['total_hours']:.0f} ч")
code/test_onboarding.py
from find_your_level import QUESTIONS, PHASES, score, level, route
from check_understanding import run_quiz, BANKS


def test_score_range_and_levels():
    assert score([0] * len(QUESTIONS)) == 0
    assert score([3] * len(QUESTIONS)) == 30
    assert level(9) == "Новичок"
    assert level(10) == "Практик"
    assert level(20) == "Практик"
    assert level(21) == "Инженер"


def test_route_beginner_is_full_228h():
    r = route(score([0] * len(QUESTIONS)))
    assert r["level"] == "Новичок"
    assert r["total_hours"] == 228        # сумма базовых часов = README / ROADMAP
    assert all(p["action"] == "focus" for p in r["plan"])


def test_route_engineer_is_shorter_but_keeps_capstone():
    r = route(score([3] * len(QUESTIONS)))
    assert r["level"] == "Инженер"
    assert r["total_hours"] < 228
    plan = {p["phase"]: p["action"] for p in r["plan"]}
    assert plan[0] == "skip"        # инженер пропускает setup
    assert plan[12] == "focus"      # capstone делают все
    assert len(r["plan"]) == len(PHASES) == 13


def test_check_understanding_all_correct():
    bank = BANKS[1]
    res = run_quiz(1, [q["correct"] for q in bank])
    assert res["verdict"] == "сдано"
    assert res["review_lessons"] == []


def test_check_understanding_flags_review_lessons():
    bank = BANKS[1]
    answers = [(q["correct"] + 1) % len(q["options"]) for q in bank]  # все мимо
    res = run_quiz(1, answers)
    assert res["verdict"] == "повторить"
    assert "1.1" in res["review_lessons"]


def test_check_understanding_has_key_phase_banks():
    # квизы готовы для контрольных точек курса: фазы 1, 4, 8
    for phase in (1, 4, 8):
        assert phase in BANKS and len(BANKS[phase]) >= 5
        bank = BANKS[phase]
        res = run_quiz(phase, [q["correct"] for q in bank])
        assert res["verdict"] == "сдано" and res["review_lessons"] == []

# find-your-level — входной квиз и персональный маршрут

Помогает студенту понять, что в курсе пройти подробно, что пролистать, а что пропустить, и сколько это займёт часов.

## Как проводить

1. Задай студенту **10 вопросов** из `find_your_level.py` (`QUESTIONS`). Для каждого покажи варианты с номерами `0..3` и попроси выбрать.
2. Собери ответы как список индексов, например `[1, 2, 0, ...]` (по одному на вопрос).
3. Посчитай результат движком (детерминированный, офлайн):

```bash
python -c "import find_your_level as f; import json; print(json.dumps(f.route(f.score([1,2,0,1,1,0,0,0,1,1])), ensure_ascii=False, indent=2))"
```

4. Покажи студенту: **уровень** (Новичок/Практик/Инженер), **итог по часам** и **план по фазам** (focus / skim / skip).
5. Предложи сохранить результат как `outputs/personal-route.md` в его проекте.

## Логика (из движка)

- Баллы 0..30 → уровень: `<10` Новичок, `10..20` Практик, `>20` Инженер.
- Маршрут: Новичок — всё focus (~228 ч); Практик — листает Фазы 0–1 (~218 ч); Инженер — пропускает Фазу 0, листает 1–2 (~206 ч). Capstone (Фаза 12) делают все.

## Файлы

- `find_your_level.py` — движок (`QUESTIONS`, `score`, `level`, `route`). Кросс-платформенно: Claude Code / Cursor / Codex.

# check-understanding — квиз по фазе

Контроль знаний без преподавателя: после фазы студент отвечает на вопросы, получает вердикт (сдано/повторить) и точечный список уроков на повтор.

## Аргумент

Номер фазы, напр. `1`. Банки вопросов лежат в `check_understanding.py` (`BANKS`). Готовы банки контрольных точек курса — фаз **1, 4, 8**; другие фазы добавляются дописыванием `BANKS[<phase>]`.

## Как проводить

1. Возьми банк фазы: `BANKS[phase]`. Задай вопросы, для каждого покажи варианты с номерами и попроси выбрать индекс.
2. Собери ответы как список индексов и посчитай результат:

```bash
python -c "import check_understanding as c; import json; print(json.dumps(c.run_quiz(1,[2,1,1,1,1]), ensure_ascii=False))"
```

3. Покажи: `score`, `verdict` (порог сдачи 80%), и `review_lessons` — какие уроки повторить по неверным ответам.

## Файлы

- `check_understanding.py` — движок (`BANKS`, `grade`, `run_quiz`). Чтобы добавить фазу — допиши `BANKS[<phase>]` списком вопросов `{q, options, correct, lesson}`.
