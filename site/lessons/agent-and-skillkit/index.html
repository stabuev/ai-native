Урок 0.3 · Рабочий агент и персональный маршрут

Фаза 0 — Setup & Tooling · Результат фазы: Настроить окружение, ключи и агента; осознанно выбрать модель под задачу.

Результат урока: запустить одного выбранного агента в личной папке ai-native-work, проверить его доступ к файлам, связать его роль с решением из 0.1, подключить два курсовых скилла оценивания и сохранить персональный маршрут с объяснением следующего шага.

Requires: интернет и аккаунт с доступом хотя бы к одному агенту — Claude Code, Cursor или Codex. API-ключ из урока 0.2 не всегда заменяет подписку или вход в выбранный продукт. Для запасного офлайн-пути достаточно Python 3 и файла диагностики.

Перед установкой открой заполненную task-to-model-matrix.md из урока 0.1. Она задаёт не обязательный бренд агента, а его честную роль:

  • если одной из задач нужны файлы, репозиторий или запуск команд, агент может быть рабочим кандидатом для этой задачи;
  • если в карте для всех задач выбран другой способ работы, агент остаётся средой для прохождения курса — это не повод переписывать прошлое решение.

Из урока 0.2 мы продолжаем переносить рабочую папку и правила обращения с секретами. API-провайдер и продукт-агент могут относиться к разным компаниям и используют разную авторизацию — совпадение между ними не требуется.

MOTTO. Сначала докажи, что агент работает в нужной папке; только потом доверяй ему работу.

Зачем агенту рабочая папка

В обычном чате модель отвечает на сообщения и вложения. Агент дополнительно получает разрешённые ему инструменты: может читать и изменять файлы, запускать команды и проверять результат. Для курса это важно — дальше ты будешь не только обсуждать решения, но и собирать артефакты в проекте.

Агент не «живёт внутри репозитория»: он работает с папкой, которую ты открыл. Если запустить его уровнем выше или в соседнем проекте, он увидит не те файлы или не найдёт их вовсе. Поэтому первая проверка урока — не красивый ответ, а наблюдаемое действие в правильной рабочей папке.

Скилл — сохранённая инструкция для повторяемой задачи. Она лежит в папке с файлом SKILL.md; агент может применить её по явному вызову или когда запрос подходит под описание. В этом уроке ты не пишешь и не разбираешь устройство скилла — это самостоятельный результат урока 4.1. Сейчас скилл нужен как готовый инструмент входной диагностики.

1. Выбери один агент

Не устанавливай все три ради сравнения. Сначала выбери в карте 0.1 одну сквозную задачу — ту, которой действительно полезны файлы, репозиторий или команды. Если такой карточки нет, запиши «агент настроен только как среда курса». Затем выбери один доступный и разрешённый твоей организацией агент и пройди весь урок в нём. Отдельное сравнение брендов здесь не нужно: в 0.1 ты уже освоил сам процесс проверки кандидатов.

Claude Codeкоманда claude.claude/skills//find-your-level

У SKILL.md общий переносимый формат, но каталог установки и способ вызова зависят от агента. Копирование файлов только в .claude/skills/ не является установкой для Codex.

Установи выбранный продукт по его текущей официальной инструкции:

  • Claude Code — установка;
  • Cursor — загрузка приложения;
  • Codex CLI — установка и первый запуск.

На macOS, Linux или WSL официальные установщики Claude Code и Codex сейчас выглядят так:

# Выполни только одну из двух команд.
curl -fsSL https://claude.ai/install.sh | bash
curl -fsSL https://chatgpt.com/codex/install.sh | sh

Для Windows и других способов установки используй вкладку своей ОС в официальной инструкции. После установки войди в аккаунт тем способом, который предлагает продукт.

2. Открой рабочую папку и проверь доступ

Продолжай в той же личной папке ai-native-work, где в 0.1 сохранил карту, а в 0.2 создал .venv, .env.example и check_env.py. Не клонируй курс и не создавай ещё один проект только ради агента. Открой ai-native-work через меню агента или сначала открой терминал именно в этой папке. pwd на macOS/Linux либо Get-Location в PowerShell должны оканчиваться на ai-native-work.

Запусти выбранного агента именно здесь или открой эту папку в Cursor. Затем дай ему задание:

Назови абсолютный путь текущей рабочей папки.
Создай в ней файл agent-check.md с тремя строками:
agent: <название агента>
workspace: <абсолютный путь>
status: ready
Прочитай файл обратно и не изменяй другие файлы.

Проверь результат сам:

pwd
cat agent-check.md

В Windows PowerShell вместо pwd и cat можно использовать Get-Location и Get-Content agent-check.md.

Проверка пройдена, если путь в терминале, путь в ответе агента и workspace внутри файла указывают на одну папку. Если пути различаются, остановись и заново открой нужную папку — дополнительные разрешения эту ошибку не исправляют.

Теперь проверь, что агент видит не только тестовый файл, но и решение из 0.1. Попроси прочитать task-to-model-matrix.md из корня открытой папки:

Прочитай task-to-model-matrix.md из текущей рабочей папки и ничего не изменяй.
Для выбранной мной сквозной задачи верни четыре поля:
задача, критерий успеха, важные ограничения, выбранный способ работы.
Отдельно скажи, нужен ли этой задаче доступ к файлам, репозиторию или командам.

Сверь ответ с исходной карточкой. Если агент исказил критерий или выбрал другую задачу, исправь перенос до дальнейшей работы. Если карта подтверждает другой способ работы, сохрани это как ограничение: агент остаётся средой курса, а не новым рабочим выбором.

3. Подключи курсовое оценивание

Сначала установи входную диагностику find-your-level: инструкцию SKILL.md и офлайн-движок find_your_level.py. Команды ниже скачивают только два файла скилла прямо в личную папку — клонировать курс не нужно. Выбери каталог своего агента в первой строке:

COURSE_SKILLS_DIR=".claude/skills"   # Claude Code
# COURSE_SKILLS_DIR=".cursor/skills" # Cursor
# COURSE_SKILLS_DIR=".agents/skills" # Codex

mkdir -p "$COURSE_SKILLS_DIR/find-your-level"
curl -fL https://raw.githubusercontent.com/stabuev/ai-native/main/.claude/skills/find-your-level/SKILL.md \
  -o "$COURSE_SKILLS_DIR/find-your-level/SKILL.md"
curl -fL https://raw.githubusercontent.com/stabuev/ai-native/main/.claude/skills/find-your-level/find_your_level.py \
  -o "$COURSE_SKILLS_DIR/find-your-level/find_your_level.py"

Рядом установи check-understanding, который понадобится после фаз 1, 4 и 8. Сейчас его запускать не нужно:

mkdir -p "$COURSE_SKILLS_DIR/check-understanding"
curl -fL https://raw.githubusercontent.com/stabuev/ai-native/main/.claude/skills/check-understanding/SKILL.md \
  -o "$COURSE_SKILLS_DIR/check-understanding/SKILL.md"
curl -fL https://raw.githubusercontent.com/stabuev/ai-native/main/.claude/skills/check-understanding/check_understanding.py \
  -o "$COURSE_SKILLS_DIR/check-understanding/check_understanding.py"

В Windows можно создать те же каталоги через проводник и сохранить четыре файла по ссылкам из команд. Не запускай скачанный код вслепую: открой файлы и убедись, что скиллы не просят секреты, а Python-скрипты не отправляют данные в сеть и пишут только в явно указанный тобой файл.

Если агент был открыт до создания каталога и не увидел скилл, перезапусти сессию в той же рабочей папке.

4. Получи маршрут

В Claude Code и Cursor вызови /find-your-level. В Codex напиши:

$find-your-level Проведи входную диагностику и сохрани маршрут в personal-route.md.

Перед диагностикой передай скиллу выбранную сквозную задачу, её критерий успеха и одну из двух ролей агента: «рабочий кандидат» или «только среда курса». Эти поля должны остаться в personal-route.md; они связывают маршрут с решением из 0.1, но не влияют на оценку опыта по фазам.

Когда последующие уроки предлагают /check-understanding 1, пользователь Codex может написать $check-understanding Проверь понимание фазы 1.

Сначала диагностика задаст 12 вопросов о действиях, которые ты уже выполнял. Это самооценка для планирования, а не экзамен. Сценарные вопросы по фазам 1, 4 и 8 появятся только там, где ты заявил практический опыт или сам попросил проверить возможность начать с контрольной попытки.

Маршрут использует три действия:

  • focus — пройти фазу полностью;
  • refresh — быстро восстановить основную модель, но выполнить практику и проверку;
  • challenge — начать с выходной проверки и артефакта фазы.

Challenge не означает «тема пройдена». Если проверка или артефакт обнаружат пробел, соответствующий урок возвращается в маршрут. Действия skip нет, а Capstone остаётся обязательным для всех.

Оценка времени показывается диапазоном. Это ориентир для календаря, а не обещание: скорость зависит от исходного опыта, выбранного проекта и глубины самостоятельной практики.

Если агент недоступен, запусти тот же движок без него:

python "$COURSE_SKILLS_DIR/find-your-level/find_your_level.py" \
  --interactive --output personal-route.md \
  --work-task "<задача из карты 0.1>" \
  --success-criterion "<критерий из той же карточки>" \
  --agent-role "<рабочий кандидат / только среда курса>"

Этот запасной путь строит тот же консервативный маршрут, но не проверяет установку агента. Вернись к шагу с agent-check.md, когда доступ появится.

Что должно остаться после урока

Используй шаблон персонального маршрута или файл, который сохранил скилл. В нём должны быть:

  • выбранный агент и подтверждённая рабочая папка;
  • сквозная задача и критерий успеха из карты 0.1;
  • честная роль агента: рабочий кандидат или только среда курса;
  • действие focus / refresh / challenge и причина для каждой фазы;
  • ориентировочный диапазон времени, а не точность до половины часа;
  • отмеченные сценарные проверки, если они действительно проводились;
  • ближайший следующий шаг — обычно урок 0.4 «Как проходить практику курса».

Урок завершён, когда ты можешь показать agent-check.md и personal-route.md, объяснить роль агента относительно карты 0.1, различие между тремя действиями маршрута и повторно запустить агента в той же папке.

## Как читать действия

- `focus` — пройти фазу полностью;
- `refresh` — быстро восстановить модель, затем выполнить практику и проверку;
- `challenge` — начать с выходной проверки и артефакта, вернувшись к урокам при пробеле.

Действия `skip` нет. Capstone всегда остаётся `focus`. Диапазоны времени — ориентир
для календаря, а не обещание скорости.

## Критерии готовности

- [ ] Агент повторно запускается в той же рабочей папке.
- [ ] `agent-check.md` содержит правильный путь.
- [ ] Сквозная задача и критерий совпадают с одной карточкой из карты 0.1.
- [ ] Роль агента не противоречит выбранному там способу работы.
- [ ] `find-your-level` и `check-understanding` лежат в каталоге выбранного агента.
- [ ] Для каждой фазы есть действие и понятная причина.
- [ ] `challenge` нигде не трактуется как автоматический пропуск.
- [ ] В маршруте нет точных часов, не подтверждённых реальным темпом обучения.
