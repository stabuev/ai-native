Урок 9.1 · Цены 2026 и за что платишь

Фаза 9 — Экономика и оркестрация моделей (FinOps) · Результат фазы: Спроектировать маршрутизацию моделей и посчитать бюджет «до/после».

Requires (только для USE IT): платный API-ключ одного из провайдеров. BUILD IT работает офлайн, без ключей и без сети.

Это вход в Фазу 9 — деньги. Мы умеем строить агентов и мульти-агентов; теперь научимся не разоряться на них. А начинается экономика с базового навыка: считать стоимость запроса до отправки.

MOTTO. Платишь за токены, а не за запросы; стоимость надо уметь считать до отправки.

PROBLEM

«Сколько это будет стоить в проде на 100k запросов в день?» — без калькулятора это гадание, а ошибка тут стоит реальных денег. Цена зависит от трёх вещей: какой модели, сколько входных и выходных токенов, и в каком режиме (обычный / кэш / batch).

Не считая заранее, легко словить счёт в 10× из-за одной неоптимальной модели или невключённого кэша. И наоборот: понимая формулу, видишь, где главный рычаг (спойлер — выбор модели), и режешь расходы осознанно. Соберём калькулятор, который считает доллары из токенов.

CONCEPT

Интуиция

Думай про API как про такси с поминутной тарификацией, а не фиксированной ценой за поездку. Платишь не «за запрос», а за пробег в токенах — и вход, и выход тарифицируются отдельно, причём выход обычно в разы дороже входа (у Claude — впятеро: вход $X, выход $5X). Разные модели — это разные классы такси: разброс цен между «эконом» (Flash) и «бизнес» (Opus) — на порядки. Поэтому первый и главный вопрос экономии — не «как короче ехать», а «на какой машине».

Два режима дают скидку: prompt caching (повторно используемый кусок входа читается из кэша за 10% цены) и batch API (асинхронная пачка — −50% на всё).

Как это работает

стоимость = (вход × цена_вход + выход × цена_выход) / 1M
   модификаторы (стэкаются):
   - prompt caching: чтение из кэша ≈ 0.1× цены входа (но запись кэша стоит 1.25×/2×)
   - batch API: −50% на вход и выход
  • PRICES — карта цен ($/1M токенов, вход/выход) по моделям.
  • cost(model, n_in, n_out, cached_in, batch) — стоимость вызова: свежий вход по полной цене, кэшированный — за 10%, выход по своей цене, всё × 0.5 в batch.
  • compare(models, n_in, n_out) — отсортировать модели по цене того же запроса.

Главный вывод формулы: множитель цена_вход/цена_выход различается между моделями на порядки, а число токенов — нет. Значит, выбор модели двигает счёт сильнее любой микрооптимизации промпта — это фундамент маршрутизации (9.2–9.3).

РАЗБОР ПО ШАГАМ

Рычаг №1 — модель. Один и тот же запрос 1000 вход / 500 выход, считаем (вход×цена_вход + выход×цена_выход) / 1e6:

Opus 4.8 ($5/$25):           (1000×5  + 500×25)  / 1e6 = 17500/1e6 = $0.01750
DeepSeek V4-Flash ($0.14/$0.28): (1000×0.14 + 500×0.28)/1e6 =   280/1e6 = $0.00028
                                                   отношение ≈ 62×

Тот же текст — в ~62 раза дороже на топ-модели. Вот почему выбор модели — рычаг №1.

Рычаг №2 — кэш. Sonnet 4.6 ($3/$15), запрос 10000 вход / 500 выход, из них 8000 входных — повторяемый контекст (системный промпт, документ):

без кэша: (10000×3 + 500×15) / 1e6 = 37500/1e6 = $0.03750
с кэшем:  свежие 2000×3 + кэш 8000×3×0.1 + выход 500×15, всё /1e6
        = (6000 + 2400 + 7500)/1e6 = 15900/1e6 = $0.01590   (−58%)

Кэш срезал больше половины, потому что огромный повторяемый вход поехал по 10% цены. Batch добавил бы ещё −50% поверх. Важная тонкость: запись в кэш стоит дороже обычного входа (1.25× для 5-минутного, 2× для часового), поэтому кэш окупается только при повторном использовании — после одного чтения для 5-минутного окна, после двух для часового. На один разовый запрос кэш не выгоден.

BUILD IT

Задание: собери калькулятор стоимости — токены → доллары, с кэшем и batch. Только стандартная библиотека, без сети.

Перед запуском. Работай в своей папке курса (ai-native/9.1-pricing/), а файлы урока клади в подпапку code/ (по соглашению курса). Нужен только Python 3 (для теста ещё pytest).

Создай файл cost_calculator.py:

  • PRICES — словарь {model: (цена_вход, цена_выход)} в $/1M токенов (актуальные цены — в USE IT ниже).
  • cost(model, n_in, n_out, cached_in=0, batch=False) — стоимость вызова в $. Неизвестная модель → KeyError. Свежий вход = n_in − cached_in по полной цене; кэшированный = cached_in × цена_вход × 0.1; выход по своей цене; сумма / 1e6; если batch — × 0.5.
  • compare(models, n_in, n_out) — список (model, стоимость) по возрастанию.

Готово, когда все тесты в test_cost_calculator.py зелёные — они проверяют: базовый расчёт (Sonnet 1M+1M = $18); Opus дороже Haiku; кэш снижает цену; batch ровно вдвое дешевле; неизвестная модель → KeyError; compare ставит дешёвую модель первой.

Подсказка. fresh_in = max(0, n_in - cached_in). Формула: (fresh_in×p_in + cached_in×p_in×0.1 + n_out×p_out) / 1e6, затем × 0.5 при batch. compare — sorted(..., key=lambda x: x[1]).

USE IT

Сверка с реальными тарифами (мульти-провайдер) — держи карту цен в одном месте и сверяйся с официальными страницами, цены меняются.

Цены на 2026-06 ($/1M, вход/выход):

  • Anthropic: Opus 4.8 — $5 / $25 · Sonnet 4.6 — $3 / $15 · Haiku 4.5 — $1 / $5. Prompt caching: чтение 0.1×, запись 1.25× (5 мин) / 2× (1 ч); Batch API — −50% на вход и выход; скидки стэкаются.
  • DeepSeek V4-Flash (open-weight, MIT) — $0.14 / $0.28 (cache-hit входа — $0.0028): самый дешёвый frontier-class, в десятки раз ниже закрытых топов.
  • OpenAI / Google — свои тарифы и кэш/batch; подставляй актуальные.

Реальный якорь масштаба (Anthropic): 10 000 тикетов поддержки по ~3 700 токенов на Haiku 4.5 ≈ $37. Та же нагрузка на Opus была бы кратно дороже — отсюда вся Фаза 9.

⚠️ Цены и модели быстро устаревают — это снимок на 2026-06-30. Перед расчётами сверься с актуальными страницами тарифов. Учти и нюанс токенайзера: Opus 4.7+ использует новый токенайзер и может тратить до 35% больше токенов на тот же текст — это тоже влияет на эффективную стоимость.

ЧАСТЫЕ ОШИБКИ

  • Считать «за запрос», а не за токены. Платишь за вход + выход раздельно, и выход дороже входа в разы. Длинный ответ дороже длинного вопроса при том же числе токенов.
  • Игнорировать выбор модели как рычаг. Разброс цен между моделями — на порядки (десятки раз). Это двигает счёт сильнее любой правки промпта; модель выбирают под задачу (9.2).
  • Думать, что кэш бесплатен. Запись в кэш стоит 1.25×/2× входа. Кэш окупается только при повторном использовании — на разовый запрос он в минус.
  • Забывать про batch на нетерминальных задачах. Асинхронная пачка — это −50% на всё. Для несрочного (отчёты, разметка) не использовать batch — переплата вдвое.
  • Брать устаревшую карту цен. Цены и модели меняются ежемесячно. Держи PRICES в одном месте и сверяй с официальными тарифами, а не с числами из памяти.

Тесты — это твоё ТЗ

code/test_cost_calculator.py
import pytest
from cost_calculator import cost, compare, PRICES


def test_basic_cost():
    # sonnet 1M вход + 1M выход = 3 + 15 = 18$
    assert cost("sonnet-4.6", 1_000_000, 1_000_000) == 18.0


def test_opus_more_expensive_than_haiku():
    assert cost("opus-4.8", 1000, 500) > cost("haiku-4.5", 1000, 500)


def test_cache_reduces_cost():
    full = cost("sonnet-4.6", 10000, 500)
    cached = cost("sonnet-4.6", 10000, 500, cached_in=8000)
    assert cached < full


def test_batch_halves_cost():
    assert cost("sonnet-4.6", 1000, 500, batch=True) == cost("sonnet-4.6", 1000, 500) / 2


def test_unknown_model_raises():
    with pytest.raises(KeyError):
        cost("gpt-9000", 100, 100)


def test_compare_sorted_cheapest_first():
    ranking = compare(PRICES, 1000, 500)
    assert ranking[0][0] == "deepseek-v4-flash"   # open-weight, самый дешёвый frontier-class
    assert ranking[-1][0] == "opus-4.8"

Эталонное решение

code/cost_calculator.pyПоказать решение
"""Цены 2026 и за что платишь: калькулятор стоимости — Build It для урока 9.1.

Без зависимостей. Считает стоимость запроса по токенам ввода/вывода с учётом
prompt caching (кэшированный вход дешевле) и batch API (−50%). Платишь за токены,
а не за запросы — поэтому стоимость надо уметь считать ДО отправки.
"""

# Цены 2026, $ за 1M токенов (вход, выход). См. README, «Карта моделей и цен».
PRICES = {
    "deepseek-v4-flash": (0.14, 0.28),   # open-weight (MIT), reasoning через thinking-mode
    "gemini-flash": (0.30, 2.50),
    "haiku-4.5": (1.00, 5.00),
    "gpt-5.x": (2.50, 15.00),
    "sonnet-4.6": (3.00, 15.00),
    "opus-4.8": (5.00, 25.00),
}

CACHE_DISCOUNT = 0.1     # кэшированный вход ≈ 10% от цены (−90%)
BATCH_DISCOUNT = 0.5     # batch API ≈ −50%


def cost(model, n_in, n_out, cached_in=0, batch=False):
    """Стоимость одного вызова в $.

    cached_in — сколько входных токенов взяты из кэша (дешевле).
    batch — пакетный режим (−50%).
    """
    if model not in PRICES:
        raise KeyError(f"нет цены для модели: {model}")
    p_in, p_out = PRICES[model]
    fresh_in = max(0, n_in - cached_in)
    total = (fresh_in * p_in + cached_in * p_in * CACHE_DISCOUNT + n_out * p_out) / 1e6
    return total * (BATCH_DISCOUNT if batch else 1.0)


def compare(models, n_in, n_out):
    """Стоимость одного запроса по моделям, по возрастанию."""
    return sorted(((m, round(cost(m, n_in, n_out), 6)) for m in models), key=lambda x: x[1])


if __name__ == "__main__":
    print("Запрос 1000/500 токенов:")
    for m, c in compare(PRICES, 1000, 500):
        print(f"  {m:<14} ${c:.5f}")
    base = cost("sonnet-4.6", 10000, 500)
    print(f"\nSonnet 10k/500: ${base:.5f}")
    print(f"  с кэшем 8k:   ${cost('sonnet-4.6', 10000, 500, cached_in=8000):.5f}")
    print(f"  batch:        ${cost('sonnet-4.6', 10000, 500, batch=True):.5f}")
