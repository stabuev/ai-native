Урок 5.1 · ИИ как аналитик

Фаза 5 — Данные и аналитика · Результат фазы: Анализировать данные с ИИ, делать NL→SQL и автоотчёты с валидацией результатов.

Requires (только для USE IT): платный API-ключ или аккаунт ChatGPT/Claude с загрузкой файлов. BUILD IT работает офлайн, без ключей и без сети.

Это вход в Фазу 5 — данные. Дальше будут Code Interpreter, NL→SQL, BI-агенты и автоотчёты. Но в основе всего лежит одна простая идея, которую мы разберём здесь до винтика: ИИ-аналитик не «понимает» таблицу магически — он строит план и исполняет его кодом.

MOTTO. Аналитика — это план: что сгруппировать, что посчитать, как агрегировать; ИИ строит план, код его исполняет.

PROBLEM

«Посмотри данные и скажи, что интересного» — звучит как магия. Кажется, будто модель смотрит на таблицу и «видит» инсайт. Это не так — и пока механика выглядит магией, ты не можешь ни проверить ответ, ни понять, почему он иногда неверный.

На деле ИИ-аналитик делает буквально следующее: превращает твой вопрос в план (по чему сгруппировать, какую колонку считать, каким агрегатом) и исполняет его кодом (pandas / Code Interpreter), а не «угадывает» цифры из головы. Соберём этот механизм на голой стандартной библиотеке — и магии не останется.

CONCEPT

Интуиция

Любой аналитический вопрос раскладывается на три кубика:

  • разрез (group_by) — по чему режем: по региону, по продукту, по дню;
  • метрика (value) — что именно меряем: сумму чего, среднее чего;
  • агрегат (agg) — как сворачиваем группу в одно число: sum, mean, count.

«Выручка по регионам» = разрез region + метрика amount + агрегат sum. «Средний чек по продукту» = разрез product + метрика amount + агрегат mean. Вопрос на человеческом языке — это и есть план из трёх кубиков; работа ИИ — собрать план, работа кода — его исполнить.

Как это работает

Механизм называется split-apply-combine — три шага:

вопрос → план(group_by, value, agg) → исполнение → результат
                                          │
   split    разложить строки по группам (по group_by)
   apply    в каждой группе применить агрегат к value (sum/mean/count)
   combine  собрать {группа: число} и при желании взять топ-N
  • load_csv(text) — прочитать таблицу в список словарей (строки).
  • aggregate(rows, group_by, value, agg) — сгруппировать строки и свернуть каждую группу в число.
  • top_n(d, n) — отсортировать результат по убыванию и взять верх.

Ровно это делает pandas (df.groupby(...).sum()), и ровно это собирает под капотом ChatGPT/Claude, когда ты грузишь CSV и спрашиваешь «топ-5 регионов по выручке». Мы делаем то же руками, чтобы видеть каждый шаг.

РАЗБОР ПО ШАГАМ

Возьмём таблицу продаж и вопрос «выручка по регионам, топ-2» (план: group_by=region, value=amount, agg=sum):

region  amount
Москва  100
Москва   50
Питер    30
Питер    70
Казань   20

split — раскладываем строки по region:

Москва → [100, 50]
Питер  → [30, 70]
Казань → [20]

apply — в каждой группе берём sum колонки amount:

Москва → 100 + 50 = 150
Питер  → 30 + 70  = 100
Казань → 20

combine — собираем {Москва: 150, Питер: 100, Казань: 20} и берём top_n(2) → [(Москва, 150), (Питер, 100)].

Теперь поменяем один кубик — agg=sum на agg=mean — на тех же данных и том же split:

apply (mean):  Москва → (100+50)/2 = 75    Питер → (30+70)/2 = 50    Казань → 20/1 = 20
combine:       {Москва: 75, Питер: 50, Казань: 20}

Данные и группировка те же, а ответ другой: «выручка» (sum 150/100/20) превратилась в «средний чек» (mean 75/50/20), а agg=count дал бы число продаж (2/2/1). Вот почему план важнее данных: один и тот же split отвечает на разные вопросы — решает выбор агрегата.

Создай файл analyze.py с тремя функциями:

ЧАСТЫЕ ОШИБКИ

  • Считать, что ИИ «видит» инсайт в таблице. Он строит план и исполняет код. Нет плана — нет воспроизводимости; проси его показать код/логику, а не только число.
  • Не задать определения метрики. «Выручка» с возвратами или без? «Активные» за какой период? Без явных допущений модель подставит свои — и цифра поедет.
  • Путать агрегаты. sum ≠ mean ≠ count. «Сколько продали» — это sum суммы или count сделок? Уточни кубик agg, иначе ответишь не на тот вопрос.
  • Доверять числу без проверки источника. ИИ не знает, откуда данные и насколько чисты. Сверяй с источником и здравым смыслом (5.5).
  • Грузить гигантский файл целиком в чат. Окно конечно (4.4); для больших данных — выборка, агрегаты заранее или SQL (5.3), а не сырой дамп.

Эталонное решение

Сначала попробуй написать сам по спеке и тестам выше — потом раскрой и сверься.

code/analyze.pyПоказать решение
"""ИИ как аналитик: мини-аналитика на stdlib под планом — Build It для урока 5.1.

Без внешних зависимостей. Промпт превращается в план анализа (что сгруппировать,
какую метрику, как агрегировать), а тулкит его исполняет — как делает ИИ-аналитик,
только мы видим механизм. В USE IT тот же план исполняет pandas / Code Interpreter.
"""
import csv
import io
from statistics import mean


def load_csv(text):
    """CSV-текст -> список словарей (строки таблицы)."""
    return list(csv.DictReader(io.StringIO(text.strip())))


def _num(v):
    try:
        return float(v)
    except (TypeError, ValueError):
        return None


def aggregate(rows, group_by, value, agg="sum"):
    """Сгруппировать rows по group_by и агрегировать value (sum/mean/count)."""
    groups = {}
    for r in rows:
        groups.setdefault(r.get(group_by), []).append(r)
    out = {}
    for key, items in groups.items():
        nums = [n for n in (_num(i.get(value)) for i in items) if n is not None]
        if agg == "count":
            out[key] = len(items)
        elif agg == "mean":
            out[key] = mean(nums) if nums else 0.0
        else:  # sum
            out[key] = sum(nums)
    return out


def top_n(d, n=3):
    """Топ-n пар (ключ, значение) по убыванию значения."""
    return sorted(d.items(), key=lambda kv: kv[1], reverse=True)[:n]


CSV_DEMO = """region,product,amount
Москва,A,100
Москва,B,50
Питер,A,30
Питер,B,70
Казань,A,20
"""

if __name__ == "__main__":
    rows = load_csv(CSV_DEMO)
    by_region = aggregate(rows, "region", "amount", "sum")
    print("Выручка по регионам:", by_region)
    print("Топ-2 региона:", top_n(by_region, 2))
    print("Средний чек по продукту:", aggregate(rows, "product", "amount", "mean"))

# Артефакт: шаблон аналитического запроса

Структурированный запрос к ИИ-аналитику — чтобы ответы были воспроизводимы и проверяемы. Механика плана — [`code/analyze.py`](../code/analyze.py).

## Шаблон

```
# Данные
{что за таблица, какие колонки и типы, источник, период}

# Вопрос
{бизнес-вопрос одним предложением}

# Метрика
{что считаем: SUM(amount) = выручка, COUNT(*) = заказы, AVG = средний чек}

# Разрез
{по чему группируем: регион / месяц / продукт; и фильтры}

# Формат ответа
{таблица топ-N / график / число + вывод одним абзацем}

# Проверка
{покажи код/SQL; проверь, что строки не потеряны и суммы сходятся}
```

## Пример

```
# Данные: sales(region, product, amount) за Q2
# Вопрос: какие регионы дают основную выручку?
# Метрика: выручка = SUM(amount)
# Разрез: по region, топ-5
# Формат: таблица регион→выручка + доля от总
# Проверка: сумма по регионам = общей выручке
```

## Правила

- **Метрику определяй явно** — иначе ИИ выберет свою (семантический слой, урок 5.3).
- **Проси показать код/SQL** — это аудируемость (Code Interpreter, урок 5.2).
- **Всегда добавляй «Проверку»** — суммы, пропуски, аномалии (урок 5.5).
- Считай токены на больших данных (урок 1.1) и сэмплируй, если таблица огромна.
