Урок 0.2 · Аккаунты, тарифы, API-ключи, окружение

Фаза 0 — Setup & Tooling · Результат фазы: Настроить окружение, ключи и агента; осознанно выбрать модель под задачу.

Requires (только для USE IT): платный API-ключ одного из провайдеров (Anthropic / OpenAI / Google). BUILD IT и проверка окружения работают офлайн, без ключей и без сети.

Этот урок — вход в реальную работу с ИИ через код. До сих пор модель можно было дёргать в веб-чате; теперь мы обращаемся к ней программно — а для этого нужны три вещи: аккаунт и API-ключ у провайдера, безопасное место для ключа и воспроизводимое окружение Python. К концу урока у тебя будет настроенный проект и первый настоящий запрос к модели из кода. Настройка делается один раз, и на неё опирается каждый следующий урок курса — поэтому проходим её аккуратно.

MOTTO. Ключи — в .env, секреты — не в коде, окружение — воспроизводимо.

PROBLEM

Первый запрос к API спотыкается не о модель, а о быт. Где взять ключ? Куда его положить, чтобы код его видел, а git — нет? Почему ANTHROPIC_API_KEY «не виден» в соседнем терминале? И как не закоммитить секрет в репозиторий?

Захардкоженный ключ утекает в историю git — и оттуда его уже не вычистить. «Работает у меня» ломается на другой машине, где не тот Python и не те пакеты. Пока окружение не зафиксировано, ты тратишь время не на ИИ, а на ловлю 401 и «module not found».

CONCEPT

Что такое API-ключ и почему он секрет

Когда ты обращаешься к модели не в веб-чате, а кодом (через API), провайдер должен понять, кто делает запрос: чей это аккаунт, с чьего баланса списывать токены, какие применять лимиты. «Удостоверением» служит API-ключ — длинная строка, которую ты создаёшь в консоли своего аккаунта: Anthropic Console, OpenAI Platform или Google AI Studio.

Ключ — это секрет, и относиться к нему надо как к паролю. Документация Google формулирует прямо: «Treat your Gemini API key like a password. If compromised, others can consume your project's quota, incur unexpected billing charges, and access private resources» — у кого ключ, тот шлёт запросы от твоего имени и тратит твои деньги, отдельного логина не нужно. Отсюда два железных правила из тех же доков: не хардкодь ключ в коде и никогда не коммить его в git («Never check API keys into source control»). Если ключ всё же утёк — заходишь в консоль, отзываешь старый и выпускаешь новый.

А подавать ключ в код удобнее всего через окружение: SDK провайдера сам читает его из переменной окружения (ANTHROPIC_API_KEY, OPENAI_API_KEY, GOOGLE_API_KEY) — в квикстарте Anthropic так и сказано: «The SDK reads ANTHROPIC_API_KEY automatically». Значит, ключу не место в исходнике; как сложить это в аккуратное и воспроизводимое окружение — ниже.

Как это работает

Здоровое окружение — это три вещи, которые работают вместе:

venv (изоляция)  +  .env (секреты)  +  .env.example (контракт)
      │                  │                     │
 свои пакеты       ключи вне кода        список ключей без
 не ломают          и вне git            значений — что
 систему           (он в .gitignore)     нужно завести
  • venv — изолированный Python: пакеты урока не конфликтуют с системными и ставятся предсказуемо.
  • .env — файл KEY=VALUE с секретами. Он в .gitignore и в репозиторий не попадает. Загрузить его в окружение помогает load_dotenv() из python-dotenv, после чего SDK берёт ключ из окружения, а не из исходника.
  • .env.example — те же ключи без значений. Коммитится как контракт: «вот что нужно завести, чтобы запустить проект».

Правило одно: секрет живёт в .env, а не в коде. Тогда ключ не утечёт в git, а проект соберётся на любой машине по .env.example.

РАЗБОР ПО ШАГАМ

Прежде чем тратить вызов API на «401 Unauthorized», .env стоит проверить офлайн. Сделаем это руками на типичном «сыром» файле:

# my keys
export ANTHROPIC_API_KEY="sk-ant-demo123"   # рабочий ключ
OPENAI_API_KEY=your-key-here

Шаг 1 — разобрать (parse_env). Идём по строкам. # my keys — комментарий, пропускаем. У второй строки снимаем export , режем кавычки и inline-комментарий # рабочий ключ. Третью берём как есть. Получаем словарь:

{ "ANTHROPIC_API_KEY": "sk-ant-demo123", "OPENAI_API_KEY": "your-key-here" }

Шаг 2 — проверить (validate) по требуемым ключам ANTHROPIC_API_KEY, OPENAI_API_KEY, GOOGLE_API_KEY:

  • ANTHROPIC_API_KEY — есть, начинается с sk-ant- → ок;
  • OPENAI_API_KEY — значение your-key-here из списка плейсхолдеров → «похоже на плейсхолдер»;
  • GOOGLE_API_KEY — в файле нет → «отсутствует».

Две проблемы пойманы за секунды и офлайн — до единого запроса к API. Ровно это делает валидатор из Build It.

BUILD IT

Задание: собери офлайн-валидатор окружения — стандартная библиотека, без сети. Он ловит типовые ошибки .env до первого запроса.

Перед запуском. Это первое кодовое упражнение курса: заведи папку урока с подпапкой code/ внутри — ai-native/0.2-accounts-keys-env/code/ — и клади туда решение и скопированный тест. Нужен только Python 3 и pytest (pip install pytest, если ещё не ставил), всё офлайн и без ключей. Настоящие ключи и venv заведём ниже, в USE IT.

Создай envcheck.py. Константы бери готовыми (это просто данные):

# значения, которыми выдаёт себя незаполненный .env
PLACEHOLDERS = {"", "your-key-here", "changeme", "xxx", "...", "todo"}
# ожидаемый префикс ключа по имени (пустой — формат не проверяем)
KEY_HINTS = {"ANTHROPIC_API_KEY": "sk-ant-", "OPENAI_API_KEY": "sk-", "GOOGLE_API_KEY": ""}

Реализуй две функции:

  • parse_env(text) — текст .env превращает в dict {KEY: VALUE}. Пропускай пустые строки и комментарии (#); снимай ведущий export ; у значения убирай кавычки и inline-комментарий ( # ...).
  • validate(env, required) — возвращает список проблем по требуемым ключам (пустой список = всё ок): ключа нет → "{key}: отсутствует"; значение из PLACEHOLDERS → "{key}: похоже на плейсхолдер"; значение не начинается с префикса из KEY_HINTS → "{key}: ожидался префикс ...".

Готово, когда тест test_envcheck.py зелёный — он проверяет разбор (комментарии, export, кавычки) и все три типа проблем валидации.

pytest code -q             # сейчас красное → реализуй две функции → стало зелёным
python code/envcheck.py    # демо: распознанные ключи и найденные проблемы

Подсказка. В parse_env иди построчно: strip, отсекай пустые и #, срезай export , дели по первому =. Значение чисти отдельно: если в кавычках — бери содержимое, иначе режь хвост #комментарий.

Внизу, в «Исходниках урока», — три способа пройти упражнение (собрать самому · подсмотреть эталон · делегировать ИИ) и тесты-ТЗ.

USE IT

Теперь заведём настоящее окружение и сделаем первый реальный запрос. Создай для курса папку ai-native/ и преврати её в проект с виртуальным окружением:

cd ai-native
python -m venv .venv && source .venv/bin/activate   # изолированный Python проекта (Windows: .venv\Scripts\activate)
pip install anthropic openai google-genai python-dotenv   # SDK + загрузчик .env

Заведи ключи в консолях провайдеров и положи их в .env. Файлы .env.example и check_env.py бери из артефакта Ship It (env-starter/):

  • Anthropic Console → ANTHROPIC_API_KEY (sk-ant-…)
  • OpenAI Platform → OPENAI_API_KEY (sk-…)
  • Google AI Studio → GOOGLE_API_KEY
cp .env.example .env       # 1) создать локальный .env
# 2) вписать в .env реальные ключи
echo ".env" >> .gitignore  # 3) .env НИКОГДА не коммитится
python check_env.py        # 4) офлайн-проверка (тот же валидатор) — exit 0 = ок

Только после зелёной проверки — первый запрос. Форма почти одинакова у всех, а ключ берётся из окружения, не из кода:

from dotenv import load_dotenv
load_dotenv()   # положить ключи из .env в окружение — дальше SDK берёт их сам

# Anthropic
from anthropic import Anthropic
print(Anthropic().messages.create(model="claude-haiku-4-5", max_tokens=50,
    messages=[{"role": "user", "content": "Привет одним словом"}]).content[0].text)

# OpenAI
from openai import OpenAI
print(OpenAI().responses.create(model="gpt-5.x", input="Привет одним словом").output_text)

# Google
from google import genai
print(genai.Client().models.generate_content(
    model="gemini-2.x-flash", contents="Привет одним словом").text)
Про имена моделей в примерах. По всему курсу gpt-5.x и gemini-2.x-flash — условные обозначения поколений, а не готовые ID: подставляй точный идентификатор из доки провайдера (у Claude ID пишем полностью, напр. claude-haiku-4-5). Модели и их ID обновляются — сверяйся с актуальным списком. И ещё: код в USE IT дальше по курсу — это наброски (импорты как здесь, данные подставляешь свои), а рабочий с тестами код всегда в BUILD IT.

Для проверки бери самую дешёвую модель (Haiku / Flash) — первый запрос не должен стоить дорого (правило из урока 0.1).

ЧАСТЫЕ ОШИБКИ

  • Ключ прямо в коде. Захардкоженный секрет утекает в git и в историю (даже если потом удалить). Ключи — только в .env, а .env — в .gitignore.
  • Закоммитить .env. Сразу добавь .env в .gitignore. .env.example (без значений) коммить, .env — никогда.
  • Проверять ключ запросом к API. 401 стоит времени и иногда денег. Сначала офлайн-валидатор (check_env.py), потом запрос.
  • Работать без venv. Глобальная установка пакетов рано или поздно ломает другой проект, а «работает у меня» не воспроизводится. Заводи venv на проект.

Тесты — это твоё ТЗ

code/test_envcheck.py
from envcheck import parse_env, validate


def test_parse_ignores_comments_and_handles_export_quotes():
    text = """
    # comment line
    export ANTHROPIC_API_KEY="sk-ant-abc"   # inline comment
    OPENAI_API_KEY = sk-xyz
    EMPTY=
    """
    env = parse_env(text)
    assert env["ANTHROPIC_API_KEY"] == "sk-ant-abc"
    assert env["OPENAI_API_KEY"] == "sk-xyz"
    assert env["EMPTY"] == ""
    assert "comment line" not in env


def test_validate_detects_missing_and_placeholder():
    env = {"ANTHROPIC_API_KEY": "your-key-here"}
    problems = validate(env, ["ANTHROPIC_API_KEY", "OPENAI_API_KEY"])
    assert any("ANTHROPIC_API_KEY" in p and "плейсхолдер" in p for p in problems)
    assert any("OPENAI_API_KEY" in p and "отсутствует" in p for p in problems)


def test_validate_detects_bad_prefix():
    problems = validate({"ANTHROPIC_API_KEY": "wrong-prefix"}, ["ANTHROPIC_API_KEY"])
    assert problems and "префикс" in problems[0]


def test_validate_passes_good_keys():
    env = {"ANTHROPIC_API_KEY": "sk-ant-ok", "OPENAI_API_KEY": "sk-ok", "GOOGLE_API_KEY": "AIza..."}
    assert validate(env, list(env)) == []

Эталонное решение

Сначала попробуй написать сам по спеке и тестам выше — потом раскрой и сверься.

code/envcheck.pyПоказать решение
"""Парсер .env + валидатор ключей окружения — Build It для урока 0.2.

Без внешних зависимостей и без сети. Разбирает файл .env (KEY=VALUE), понимает
комментарии, `export`, кавычки и inline-комментарии; проверяет, что нужные ключи
заданы и не остались плейсхолдерами. Реальный первый запрос к API — в USE IT.
"""
from __future__ import annotations

# Значения, которые выдают незаполненный .env.
PLACEHOLDERS = {"", "your-key-here", "changeme", "xxx", "...", "todo"}

# Грубая проверка формата по префиксу (без сети). Пустой префикс — не проверяем.
KEY_HINTS = {
    "ANTHROPIC_API_KEY": "sk-ant-",
    "OPENAI_API_KEY": "sk-",
    "GOOGLE_API_KEY": "",
}


def _clean_value(val: str) -> str:
    """Снять кавычки и inline-комментарий со значения."""
    val = val.strip()
    if val[:1] in {'"', "'"}:          # значение в кавычках — берём содержимое
        quote = val[0]
        return val[1:].split(quote, 1)[0]
    return val.split(" #", 1)[0].strip()   # без кавычек — режем ` #комментарий`


def parse_env(text: str) -> dict:
    """Текст .env -> dict. Пустые строки и комментарии игнорируются."""
    env = {}
    for raw in text.splitlines():
        line = raw.strip()
        if not line or line.startswith("#"):
            continue
        if line.startswith("export "):
            line = line[len("export "):].strip()
        if "=" not in line:
            continue
        key, _, val = line.partition("=")
        env[key.strip()] = _clean_value(val)
    return env


def validate(env: dict, required) -> list[str]:
    """Список проблем по требуемым ключам (пустой список = всё в порядке)."""
    problems = []
    for key in required:
        val = env.get(key)
        if val is None:
            problems.append(f"{key}: отсутствует")
        elif val.lower() in PLACEHOLDERS:
            problems.append(f"{key}: похоже на плейсхолдер")
        else:
            hint = KEY_HINTS.get(key, "")
            if hint and not val.startswith(hint):
                problems.append(f"{key}: не похоже на ключ (ожидался префикс {hint!r})")
    return problems


if __name__ == "__main__":
    sample = """
    # пример .env
    export ANTHROPIC_API_KEY="sk-ant-demo123"   # ключ Anthropic
    OPENAI_API_KEY=your-key-here
    """
    env = parse_env(sample)
    print("Распознано ключей:", list(env))
    problems = validate(env, ["ANTHROPIC_API_KEY", "OPENAI_API_KEY", "GOOGLE_API_KEY"])
    print("Проблемы:", problems or "нет")

# env-starter

Drop-in набор для воспроизводимого окружения в любом проекте курса.

## Файлы

- `.env.example` — контракт ключей (без значений). Коммитится.
- `check_env.py` — самодостаточный валидатор (без зависимостей, без сети).

## Использование

```bash
cp .env.example .env          # 1) создать локальный .env
# 2) вписать реальные ключи из консолей Anthropic / OpenAI / Google
echo ".env" >> .gitignore     # 3) НИКОГДА не коммить .env
python check_env.py           # 4) проверить, что всё на месте (exit 0 = ок)
```

## Зачем

Ловит типовые ошибки (нет ключа, остался `your-key-here`, неверный префикс) офлайн —
до первого 401 от API. `check_env.py` можно повесить в CI или pre-commit.
outputs/env-starter/check_env.py
"""check_env.py — самодостаточный валидатор окружения (drop-in, без зависимостей).

Запуск:  python check_env.py [путь_к_.env]   (по умолчанию ./.env)
Код выхода 0 — всё ок, 1 — есть проблемы. Сети не делает.
"""
import sys
from pathlib import Path

PLACEHOLDERS = {"", "your-key-here", "changeme", "xxx", "...", "todo"}
KEY_HINTS = {"ANTHROPIC_API_KEY": "sk-ant-", "OPENAI_API_KEY": "sk-", "GOOGLE_API_KEY": ""}
REQUIRED = list(KEY_HINTS)


def _clean(val):
    val = val.strip()
    if val[:1] in {'"', "'"}:
        return val[1:].split(val[0], 1)[0]
    return val.split(" #", 1)[0].strip()


def parse_env(text):
    env = {}
    for raw in text.splitlines():
        line = raw.strip()
        if not line or line.startswith("#") or "=" not in line:
            continue
        if line.startswith("export "):
            line = line[len("export "):].strip()
        key, _, val = line.partition("=")
        env[key.strip()] = _clean(val)
    return env


def validate(env, required=REQUIRED):
    problems = []
    for key in required:
        val = env.get(key)
        if val is None:
            problems.append(f"{key}: отсутствует")
        elif val.lower() in PLACEHOLDERS:
            problems.append(f"{key}: похоже на плейсхолдер")
        else:
            hint = KEY_HINTS.get(key, "")
            if hint and not val.startswith(hint):
                problems.append(f"{key}: ожидался префикс {hint!r}")
    return problems


if __name__ == "__main__":
    path = Path(sys.argv[1] if len(sys.argv) > 1 else ".env")
    if not path.exists():
        print(f"Нет файла {path}. Скопируй .env.example в .env и заполни.")
        sys.exit(1)
    problems = validate(parse_env(path.read_text(encoding="utf-8")))
    if problems:
        print("Проблемы окружения:")
        for p in problems:
            print("  -", p)
        sys.exit(1)
    print("Окружение в порядке: все ключи заданы.")
