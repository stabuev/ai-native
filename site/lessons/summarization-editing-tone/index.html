Урок 3.1 · Резюмирование, редактура, тон

Фаза 3 — Текст и документы · Результат фазы: Строить пайплайны резюмирования и редактуры; собрать базовый RAG над своим корпусом.

Requires (только для USE IT): платный API-ключ одного из провайдеров. BUILD IT работает офлайн, без ключей и без сети.

Фаза 3 — про работу с текстом и документами, и начинается она с редакторского конвейера. Большая часть рабочих задач — это текст: письма, отчёты, переписка. Здесь мы строим пайплайн «сжать → почистить → задать тон», где каждая стадия — отдельный шаг с контрактом «текст → текст». Это прямое применение декомпозиции из урока 2.3 к тексту, а артефакт — переиспользуемый скилл editor.

MOTTO. Текст обрабатывают конвейером: сжать → почистить → задать тон.

PROBLEM

«Сделай покороче и поприличнее» одним махом ненадёжно: то смысл потеряется при сжатии, то тон уедет не туда, и непонятно, какая часть виновата. На длинных письмах и отчётах это особенно больно. Нужен конвейер с раздельными стадиями, которые можно настраивать, менять и проверять по отдельности — а не один расплывчатый запрос.

CONCEPT

Интуиция

Текст обрабатывают как на сборочной линии: каждая стадия делает одно понятное преобразование и передаёт результат дальше. У всех стадий один контракт — «текст на входе, текст на выходе», поэтому их легко переставлять, заменять и оценивать. Один мегазапрос «улучши всё» такого контроля не даёт.

Как это работает

сырой текст → [черновик/резюме] → [редактура] → [тон] → готово
                 что важно            убрать лишнее   под аудиторию

Три стадии конвейера:

  • Резюме. Сжать до главного. Тут важно различать два подхода:
  • Extractive (наш офлайн-билд) — выбрать самые важные существующие предложения, ничего не переписывая. Важность оцениваем по частоте значимых слов: предложение из частых слов темы весит больше. Прозрачно и дёшево, но «рвано».
  • Abstractive (то, что делает LLM в USE IT) — пересказать смысл своими словами. Глаже и человечнее, но дороже и может «досочинить» (привет, галлюцинации из 1.4).
  • Редактура (tighten). Убрать слова-филлеры («очень», «просто», «буквально») и лишние пробелы — текст становится плотнее без потери смысла.
  • Тон (set_tone). Подогнать под аудиторию: формальный/неформальный. В офлайн-билде это словарь замен-маркеров (привет → Здравствуйте), в проде — инструкция модели «перепиши в деловом тоне».

Почему именно конвейер, а не один промпт: каждую стадию можно менять и измерять независимо (eval из урока 2.5 — на каждую стадию свой набор кейсов), а контракт «текст → текст» позволяет заменить офлайн-функцию на вызов модели, не трогая остальной пайплайн.

РАЗБОР ПО ШАГАМ

Прогоним конвейер руками на тексте:

Кошки любят рыбу. Кошки очень любят свежую рыбу. Погода сегодня внезапно испортилась.

Шаг 1 — summarize_extractive(text, 1). Считаем частоту значимых слов (длиннее 3 букв) по всему тексту: кошки×2, любят×2, рыбу×2, остальные по 1. Оцениваем каждое предложение суммой частот его слов:

"Кошки любят рыбу."                  → 2+2+2 = 6
"Кошки очень любят свежую рыбу."     → 2+1+2+1+2 = 8   ← максимум
"Погода сегодня внезапно испортилась." → 1+1+1+1 = 4

Берём одно предложение с максимальным score → «Кошки очень любят свежую рыбу.» (редкое предложение про погоду отброшено).

Шаг 2 — tighten. Убираем филлер очень и схлопываем пробелы → «Кошки любят свежую рыбу.»

Шаг 3 — set_tone(…, "formal"). В тексте нет маркеров из словаря (привет/пока/спасибо), поэтому он не меняется → «Кошки любят свежую рыбу.»

Итог конвейера — "Кошки любят свежую рыбу.". Каждый шаг видно отдельно: если результат не нравится, понятно, какую стадию крутить.

FILLER_WORDS = {"очень", "просто", "буквально", "реально", "вообще", "типа"}
TONE = {
    "formal": {"привет": "Здравствуйте", "пока": "До свидания", "спасибо": "Благодарю"},

Реализуй стадии (все с контрактом «строка → строка», кроме split_sentences):

  • split_sentences(text) — разбей на предложения (по .!?), верни непустые, обрезанные по краям.
  • summarize_extractive(text, n=2) — extractive-резюме: если предложений ≤ n, верни все; иначе посчитай частоты значимых слов (длиннее 3 букв) по всему тексту, оцени каждое предложение суммой частот его слов, возьми n лучших и верни в исходном порядке.
  • tighten(text) — убери слова из FILLER_WORDS (сравнивай без пунктуации по краям, регистронезависимо) и схлопни пробелы.
  • set_tone(text, tone) — замени маркеры по словарю TONE[tone] (регистронезависимо).
  • run_pipeline(text, stages) — прогони текст через список стадий (callable str -> str).

Готово, когда тест test_editing_pipeline.py зелёный — он проверяет, что резюме короче и держит тему (а редкое предложение отброшено), что мало предложений возвращаются целиком, что tighten убирает филлеры и пробелы, что set_tone меняет маркеры, и что run_pipeline композирует стадии.

Подсказка. В summarize_extractive держи частоты в Counter, а функцию score(s) считай как сумму freq[w] по словам предложения (короткие слова дадут 0). Верни выбранные индексы, отсортированные по возрастанию, — так сохранится исходный порядок.

USE IT

Те же стадии — промптами к модели на реальных документах и переписке (в проде резюме и тон делаются abstractive, своими словами). Мульти-провайдер:

def stage(instruction, text, model="claude-sonnet-4-6"):
    p = f"{instruction}\n\nТекст:\n{text}"
    return Anthropic().messages.create(model=model, max_tokens=600,
        messages=[{"role": "user", "content": p}]).content[0].text
    # OpenAI:  OpenAI().responses.create(model="gpt-5.x", input=p).output_text
    # Google:  genai.Client().models.generate_content(model="gemini-2.x-flash", contents=p).text

draft = stage("Сожми в 3 пункта", document)

ЧАСТЫЕ ОШИБКИ

  • Один мегазапрос «улучши всё». Сжатие, чистку и тон лучше делать раздельными стадиями — иначе непонятно, что сломало результат, и нечего настраивать по отдельности.
  • Путать extractive и abstractive. Extractive выбирает готовые предложения (ничего не выдумает, но «рвано»); abstractive переписывает (гладко, но может досочинить). Под факты — следи за abstractive (фактчекинг из 1.4).
  • Агрессивная чистка. Срезать «воду» полезно, но удаление слов может менять смысл и тон — проверяй результат, особенно на коротких текстах.
  • Тон важнее смысла. Сначала верное содержание (резюме + редактура), потом тон. Красиво, но неверно — хуже, чем сухо, но верно.

code/editing_pipeline.pyПоказать решение
"""Резюмирование, редактура, тон: текстовый пайплайн — Build It для урока 3.1.

Без зависимостей. Пайплайн «черновик → редактура → тон»: extractive-резюме
(частотная оценка предложений), чистка (filler-слова и пробелы), смена тона
(словарь маркеров). Стадии — детерминированные функции (офлайн); в USE IT
каждую стадию играет модель.
"""
import re
from collections import Counter

_SENT = re.compile(r"[^.!?]+[.!?]?")
FILLER_WORDS = {"очень", "просто", "буквально", "реально", "вообще", "типа"}


def split_sentences(text):
    return [s.strip() for s in _SENT.findall(text) if s.strip()]


def summarize_extractive(text, n=2):
    """Черновик: n самых «важных» предложений по частоте значимых слов."""
    sents = split_sentences(text)
    if len(sents) <= n:
        return " ".join(sents)
    words = re.findall(r"\w+", text.lower())
    freq = Counter(w for w in words if len(w) > 3)

    def score(s):
        return sum(freq[w] for w in re.findall(r"\w+", s.lower()))

    top = sorted(range(len(sents)), key=lambda i: score(sents[i]), reverse=True)[:n]
    return " ".join(sents[i] for i in sorted(top))      # в исходном порядке


def tighten(text):
    """Редактура: убрать слова-филлеры и схлопнуть пробелы."""
    kept = [w for w in text.split() if w.lower().strip(",.!?") not in FILLER_WORDS]
    return re.sub(r"\s+", " ", " ".join(kept)).strip()


TONE = {
    "formal": {"привет": "Здравствуйте", "пока": "До свидания", "спасибо": "Благодарю"},
    "casual": {"здравствуйте": "привет", "благодарю": "спасибо"},
}


def set_tone(text, tone):
    """Тон: заменить маркеры по словарю выбранного тона."""
    for src, dst in TONE.get(tone, {}).items():
        text = re.sub(src, dst, text, flags=re.IGNORECASE)
    return text


def run_pipeline(text, stages):
    """Прогнать текст через список стадий (callable: str -> str)."""
    for fn in stages:
        text = fn(text)
    return text


if __name__ == "__main__":
    doc = ("Кошки любят рыбу. Кошки очень любят свежую рыбу. "
           "Погода сегодня внезапно испортилась.")
    pipeline = [lambda t: summarize_extractive(t, 1), tighten,
                lambda t: set_tone(t, "formal")]
    print("Исходник:", doc)
    print("Итог    :", run_pipeline(doc, pipeline))

## Как пользоваться

1. **Черновик/резюме** — сжать до главного (extractive или промптом «сожми в N пунктов»).
2. **Редактура** — убрать воду, филлеры, повторы; короче и яснее.
3. **Тон** — привести под аудиторию (деловой / нейтральный / дружелюбный).

    return re.sub(r"\s+", " ", " ".join(kept)).strip()

    "formal": {"привет": "Здравствуйте", "пока": "До свидания", "спасибо": "Благодарю"},
