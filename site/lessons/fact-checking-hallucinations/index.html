Урок 11.2 · Фактчекинг и борьба с галлюцинациями

Фаза 11 — Этика, безопасность, governance · Результат фазы: Оценить риски, настроить приватность и написать политику использования ИИ.

Requires (только для USE IT): платный API-ключ одного из провайдеров. BUILD IT работает офлайн, без ключей и без сети.

В 11.1 мы закрыли утечку данных. Теперь — второй governance-риск: выдуманные факты. Модель уверенно генерирует то, чего нет в источниках (урок 1.4), и в отчёте или письме клиенту это стоит дорого.

MOTTO. Доверяй, но заземляй: каждое фактическое утверждение должно опираться на источник.

PROBLEM

Модель звучит одинаково уверенно, права она или галлюцинирует. Неверная цифра в отчёте, выдуманная ссылка в письме, несуществующий пункт договора — всё это выглядит правдоподобно и потому опасно (та же ловушка, что в 5.5 с данными).

«Перечитать глазами» не масштабируется. Нужен пайплайн фактчекинга: разложить ответ на отдельные утверждения и проверить каждое на заземлённость (grounding) — есть ли под ним опора в доверенных источниках. Незаземлённое = кандидат в галлюцинации и в ручную проверку. Соберём такой пайплайн.

CONCEPT

Интуиция

Фактчекинг — это работа редактора-проверяющего, а не «верю/не верю». Редактор берёт текст, разбивает на конкретные утверждения и под каждым ищет ссылку на источник: есть опора — ок, нет опоры — на полях ставит «проверить». Он не решает, правда ли это «в принципе», — он проверяет, подкреплено ли это тем, чему мы доверяем.

Ключевой сдвиг: галлюцинация — это не «модель соврала», а утверждение без опоры. Поэтому фокус не на «звучит ли убедительно», а на «есть ли источник».

Как это работает

ответ → extract_claims (разбить на утверждения)
           │ для каждого:
   verify_claim(claim, sources): есть ли заземление в источниках?
           │
   supported / unsupported → отчёт (что проверять руками)
  • extract_claims(text) — разбить ответ на отдельные утверждения (по предложениям).
  • verify_claim(claim, sources) — заземлено ли утверждение: считаем перекрытие слов с каждым источником (Jaccard), и если максимум ≥ порога → supported, иначе unsupported.
  • fact_check(text, sources) — отчёт по всем утверждениям: что заземлено, что под вопросом.

Это грубое лексическое заземление — по совпадению слов. Оно ловит явное (выдуманный факт не пересекается с источниками), но у него есть фундаментальный предел, который мы увидим в разборе. В проде заземление делают NLI/entailment-моделью (проверяет логическое следование, а не слова) или эмбеддингами.

РАЗБОР ПО ШАГАМ

Источник: «Возврат товара возможен в течение 14 дней… Гарантия на электронику — 12 месяцев». Проверяем ответ "Возврат возможен в течение 14 дней. Гарантия длится 100 лет." (порог 0.4):

claim 1: "Возврат возможен в течение 14 дней"
   Jaccard с источником = 0.6 ≥ 0.4 → supported   (реально заземлено)

claim 2: "Гарантия длится 100 лет"
   Jaccard ≈ 0.111 < 0.4 → unsupported            (выдумка поймана)

Пока всё хорошо: выдуманные «100 лет» не пересеклись с источником и помечены на проверку. Но вот предел лексического метода:

claim: "Возврат НЕВОЗМОЖЕН в течение 14 дней"   (прямое отрицание источника!)
   Jaccard ≈ 0.455 ≥ 0.4 → supported  ← ЛОЖНО заземлено

Утверждение, противоречащее источнику, делит с ним почти все слова (возврат, в течение 14 дней) и проходит порог — хотя смысл обратный. Это и есть граница: совпадение слов ≠ логическое следование. Лексический фактчекер ловит «не из источника», но слеп к отрицаниям и подменам чисел. Поэтому он — первый дешёвый фильтр, а решает entailment-модель (NLI) или человек. Поменяй Jaccard на NLI — контракт «извлеки → проверь заземление → пометь» не изменится.

BUILD IT

Задание: собери пайплайн фактчекинга — извлечение утверждений, проверка заземления, отчёт. Только стандартная библиотека (re), без сети.

Перед запуском. Работай в своей папке курса (ai-native/11.2-fact-check/), а файлы урока клади в подпапку code/ (по соглашению курса). Нужен только Python 3 (для теста ещё pytest).

Создай файл fact_check.py:

  • extract_claims(text) — разбить текст на утверждения по [.!?] (пустые отбросить).
  • verify_claim(claim, sources, threshold=0.4) — для каждого источника посчитать Jaccard слов (|A∩B| / |A∪B|); если максимум ≥ threshold → "supported", иначе "unsupported".
  • fact_check(text, sources) — вернуть список {claim, status} по всем утверждениям.

Готово, когда все тесты в test_fact_check.py зелёные — они проверяют: разбиение на утверждения; заземлённое утверждение → supported; выдуманное → unsupported; отчёт содержит и то, и другое.

pytest code -q          # красное → реализуй пайплайн → зелёное
python code/fact_check.py # демо: «14 дней» supported, «100 лет» unsupported

Подсказка. Слова — set(re.findall(r"\w+", text.lower())). Jaccard — len(a & b) / len(a | b) (защити от деления на ноль). Статус — по максимуму перекрытия среди всех источников.

Внизу, в «Исходниках урока», — три способа пройти упражнение (собрать самому · подсмотреть эталон · делегировать ИИ) и тесты-ТЗ.

USE IT

Перекрёстная проверка моделями и источниками (мульти-провайдер):

  • Grounding/цитаты — проси модель цитировать источник под каждый факт (Anthropic «reduce hallucinations»): без цитаты факт не считается заземлённым.
  • Перекрёстная проверка — задай тот же вопрос другой модели; расхождение ответов = красный флаг на ручную проверку.
  • RAG как источник истины — отвечать только по найденным документам (Фаза 3) и явно помечать незаземлённое.
  • Право на «не знаю» — разреши модели отказаться, это снижает выдумывание (1.4).
  • Entailment вместо слов — для серьёзного фактчекинга бери NLI-модель (проверяет следование), а не лексическое перекрытие: оно слепо к отрицаниям и подменам чисел.

ЧАСТЫЕ ОШИБКИ

  • Принимать уверенный тон за правду. Модель звучит одинаково, права она или нет. Фактчек проверяет опору на источник, а не убедительность формулировки.
  • Считать лексическое совпадение доказательством. Совпадение слов ≠ логическое следование: утверждение с отрицанием («невозможен») или подменой числа делит слова с источником и ложно проходит. Для серьёзного — NLI/entailment.
  • Проверять ответ целиком, а не по утверждениям. Длинный ответ смешивает верное и выдуманное. Разбивай на отдельные claims — иначе одна галлюцинация прячется среди фактов.
  • Не давать право на «не знаю». Если модель обязана ответить, она выдумает. Явное разрешение отказаться + grounding снижают галлюцинации (1.4).
  • Доверять одному источнику/модели. Перекрёстная проверка (другая модель, другой источник) ловит то, что один прогон пропустит; расхождение — сигнал к ручной проверке.

from fact_check import extract_claims, verify_claim, fact_check

SOURCES = [
    "Возврат товара возможен в течение 14 дней с момента покупки.",
    "Гарантия на электронику составляет 12 месяцев.",
]


def test_extract_claims_splits_sentences():
    claims = extract_claims("Первое утверждение. Второе утверждение!")
    assert len(claims) == 2


def test_grounded_claim_supported():
    assert verify_claim("Возврат возможен в течение 14 дней", SOURCES) == "supported"


def test_ungrounded_claim_unsupported():
    assert verify_claim("Гарантия длится 100 лет на всё", SOURCES) == "unsupported"


def test_fact_check_report():
    report = fact_check("Возврат возможен в течение 14 дней. Доставка на Марс бесплатна.", SOURCES)
    statuses = {r["claim"]: r["status"] for r in report}
    assert any(s == "supported" for s in statuses.values())
    assert any(s == "unsupported" for s in statuses.values())

Эталонное решение

Сначала попробуй написать сам по спеке и тестам выше — потом раскрой и сверься.

code/fact_check.pyПоказать решение
"""Фактчекинг: проверка утверждений на заземление — Build It для урока 11.2.

Без зависимостей. Разбиваем ответ на утверждения и проверяем каждое на
заземлённость в источниках (grounding): есть ли утверждение в нашей базе фактов.
Незаземлённое — кандидат в галлюцинации. В USE IT — перекрёстная проверка
моделями и реальными источниками.
"""
import re


def _tokens(text):
    return set(re.findall(r"\w+", text.lower()))


def extract_claims(text):
    """Разбить текст на утверждения (по предложениям)."""
    return [s.strip() for s in re.split(r"[.!?]\s*", text) if s.strip()]


def verify_claim(claim, sources, threshold=0.4):
    """Заземлено ли утверждение в источниках. 'supported' | 'unsupported'."""
    c = _tokens(claim)
    if not c:
        return "unsupported"
    best = 0.0
    for src in sources:
        s = _tokens(src)
        jaccard = len(c & s) / len(c | s) if (c | s) else 0.0
        best = max(best, jaccard)
    return "supported" if best >= threshold else "unsupported"


def fact_check(text, sources):
    """Отчёт по утверждениям: что заземлено, что нет."""
    return [{"claim": cl, "status": verify_claim(cl, sources)}
            for cl in extract_claims(text)]


if __name__ == "__main__":
    sources = [
        "Возврат товара возможен в течение 14 дней с момента покупки.",
        "Гарантия на электронику составляет 12 месяцев.",
    ]
    answer = "Возврат возможен в течение 14 дней. Гарантия длится 100 лет."
    for r in fact_check(answer, sources):
        print(f"[{r['status']}] {r['claim']}")

# Артефакт: чек-лист фактчекинга

Перед тем как выпустить ответ ИИ наружу. Пайплайн в коде — [`code/fact_check.py`](../code/fact_check.py).

## Когда фактчекать обязательно

- [ ] Ответ содержит **числа, даты, имена, цитаты, ссылки**.
- [ ] Ответ уходит **наружу** (клиент, отчёт, публикация).
- [ ] **Высокая цена ошибки** (юр./фин./мед.).

## Как проверять

- [ ] **Разложи на утверждения** (`extract_claims`) и проверь каждое.
- [ ] **Заземление**: каждое фактическое утверждение опирается на источник (`verify_claim`).
- [ ] **Цитаты/ссылки**: модель приводит источник для каждого факта.
- [ ] **Перекрёстная проверка**: тот же вопрос на другой модели/прогоне — расхождения = флаг.
- [ ] **RAG как истина**: отвечать только по найденным документам (Фаза 3).
- [ ] **«Не знаю» разрешено** — лучше воздержаться, чем выдумать (урок 1.4).

## Что эскалировать человеку

- [ ] Незаземлённые утверждения (`unsupported`).
- [ ] Противоречия между источниками/моделями.
- [ ] Критичные решения с высокой ценой ошибки.

## Применение (код)

```python
from fact_check import fact_check
report = fact_check(answer, sources)
unsupported = [r["claim"] for r in report if r["status"] == "unsupported"]
if unsupported:

```
