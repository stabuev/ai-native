Урок 4.5 · Агентные платформы: OpenClaw / Hermes

Фаза 4 — Скиллы, память и проекты (слой персистентности) · Результат фазы: Упаковывать способности в SKILL.md, хранить контекст в памяти и собирать рабочие проекты.

Requires (только для USE IT): self-hosted платформа (OpenClaw/Hermes) + API-ключ модели. BUILD IT работает офлайн, без ключей и без сети.

Это финал фазы — всё собирается в одно. Скилл (4.1), память (4.2), проект (4.3), бюджет контекста (4.4) по отдельности — детали. Агентная платформа склеивает их в постоянного персонального агента, который живёт у тебя и доступен отовсюду.

MOTTO. Скилл + память + мульти-модель = персональный агент, который живёт у тебя и доступен везде.

PROBLEM

У тебя уже есть скиллы и память — но как превратить их в живого ассистента, который не исчезает в конце сессии и доступен не только в одном чате? В 2026 для этого появились self-hosted агентные платформы: они грузят твои скиллы, помнят контекст между сессиями, ходят в несколько моделей и отвечают прямо в твоих мессенджерах.

Ключевое слово — self-hosted: агент крутится на твоём железе, данные не уходят наружу (мост к приватности, 11.1), а скиллы переносимы между платформами по общему стандарту. Чтобы это не было «магией», соберём уменьшенный рантайм сами и потом сравним готовые платформы.

CONCEPT

Интуиция

Представь личного секретаря, который всегда на месте. У него есть набор навыков (скиллы), он помнит ваши прошлые разговоры (память), может позвонить разным специалистам под задачу (мульти-модель и инструменты) и принимает поручения по любому каналу — телефон, мессенджер, почта. Агентная платформа — это и есть такой секретарь в коде: один процесс, который связывает всё, что мы строили в фазе, и работает без тебя за пультом.

В отличие от чата в браузере, он твой: запущен на твоём сервере, со своими ключами и данными.

Как это работает

self-hosted агент-рантайм (OpenClaw / Hermes)
   ├─ skills (SKILL.md / agentskills.io)   ← способности (4.1)
   ├─ memory (между сессиями)               ← контекст (4.2)
   ├─ model router (любая модель)           ← мульти-модель (Фаза 9)
   ├─ MCP / tools                           ← инструменты (Фаза 6)
   └─ channels (Telegram / Slack / …)       ← доступ отовсюду

Внутри цикл на каждое входящее сообщение прост: запомнить → выбрать скилл → исполнить.

  • load(skill) — зарегистрировать скилл в реестре (парсим SKILL.md как в 4.1).
  • route(message) — выбрать подходящий скилл по совпадению слов сообщения с description (тот же матчинг, что в 4.1; в реальной платформе — семантический).
  • handle(message) — записать сообщение в память, сроутить, исполнить хендлер скилла, вернуть результат.

Это и есть скелет OpenClaw/Hermes: реестр скиллов + маршрутизация + память между сообщениями. Каналы и роутер моделей — обвязка вокруг этого ядра.

РАЗБОР ПО ШАГАМ

Прогоним handle руками. В рантайм загружены два скилла (по их description):

editor : "редактирует и сокращает текст письма"   handler → "сокращено"
calc   : "считает арифметику числа и проценты"     handler → "42"

Приходит сообщение «отредактируй текст письма»:

1. memory.append("отредактируй текст письма")     → в памяти 1 сообщение
2. route: слова {отредактируй, текст, письма}
     ∩ editor = {текст, письма} → 2   ← выбран
     ∩ calc   = {}              → 0
3. handler editor("...") → "сокращено"
   → {"skill": "editor", "result": "сокращено"}

Победил editor (совпало 2 слова против 0). Следующее «посчитай проценты» уйдёт в calc (совпадёт проценты), а память вырастет до 2. А «погода завтра» даст 0 совпадений у обоих → route вернёт None, и рантайм честно ответит «нет подходящего скилла» — это та же логика выбора, что в уроке 4.1, только теперь с памятью и исполнением.

Перед запуском. Работай в своей папке курса (ai-native/4.5-agent-platforms/), а файлы урока клади в подпапку code/ (по соглашению курса). Нужен только Python 3 (для теста ещё pytest).

Создай файл skill_runtime.py: функцию parse_skill(text) (как в 4.1 — frontmatter name/description + тело) и класс Runtime с памятью:

  • load(skill_text, handler=None) — распарсить скилл, положить в self.skills[name], а handler (функцию-исполнитель) — в self.handlers[name]. Верни имя.
  • route(message) — вернуть имя скилла с наибольшим пересечением слов сообщения и его description; если совпадений нет ни у кого — None.
  • handle(message) — добавить сообщение в self.memory, вызвать route; если скилл не найден — вернуть {"skill": None, "result": "нет подходящего скилла"}; иначе исполнить хендлер (если есть) и вернуть {"skill": name, "result": ...}.

Готово, когда все тесты в test_skill_runtime.py зелёные — они проверяют: parse_skill достаёт name/description; load регистрирует оба скилла; route выбирает релевантный и даёт None на нерелевантном; handle исполняет хендлер и копит память между сообщениями.

USE IT

Готовые платформы (self-hosted, MIT, model-agnostic) делают это в полный рост:

  • OpenClaw — гейтвей-оркестратор, messaging-first (Telegram / Slack / Signal / …), маркетплейс скиллов ClawHub, любой OpenAI-совместимый бэкенд (включая локальные модели через Ollama).
  • Hermes Agent (Nous Research) — self-improving loop: после задачи агент пишет переиспользуемый скилл и улучшает его; многослойная модель безопасности; умеет импортировать конфигурацию из OpenClaw.
  • Оба ставятся недорого (VPS начального уровня + стоимость модели), а скиллы переносимы между ними по стандарту agentskills.io.

⚠️ Self-host решает приватность и суверенитет (11.1), но это исполнение кода у тебя — нужен sandbox и защита от prompt injection (11.5). Свой агент = своя ответственность за безопасность.

ЧАСТЫЕ ОШИБКИ

  • Считать платформу «магией». Внутри — простое ядро: реестр скиллов + маршрутизация + память. Всё остальное (каналы, роутер моделей) — обвязка вокруг него.
  • Забыть про безопасность self-host. Агент исполняет код и ходит в инструменты на твоём железе. Без sandbox и защиты от prompt injection (11.5) это дыра, а не удобство.
  • Привязаться к одной платформе. Скиллы переносимы (agentskills.io) — пиши их по стандарту, чтобы перенести между OpenClaw/Hermes и не переделывать.
  • Путать self-hosted агента и облачный чат. Облачный чат — чужой сервер и чужие данные; self-host — твоё железо, ключи и контроль (приватность, 11.1). Это разный профиль рисков и затрат.
  • Грузить память без разбора. Память копится на каждое сообщение — без компакции (4.4) она раздует контекст. Подмешивай релевантное, режь старое.

from skill_runtime import Runtime, parse_skill, EDITOR_SKILL, CALC_SKILL


def _rt():
    rt = Runtime()
    rt.load(EDITOR_SKILL, handler=lambda m: "сокращено")
    rt.load(CALC_SKILL, handler=lambda m: "42")
    return rt


def test_parse_skill():
    s = parse_skill(EDITOR_SKILL)
    assert s["name"] == "editor"
    assert "редактирует" in s["description"]


def test_load_registers_skills():
    rt = _rt()
    assert set(rt.skills) == {"editor", "calc"}


def test_route_picks_relevant_skill():
    rt = _rt()
    assert rt.route("отредактируй текст письма") == "editor"
    assert rt.route("посчитай проценты") == "calc"
    assert rt.route("прогноз погоды") is None


def test_handle_executes_and_remembers():
    rt = _rt()
    res = rt.handle("отредактируй текст письма")
    assert res["skill"] == "editor" and res["result"] == "сокращено"
    rt.handle("посчитай проценты")
    assert len(rt.memory) == 2          # память между сообщениями


def test_handle_no_skill():
    assert _rt().handle("прогноз погоды")["skill"] is None

Эталонное решение

Сначала попробуй написать сам по спеке и тестам выше — потом раскрой и сверься.

code/skill_runtime.pyПоказать решение
"""Агентные платформы: загрузка скилла в локальный рантайм — Build It 4.5.

Без зависимостей. Платформы вроде OpenClaw/Hermes — это локальный рантайм, который
грузит скиллы (SKILL.md), роутит входящее сообщение в подходящий скилл, исполняет
его и держит память между сообщениями. Здесь собираем уменьшенную версию такого
рантайма. В USE IT — настоящая платформа со скиллами + память + мульти-модель + MCP.
"""
import re


def parse_skill(text):
    """SKILL.md -> {name, description, body} (frontmatter без PyYAML, как в 4.1)."""
    m = re.match(r"\s*---\s*\n(.*?)\n---\s*\n?(.*)", text, re.DOTALL)
    meta, body = {}, text.strip()
    if m:
        for line in m.group(1).splitlines():
            if ":" in line:
                k, _, v = line.partition(":")
                meta[k.strip()] = v.strip().strip('"').strip("'")
        body = m.group(2).strip()
    return {"name": meta.get("name"), "description": meta.get("description") or "", "body": body}


def _w(text):
    return set(re.findall(r"\w+", (text or "").lower()))


class Runtime:
    """Локальный агент-рантайм: грузит скиллы, роутит сообщение, держит память."""

    def __init__(self):
        self.skills = {}
        self.handlers = {}
        self.memory = []

    def load(self, skill_text, handler=None):
        s = parse_skill(skill_text)
        self.skills[s["name"]] = s
        if handler:
            self.handlers[s["name"]] = handler
        return s["name"]

    def route(self, message):
        """Выбрать скилл по пересечению слов сообщения и description (как в 4.1)."""
        q = _w(message)
        best, best_score = None, 0
        for name, s in self.skills.items():
            score = len(q & _w(s["description"]))
            if score > best_score:
                best, best_score = name, score
        return best

    def handle(self, message):
        """Обработать сообщение: роут → исполнение → запись в память."""
        self.memory.append(message)
        name = self.route(message)
        if name is None:
            return {"skill": None, "result": "нет подходящего скилла"}
        handler = self.handlers.get(name)
        return {"skill": name, "result": handler(message) if handler else f"[{name}] выполнено"}


EDITOR_SKILL = "---\nname: editor\ndescription: редактирует и сокращает текст письма\n---\n# editor\n"
CALC_SKILL = "---\nname: calc\ndescription: считает арифметику числа и проценты\n---\n# calc\n"

if __name__ == "__main__":
    rt = Runtime()
    rt.load(EDITOR_SKILL, handler=lambda m: "сокращено")
    rt.load(CALC_SKILL, handler=lambda m: "42")
    print(rt.handle("отредактируй текст письма"))
    print(rt.handle("посчитай проценты"))
    print(rt.handle("погода завтра"))
    print("в памяти сообщений:", len(rt.memory))

# Артефакт: свой агент на self-hosted платформе

Как упаковать свой навык и запустить персонального агента у себя. Мини-рантайм — [`code/skill_runtime.py`](../code/skill_runtime.py).

## Шаги

1. **Скилл** — оформи навык как `SKILL.md` (урок 4.1) или по стандарту agentskills.io (переносим между платформами).
2. **Платформа** — поставь self-hosted рантайм:
   - **OpenClaw** — `npm install -g openclaw@latest && openclaw onboard` (гейтвей + каналы).
   - **Hermes** — `curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash` (self-improving loop; умеет импортировать конфиг из OpenClaw).
3. **Модель** — подключи любую (OpenAI-совместимый API или локальная через Ollama) — model-agnostic.
4. **Память + MCP** — включи память между сессиями (4.2) и инструменты через MCP (Ф6).
5. **Каналы** — Telegram/Slack/Signal/… для доступа отовсюду.

## Как это работает (мини-версия)

```python
from skill_runtime import Runtime
rt = Runtime()
rt.load(MY_SKILL_MD, handler=my_fn)   # реестр скиллов
rt.handle("сообщение пользователя")   # роут → скилл → память
```

## Выбор платформы

| | OpenClaw | Hermes |
|---|---|---|
| Сильная сторона | каналы, экосистема, ClawHub | self-improving loop, безопасность |
| Модели | любой OpenAI-совместимый (вкл. Ollama) | любые, лучше Hermes-серия |
| Лицензия | MIT, self-host | MIT, self-host |

## Правила безопасности

- Self-host решает приватность/суверенитет данных (урок 11.1), но это **исполнение кода у тебя**.
- Обязательны **sandbox**, least-privilege инструментов и защита от **prompt injection** (урок 11.5).
- Человек на критичных/необратимых действиях (урок 7.3).
