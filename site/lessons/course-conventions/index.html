Урок 0.4 · Соглашения курса

Фаза 0 — Setup & Tooling · Результат фазы: Настроить окружение, ключи и агента; осознанно выбрать модель под задачу.

Последний урок Фазы 0 — про то, как устроен сам курс. Дальше будет 59 уроков в едином формате: фиксированные папки, 6 шагов разбора и проверяемый критерий готовности. Поймёшь формат один раз — и любой урок читается одинаково, а свои материалы сможешь оформлять так же (и проверять автоматически).

MOTTO. Единый формат урока — чтобы код, разбор и артефакт всегда лежали на своих местах.

PROBLEM

Когда у каждого урока своя раскладка, репозиторий превращается в свалку: непонятно, где код, где разбор, где готовый артефакт, а «урок готов или нет» можно понять только глазами. На 59 уроков это смертельно. Нужен общий контракт структуры и автоматическая проверка готовности.

CONCEPT

Интуиция

Соглашение — это контракт, а не бюрократия. Договорились раз: что где лежит и что значит «готово» — и дальше не думаешь об этом на каждом уроке. Главный выигрыш в том, что готовность становится машинно-проверяемой: вместо «вроде всё на месте» — функция, которая возвращает список того, чего не хватает. Такую проверку можно повесить в CI, чтобы незаконченные уроки не проходили.

Как это работает

Каждый урок — это тройка папок и цикл шагов в docs/ru.md:

phases/<NN>-<phase-slug>/<NN>-<lesson-slug>/
├── code/      Build It (с нуля) + Use It + тест test_*.py   ← может отсутствовать
├── docs/ru.md разбор по шагам (MOTTO · PROBLEM · CONCEPT · BUILD IT · USE IT · SHIP IT)
└── outputs/   артефакт: prompt · skill · agent · MCP

Definition of Done (актуальный): обязательны заполненный docs/ru.md со всеми секциями и непустой outputs/ с артефактом. А вот код и тест — не догма: концептуальные/обзорные уроки бывают без code/, а тест добавляем там, где он реально что-то проверяет. Если код в уроке есть — он запускается офлайн на CPU без платных ключей, и pytest <lesson>/code зелёный.

Где код содержательный, BUILD IT оформляется как упражнение: спека + тесты как ТЗ, эталонное решение под катом, и три пути пройти его — собрать самому, подсмотреть эталон или делегировать ИИ (с обязательной самопроверкой тестами). Это и есть AI-native практика 4D из урока 0.1.

РАЗБОР ПО ШАГАМ

Посмотрим, как готовность проверяется кодом. scaffold_lesson() создаёт пустой урок (папки + стаб ru.md со всеми секциями), а validate_lesson() возвращает список нарушений.

Шаг 1 — свежий стаб. Папки и ru.md со всеми секциями есть, но outputs/ пуст:

validate_lesson(новый_стаб) → ["пустой outputs/ (нет артефакта)"]

Секции на месте (их даёт стаб), кода нет — но это не нарушение (урок может быть концептуальным). Не хватает только артефакта.

Шаг 2 — положили артефакт в outputs/:

validate_lesson(урок) → []      # пустой список = готов по структуре

Готово: docs/ru.md со всеми секциями плюс артефакт. Код и тест — по необходимости урока. Именно так «готовность» перестаёт быть вкусовщиной и проверяется одной функцией.

BUILD IT

Scaffolder и валидатор урока с нуля, без зависимостей (читай и запусти): code/conventions.py.

  • scaffold_lesson(root, phase_dir, lesson_dir, id, title) — создаёт дерево code/docs/outputs и стаб docs/ru.md со всеми шагами;
  • validate_lesson(path) — список нарушений DoD: нет docs/ru.md или секции, пустой outputs/. Код/тест не требует — они по необходимости.
python code/conventions.py
pytest code -q

Демо создаёт демо-урок во временной папке и тут же проверяет: свежий стаб «не готов» (нет артефакта) — ровно как и должно быть до наполнения.

USE IT

Те же соглашения уже автоматизированы скиллом курса — /lesson-author <id> (см. .claude/skills/lesson-author):

/lesson-author 2.3     # находит урок в README, заводит папки, пишет стаб ru.md,
                       # ведёт по Build It / Use It / Ship It и прогоняет pytest

Цикл сдачи урока: scaffold → наполнить (где есть код — оформить как упражнение) → validate_lesson зелёный → отметить [x] в PROGRESS.md → коммит после всей фазы.

SHIP IT

Артефакт: Заготовка репозитория курса → outputs/course-kit/

Мини-кит для старта своего курса в этом формате: conventions.py (scaffolder + валидатор) и CONVENTIONS.md (контракт структуры и DoD). Тот же валидатор можно повесить в CI, чтобы незаконченные уроки не проходили проверку.

ЧАСТЫЕ ОШИБКИ

  • Считать «код+тест» обязательными для каждого урока. Нет: обязательны только разбор и артефакт. Код и тест — там, где они реально объясняют механизм; иначе получается тест ради галочки.
  • Проверять готовность глазами. «Вроде всё на месте» не масштабируется на 59 уроков. Гоняй validate_lesson (хоть в CI) — он вернёт конкретный список нехватки.
  • Свой формат на каждый урок. Разъезжающаяся структура превращает репозиторий в свалку. Держи фиксированные code/docs/outputs и общие секции.
  • Складывать артефакт не в outputs/. Артефакт (prompt · skill · agent · MCP) живёт в outputs/ — там его ищут и студент, и валидатор.

Код урока (Build It)

code/conventions.py
"""Соглашения курса: scaffolder + валидатор урока — Build It для урока 0.4.

Без зависимостей. `scaffold_lesson()` создаёт дерево урока
phases/<NN>/<lesson>/{code,docs,outputs} и стаб docs/ru.md по 6 шагам.
`validate_lesson()` проверяет Definition of Done: docs/ru.md со всеми секциями и
непустой артефакт в outputs/. Код и тест — по необходимости урока (концептуальные
уроки бывают без code/), поэтому их отсутствие нарушением не считается.
"""
from __future__ import annotations
from pathlib import Path

# Секции, обязательные в docs/ru.md (MOTTO — отдельной строкой-цитатой).
SECTIONS = ("PROBLEM", "CONCEPT", "BUILD IT", "USE IT", "SHIP IT")

STUB_TEMPLATE = """# Урок {id} · {title}

> **MOTTO.** <идея урока в одну строку>

## PROBLEM
<конкретная боль>

## CONCEPT
<интуиция, диаграмма>

## BUILD IT
<механизм с нуля в `code/`, если урок его требует; иначе обзорно>

## USE IT
<то же через реальный инструмент/API>

## SHIP IT
**Артефакт:** <из колонки Ship It> → `outputs/`

---
**DoD:** ru.md заполнен, артефакт на месте (код запускается и тест зелёный — если урок их предполагает). (стаб — заполнить через /lesson-author {id})
"""


def scaffold_lesson(root, phase_dir: str, lesson_dir: str, lesson_id: str, title: str) -> Path:
    """Создать папки урока и стаб docs/ru.md. Возвращает путь к уроку."""
    base = Path(root) / "phases" / phase_dir / lesson_dir
    for sub in ("code", "docs", "outputs"):
        (base / sub).mkdir(parents=True, exist_ok=True)
    doc = base / "docs" / "ru.md"
    if not doc.exists():
        doc.write_text(STUB_TEMPLATE.format(id=lesson_id, title=title), encoding="utf-8")
    return base


def validate_lesson(lesson_path) -> list[str]:
    """Список нарушений DoD (пустой список = урок готов по структуре)."""
    base = Path(lesson_path)
    problems = []

    doc = base / "docs" / "ru.md"
    if not doc.exists():
        problems.append("нет docs/ru.md")
    else:
        text = doc.read_text(encoding="utf-8")
        for sec in SECTIONS:
            if sec not in text:
                problems.append(f"в docs/ru.md нет секции {sec}")

    # Код и тест не обязательны: концептуальные уроки бывают без code/,
    # а тест добавляем там, где он осмыслен (см. CLAUDE.md, Definition of Done).

    out = base / "outputs"
    artifacts = [p for p in out.glob("*") if p.name != ".gitkeep"] if out.exists() else []
    if not artifacts:
        problems.append("пустой outputs/ (нет артефакта)")

    return problems


if __name__ == "__main__":
    import tempfile
    with tempfile.TemporaryDirectory() as tmp:
        lesson = scaffold_lesson(tmp, "99-demo", "99-demo-lesson", "9.9", "Демо-урок")
        print("Создан урок:", lesson.relative_to(tmp))
        print("Проверка свежего стаба:", validate_lesson(lesson))
code/test_conventions.py
from conventions import scaffold_lesson, validate_lesson, SECTIONS


def test_scaffold_creates_tree_and_stub(tmp_path):
    base = scaffold_lesson(tmp_path, "99-demo", "99-demo-lesson", "9.9", "Демо")
    assert (base / "code").is_dir()
    assert (base / "docs" / "ru.md").is_file()
    assert (base / "outputs").is_dir()
    text = (base / "docs" / "ru.md").read_text(encoding="utf-8")
    for sec in SECTIONS:
        assert sec in text                      # стаб содержит все 6 шагов


def test_validate_flags_missing_artifact(tmp_path):
    base = scaffold_lesson(tmp_path, "99-demo", "99-demo-lesson", "9.9", "Демо")
    problems = validate_lesson(base)            # ru.md со всеми секциями есть, но outputs/ пуст
    assert any("outputs" in p for p in problems)


def test_validate_passes_conceptual_lesson_without_code(tmp_path):

Артефакт (Ship It)

outputs/course-kit/CONVENTIONS.md
# Соглашения курса (course-kit)

Контракт структуры и Definition of Done для курса в формате **Build It / Use It / Ship It**.
Используй вместе с `conventions.py` (scaffolder + валидатор) из этой же папки.

## Структура урока

```
phases/<NN>-<phase-slug>/<NN>-<lesson-slug>/
├── code/      Build It (с нуля, без фреймворков) + Use It + тест test_*.py
├── docs/ru.md разбор по 6 шагам
└── outputs/   артефакт: prompt · skill · agent · MCP
```

## 6 шагов в docs/ru.md

1. **MOTTO** — идея урока в одну строку.
2. **PROBLEM** — конкретная боль.
3. **CONCEPT** — интуиция и диаграммы (ASCII/mermaid).
4. **BUILD IT** — реализация с нуля в `code/`, запускается на CPU офлайн. Не догма: концептуальные/обзорные уроки бывают без `code/`. Где код содержательный — оформляй BUILD IT как **упражнение**: спека + тесты как ТЗ, эталон под катом, и три пути пройти его (собрать самому · подсмотреть эталон · делегировать ИИ).
5. **USE IT** — то же через реальный инструмент/API (платный ключ → пометка `**Requires:**`).
6. **SHIP IT** — артефакт в `outputs/` + как переиспользовать.

## Definition of Done

- `docs/ru.md` со всеми секциями (`MOTTO·PROBLEM·CONCEPT·BUILD IT·USE IT·SHIP IT`).
- В `outputs/` обещанный артефакт (непустой).
- **Если урок предполагает Build It-код:** рабочий код и, где осмысленно, проходящий `test_*.py`; `pytest <lesson>/code` зелёный. Код/тест только там, где они реально что-то объясняют.
- Код без платных ключей (исключения помечены `**Requires:**`).

## Автоматизация

python conventions.py                 # демо: scaffold + validate во временной папке
python -c "from conventions import validate_lesson as v; print(v('phases/01-how-llms-work/01-tokens-and-tokenization'))"

# lesson-author — автор уроков курса AI Native

Скилл помогает писать уроки единообразно по каркасу курса (см. `CLAUDE.md` и `README.md`).

## Аргумент

ID урока, напр. `2.3`. Найди его в `README.md` (название, Build It / Use It / Ship It, папку).

## Шаги

1. **Определи папку** из колонки «Папка» в README. Создай `code/`, `docs/`, `outputs/` если их нет.
2. **Напиши `docs/ru.md`** по шаблону ниже — все 6 шагов, на русском.
3. **Build It → `code/` (если урок этого требует).** Реализуй механизм урока с нуля, без внешних фреймворков; должно запускаться на CPU офлайн. Концептуальные/обзорные уроки (setup, карты, обзоры) либо уроки, чей механизм дублирует более поздний, могут не иметь `code/` — тогда переходи к Ship It.
4. **Тест → `code/test_<name>.py`, где он осмыслен.** Для нетривиального кода — минимум один проходящий юнит-тест на ключевое поведение. Тривиальный код (данные, простая формула) можно оставить без теста.
   - **Оформи Build It как упражнение** (для содержательного кода). Поставь маркер `<!-- exercise -->` (сайт раскроет тесты как ТЗ и свернёт решение в «Показать решение»). Задачу ставь **подробно**, чтобы было ясно, что делать:
     1. одна строка «**Задание:** собери …»;
     2. какой файл создать и **контракт каждой функции** — вход, что делает, выход/инвариант;
     3. «**Готово, когда** все тесты зелёные» + что должно показывать демо;
     4. команды запуска (red→green);
     5. короткая подсказка «с чего начать»;
     6. ссылка `[эталонное решение](#lesson-files)`.
     - Блок «**Три пути**» (собери сам · подсмотри эталон · делегируй ИИ) и кнопку «Скопировать промпт для ИИ» сайт вставляет **сам** для exercise-уроков — в `docs/ru.md` их писать не нужно, только укажи в конце BUILD IT, что три пути и тесты ждут в [«Исходниках урока»](#lesson-files).
5. **Use It.** Покажи то же через реальный инструмент/API. Платный ключ → пометь `**Requires:**` в шапке урока.
6. **Ship It → `outputs/`.** Положи артефакт, указанный в колонке Ship It (prompt/skill/agent/MCP), и опиши переиспользование.
7. **Материалы.** Добавь блок `## Материалы` — 2–4 ссылки на офиц. доки/первоисточники по теме урока. **Каждый URL проверь** (WebSearch/WebFetch), не выдумывай ссылки.
8. **Проверь DoD** (см. CLAUDE.md); если в уроке есть код — прогони `pytest` по папке урока.
9. **Отметь готовность (автообновление).** Когда DoD выполнен:
   - в `PROGRESS.md` поменяй `- [ ] <id> …` на `- [x] <id> …`;
   - в конце `docs/ru.md` в строке DoD добавь `✅ **Урок готов**`.
10. **Коммит после фазы.** Когда **все** уроки фазы отмечены `[x]` в `PROGRESS.md`, сделай один коммит на фазу:
   ```bash
   git add -A && git commit -m "Phase <N> (<phase-slug>): lessons complete"
   ```
   Внутри фазы отдельные уроки не коммить — только по завершении всей фазы. Пуш оставь пользователю.
