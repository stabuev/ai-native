Урок 10.4 · Observability и трейсинг агентов

Фаза 10 — Production и интеграция · Результат фазы: Встроить ИИ в рабочий инструмент с мониторингом токенов и стоимости.

Requires (только для USE IT): observability-бэкенд. BUILD IT работает офлайн, без ключей и без сети.

Это финал Фазы 10. Мониторинг (10.2) дал агрегаты, телеметрия (10.3) — расходы и воспроизводимость. Но когда агент выдал странный результат, агрегатов мало — нужно увидеть весь его путь по шагам. Это трейсинг.

MOTTO. Агент недетерминирован — без трейса шагов отладка в проде превращается в гадание.

PROBLEM

Агент (Фаза 7) — это цепочка шагов: retrieve → llm_call → tool → ещё llm_call. Когда он отвечает не так или висит 10 секунд, метрика «error_rate 0.2» не поможет: непонятно, на каком шаге сломалось, какой инструмент вернул мусор, где ушло время и токены.

Нужно видеть дерево выполнения: какие шаги были, как они вложены, сколько каждый длился, с какими атрибутами (модель, токены, k у retrieve). Это трейсинг — вложенные spans на каждый шаг, как в OpenTelemetry GenAI. Соберём мини-трейсер с нуля.

CONCEPT

Интуиция

Трейс — это подробный счёт-детализация звонка, а не «итого за месяц». Мониторинг говорит «потрачено столько, ошибок столько-то»; трейс раскрывает один конкретный запрос по шагам: вот retrieve занял 50мс, вот llm_call — 1.2с (вот где время!), вот tool вернул ошибку (вот где сломалось). Шаги вложены, как пункты и подпункты счёта: agent_run содержит retrieve, llm_call, tool.

Span — это «замер одного шага»: имя, атрибуты, начало/конец, длительность, и кто его родитель (внутри какого шага он случился). Дерево spans показывает и узкое место (самый долгий), и точку отказа.

Как это работает

trace (один запрос)
 └─ span: agent_run
     ├─ span: retrieve (k=3)        ← attrs, длительность
     ├─ span: llm_call (model, токены)
     └─ span: tool (args, результат)
   → дерево spans: где узкое место, где упало, сколько стоило
  • span(name, **attrs) — контекст-менеджер: при входе фиксирует старт и родителя (текущий span из стека), при выходе — конец и длительность. Вложенность — через стек.
  • total_duration() — сумма длительностей корневых spans (не двойной счёт: дети уже внутри родителя).
  • slowest() — самый долгий span — узкое место.
Что такое контекст-менеджер (with). @contextmanager превращает функцию с yield в объект для with: код до yield выполняется при входе в блок with tracer.span("retrieve"):, код после yield — при выходе из него (даже если внутри случилась ошибка — за это отвечает try/finally). Поэтому «замер начался» пишется автоматически на входе, а «замер закончился, длительность такая-то» — на выходе; тебе остаётся лишь обернуть шаг в with.

Ключ — вложенность через стек: пока span открыт, он на вершине стека и становится родителем всех, кто открылся внутри. Так из плоского списка замеров восстанавливается дерево «шаг → подшаг».

РАЗБОР ПО ШАГАМ

Прогоним два вложенных span с фейковым clock [0, 1, 3, 6] (детерминированно для теста): a открылся в 0, внутри b открылся в 1 и закрылся в 3, затем a закрылся в 6.

вход a:  стек [a],     a.parent = None, a.start = 0
вход b:  стек [a,b],   b.parent = a (вершина стека), b.start = 1
выход b: b.end = 3 → b.duration = 3−1 = 2,  стек [a]
выход a: a.end = 6 → a.duration = 6−0 = 6,  стек []

spans: a{parent:None, dur:6}, b{parent:a, dur:2}
total_duration() → 6   (только корневой a; b уже внутри него)
slowest()        → a   (6 > 2)

Видно три вещи. Родитель определяется стеком: b открылся, пока a был на вершине → b.parent = a. Длительность — разница конца и старта. total считает только корни (a), иначе время b посчиталось бы дважды (и в a, и отдельно). slowest = a — узкое место. На реальном агенте a это agent_run, и если slowest указал на llm_call внутри — ты сразу знаешь, что тормозит модель, а не retrieval. Поменяй фейковый clock на time.perf_counter — трейсер заработает на настоящих шагах.

ЧАСТЫЕ ОШИБКИ

  • Отлаживать агента по агрегатам. «error_rate 0.2» не скажет, на каком шаге сломалось. Нужен трейс конкретного запроса — дерево spans по шагам.
  • Плоский лог вместо вложенных spans. Без родителя не видно, что внутри чего; теряется структура «шаг → подшаг». Трейсинг — это дерево, а не список строк.
  • Двойной счёт времени. Если суммировать длительности всех spans, время вложенных посчитается дважды. total берёт только корневые — дети уже внутри них.
  • Span не закрывается при исключении. Без try/finally упавший шаг оставит span открытым и сломает стек/дерево. Закрывай span всегда.
  • Самописный формат spans. Свой формат привязывает к одному инструменту. OpenTelemetry GenAI (gen_ai.*) читается Phoenix/LangSmith/Langfuse — меньше лок-ина (как и в 10.2).

code/test_tracer.py
from tracer import Tracer


def _fake_clock(values):
    it = iter(values)
    return lambda: next(it)


def test_nested_spans_and_parent():
    # последовательность вызовов часов: a.start, b.start, b.end, a.end
    tr = Tracer(clock=_fake_clock([0, 1, 3, 6]))
    with tr.span("a"):
        with tr.span("b"):
            pass
    by_name = {s["name"]: s for s in tr.spans}
    assert by_name["b"]["parent"] == "a"
    assert by_name["a"]["parent"] is None
    assert by_name["b"]["duration"] == 2
    assert by_name["a"]["duration"] == 6


def test_total_duration_counts_roots():
    tr = Tracer(clock=_fake_clock([0, 1, 3, 6]))
    with tr.span("a"):
        with tr.span("b"):
            pass
    assert tr.total_duration() == 6        # только корневой span


def test_slowest_span():
    tr = Tracer(clock=_fake_clock([0, 1, 3, 6]))
    with tr.span("a"):
        with tr.span("b"):
            pass
    assert tr.slowest()["name"] == "a"


def test_attributes_recorded():
    tr = Tracer(clock=_fake_clock([0, 1]))
    with tr.span("llm", model="sonnet-4.6"):
        pass
    assert tr.spans[0]["attrs"]["model"] == "sonnet-4.6"

Эталонное решение (Build It)

Сначала попробуй написать сам по спеке и тестам выше — потом раскрой и сверься.

code/tracer.pyПоказать решение
"""Observability и трейсинг агентов: spans с нуля — Build It 10.4.

Без зависимостей. Чтобы отлаживать агента в проде, нужен трейс: вложенные spans
(шаг → подшаг) с длительностью и атрибутами — как в OpenTelemetry GenAI. Собираем
мини-трейсер. В USE IT — LangSmith / Langfuse / Arize поверх того же принципа.
"""
import time
from contextlib import contextmanager


class Tracer:
    """Сбор вложенных spans. clock можно подменить для воспроизводимых тестов."""

    def __init__(self, clock=None):
        self.spans = []
        self._stack = []
        self._clock = clock or time.perf_counter

    @contextmanager
    def span(self, name, **attrs):
        sp = {"name": name, "attrs": attrs,
              "parent": self._stack[-1]["name"] if self._stack else None,
              "start": self._clock()}
        self.spans.append(sp)
        self._stack.append(sp)
        try:
            yield sp
        finally:
            sp["end"] = self._clock()
            sp["duration"] = sp["end"] - sp["start"]
            self._stack.pop()

    def total_duration(self):
        """Суммарная длительность корневых spans."""
        return sum(s["duration"] for s in self.spans if s["parent"] is None)

    def slowest(self):
        """Самый долгий span (узкое место)."""
        finished = [s for s in self.spans if "duration" in s]
        return max(finished, key=lambda s: s["duration"]) if finished else None


if __name__ == "__main__":
    tr = Tracer()
    with tr.span("agent_run"):
        with tr.span("retrieve", k=3):
            time.sleep(0.01)
        with tr.span("llm_call", model="sonnet-4.6"):
            time.sleep(0.02)
    print("spans:", [(s["name"], s["parent"], round(s["duration"] * 1000)) for s in tr.spans])
    print("total ms:", round(tr.total_duration() * 1000))
    print("slowest:", tr.slowest()["name"])

# observability — трейсинг шагов агента (spans)

Делает путь агента видимым: вложенные spans (шаг → подшаг) с атрибутами и длительностью — мини-OpenTelemetry для отладки в проде.

## Как пользоваться

1. Оберни шаги агента в spans:

```python
import tracer
tr = tracer.Tracer()
with tr.span("agent_run"):
    with tr.span("retrieve", k=3):
        ...
    with tr.span("llm_call", model="sonnet-4.6"):
        ...
```

2. Смотри дерево: `tr.spans` (name/parent/duration/attrs), `tr.total_duration()`, `tr.slowest()` — узкое место.
3. Для тестов подменяй `clock` на детерминированный.

## Файлы

- `tracer.py` — `Tracer.span(...)`, `total_duration()`, `slowest()`.

## Когда применять

Когда агент выдал странный результат или тормозит — трейс покажет, на каком шаге. В проде переноси на OpenTelemetry GenAI + Phoenix/LangSmith/Langfuse (те же spans, дашборды и evals). Связь: мониторинг (10.2), телеметрия (10.3), evals (7.5).
