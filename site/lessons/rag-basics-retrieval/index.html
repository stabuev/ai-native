Урок 3.3 · Основы RAG: retrieval

Фаза 3 — Текст и документы · Результат фазы: Строить пайплайны резюмирования и редактуры; собрать базовый RAG над своим корпусом.

Requires (только для USE IT): платный API-ключ одного из провайдеров. BUILD IT работает офлайн, без ключей и без сети.

Этот урок — про то, как дать модели знание твоих документов. Модель не знает твой корпус и придумывает (урок 1.4), а целиком он в окно не влезает (3.2). RAG решает это так: на запрос находим релевантные куски и кладём их в промпт. Сердце RAG — retrieval (поиск нужного), и мы соберём его сами на TF-IDF, чтобы понять механизм, прежде чем брать нейроэмбеддинги.

MOTTO. RAG — это «найди нужное, потом ответь»: retrieval решает, что модель вообще увидит.

PROBLEM

Спросишь модель про свои внутренние документы — она либо не знает, либо уверенно выдумывает. Положить весь корпус в контекст нельзя: не влезет и дорого (3.2). Значит, на каждый запрос нужно достать только релевантные куски и подложить их в промпт. Вопрос — как из тысяч кусков найти те самые. Это и есть retrieval, и от его качества зависит весь RAG: подложишь мусор — получишь мусор.

CONCEPT

Интуиция

RAG — это экзамен с открытой книгой: не зубрить весь учебник, а быстро найти нужную страницу и ответить по ней. «Найти страницу» = retrieval. Чтобы искать по смыслу, тексты переводят в векторы (эмбеддинги) так, что близкое по смыслу оказывается рядом в пространстве, — и берут ближайшие к запросу.

Как это работает

корпус → эмбеддинги (вектор на документ)
запрос → эмбеддинг запроса
            │ косинусная близость
            ▼
     топ-k ближайших кусков → в промпт модели

В Build It роль эмбеддинга играет TF-IDF — классический способ превратить текст в вектор по важности слов:

  • TF (term frequency) — как часто слово встречается в этом документе (доля от всех слов).
  • IDF (inverse document frequency) — насколько слово редкое по корпусу: log((n+1)/(df+1)) + 1, где n — число документов, df — в скольких из них слово есть. Редкое слово (df мал) → большой idf; «общее» слово вроде «и», «для» → маленький.
  • TF-IDF = TF × IDF. Слово важно, если часто встречается здесь, но редко вообще. Так вектор документа выделяет его характерные слова.

Косинусная близость сравнивает направления двух векторов: 1.0 — смотрят в одну сторону (тексты делят важные слова), 0.0 — общих значимых слов нет. search считает косинус запроса к каждому документу и берёт топ-k.

Косинус на пальцах. Возьмём два крошечных вектора по двум словам: a = [1, 0] и b = [1, 1]. Косинус = скалярное произведение / произведение длин = (1·1 + 0·1) / (√1 · √2) = 1 / 1.41 ≈ 0.71. Одно общее ненулевое слово из двух → близость ~0.71; будь все слова общими, вышло бы 1.0, ни одного общего — 0.0. В уроке векторы длиннее (по компоненте на каждое значимое слово) и взвешены tf-idf, но арифметика ровно та же — поэтому числа в разборе ниже можно принять на веру или пересчитать этим же способом.

В проде TF-IDF меняют на нейроэмбеддинги: они ловят смысл, а не точное совпадение слов, поэтому «питомцы» найдут документ про «животных», а TF-IDF — нет (он лексический). Но принцип «вектор + близость + топ-k» ровно тот же.

РАЗБОР ПО ШАГАМ

Возьмём корпус из 4 документов и прогоним search (реальные косинус-скоры):

1. Python — язык программирования общего назначения.
2. Кошки — домашние животные, любят спать днём.
3. Машинное обучение использует данные для предсказаний.
4. Собаки — верные домашние животные и любят прогулки.

Запрос «язык программирования»:

0.632  Python — язык программирования…     ← делит редкие слова «язык», «программирования»
0.000  остальные                           ← общих значимых слов нет

Запрос «домашние животные»:

0.505  Кошки — домашние животные…
0.460  Собаки — … домашние животные…
0.000  Python / Машинное обучение

Оба «животных» документа всплыли выше Python и ML. Заметь: score тут ~0.5, ниже, чем 0.632 у Python. Почему? Слова «домашние»/«животные» встречаются в двух документах (их idf ниже), а «программирования» — только в одном (idf выше), поэтому редкое совпадение весит сильнее.

Запрос «квантовая криптография блокчейн»:

0.000  все документы

Ни одного из этих слов в корпусе нет → нет пересечения → score 0. Это важный урок RAG: retrieval может ничего не найти, и это надо ловить, а не подкладывать модели нерелевантный кусок.

BUILD IT

Задание: собери TF-IDF ретривер с нуля — стандартная библиотека (math, re, collections), без сети.

Перед запуском. Работай в своей папке курса (ai-native/3.3-rag-retrieval/), а файлы урока клади в подпапку code/ (по соглашению курса). Нужен только Python 3, всё офлайн (для теста ещё pytest).

Создай retriever.py и реализуй:

  • _tokens(text) — слова в нижнем регистре (re.findall(r"\w+", text.lower())).
  • _tfidf(tokens, idf) — посчитай TF (частоту каждого слова как долю от всех) и верни вектор-словарь {слово: tf * idf.get(слово, 0)}.
  • build_index(docs) — токенизируй документы; посчитай df (в скольких документах встречается слово); idf = log((n+1)/(df+1)) + 1; верни {"docs": [...], "idf": {...}, "vectors": [tf-idf каждого дока]}.
  • _cosine(a, b) — косинус двух разреженных векторов-словарей: скалярное произведение по общим словам, делённое на произведение норм; 0.0, если норма нулевая.
  • search(query, index, k=3) — посчитай tf-idf запроса, оцени косинус к каждому документу, верни топ-k по убыванию как [{"doc": ..., "score": ...}].

ЧАСТЫЕ ОШИБКИ

  • Класть в промпт весь корпус. Не влезет и дорого (3.2). Смысл RAG — достать только топ-k релевантных кусков.
  • Игнорировать «ничего не нашлось». Если лучший score близок к 0 — релевантного нет; подложить случайный кусок хуже, чем честно сказать «не нашёл» (привет границам доверия из 1.4).
  • Ждать от лексического поиска понимания синонимов. TF-IDF матчит точные слова: «питомцы» не найдут «животных». Для смысла нужны нейроэмбеддинги (USE IT) или гибрид (урок 3.5).
  • Грузить документы без чанкинга. Искать по гигантским документам бесполезно — режь их на куски (урок 3.2), а индексируй и ищи уже по кускам.

"""RAG retrieval: TF-IDF ретривер с нуля — Build It для урока 3.3.

Без зависимостей. «Эмбеддинг» здесь — TF-IDF вектор слова; близость — косинус.
Принцип тот же, что у настоящего RAG: представить тексты числами и найти
ближайшие к запросу. В USE IT TF-IDF заменяется на нейроэмбеддинги.
"""
import math
import re
from collections import Counter


def _tokens(text):
    return re.findall(r"\w+", text.lower())


def _tfidf(tokens, idf):
    tf = Counter(tokens)
    total = sum(tf.values()) or 1
    return {w: (c / total) * idf.get(w, 0.0) for w, c in tf.items()}


def build_index(docs):
    """docs: list[str] -> индекс с idf и tf-idf векторами документов."""
    toks = [_tokens(d) for d in docs]
    df = Counter()
    for t in toks:
        df.update(set(t))
    n = len(docs)
    idf = {w: math.log((n + 1) / (c + 1)) + 1 for w, c in df.items()}
    return {"docs": list(docs), "idf": idf, "vectors": [_tfidf(t, idf) for t in toks]}


def _cosine(a, b):
    common = set(a) & set(b)
    dot = sum(a[w] * b[w] for w in common)
    na = math.sqrt(sum(v * v for v in a.values()))
    nb = math.sqrt(sum(v * v for v in b.values()))
    return dot / (na * nb) if na and nb else 0.0


def search(query, index, k=3):
    """Топ-k документов по косинусной близости к запросу."""
    qv = _tfidf(_tokens(query), index["idf"])
    scored = sorted(
        ((_cosine(qv, v), i) for i, v in enumerate(index["vectors"])),
        key=lambda x: x[0], reverse=True,
    )
    return [{"doc": index["docs"][i], "score": round(s, 4)} for s, i in scored[:k]]


if __name__ == "__main__":
    corpus = [
        "Python — язык программирования общего назначения.",
        "Кошки — домашние животные, любят спать днём.",
        "Машинное обучение использует данные для предсказаний.",
        "Собаки — верные домашние животные и любят прогулки.",
    ]
    index = build_index(corpus)
    for q in ["домашние животные", "язык программирования"]:
        print(f"\nЗапрос: {q}")
        for hit in search(q, index, k=2):
            print(f"  {hit['score']:.3f}  {hit['doc']}")

outputs/mini_rag.py
"""mini_rag.py — артефакт урока 3.3: самодостаточный RAG-скелет (retrieve → augment).

Без зависимостей и без сети. TF-IDF retrieval + сборка промпта с контекстом.
Шаг generate оставлен заглушкой: подставь вызов модели (Claude/OpenAI/Gemini).

Запуск:  python mini_rag.py
"""
import math
import re
from collections import Counter


def _tokens(t):
    return re.findall(r"\w+", t.lower())


def _tfidf(tokens, idf):
    tf = Counter(tokens)
    total = sum(tf.values()) or 1
    return {w: (c / total) * idf.get(w, 0.0) for w, c in tf.items()}


def build_index(docs):
    toks = [_tokens(d) for d in docs]
    df = Counter()
    for t in toks:
        df.update(set(t))
    n = len(docs)
    idf = {w: math.log((n + 1) / (c + 1)) + 1 for w, c in df.items()}
    return {"docs": list(docs), "idf": idf, "vectors": [_tfidf(t, idf) for t in toks]}


def _cosine(a, b):
    dot = sum(a[w] * b[w] for w in set(a) & set(b))
    na = math.sqrt(sum(v * v for v in a.values()))
    nb = math.sqrt(sum(v * v for v in b.values()))
    return dot / (na * nb) if na and nb else 0.0


def retrieve(query, index, k=3):
    qv = _tfidf(_tokens(query), index["idf"])
    scored = sorted(((_cosine(qv, v), i) for i, v in enumerate(index["vectors"])),
                    key=lambda x: x[0], reverse=True)
    return [index["docs"][i] for s, i in scored[:k] if s > 0]


def build_prompt(query, contexts):
    """augment: собрать промпт с найденным контекстом и требованием отвечать по нему."""
    ctx = "\n".join(f"- {c}" for c in contexts) or "(контекст не найден)"
    return (f"Ответь на вопрос ТОЛЬКО по контексту ниже. "
            f"Если ответа в контексте нет — скажи «не знаю».\n\n"
            f"Контекст:\n{ctx}\n\nВопрос: {query}\nОтвет:")


def answer(query, index, generate=None, k=3):
    """retrieve → augment → generate. generate(prompt)->str; по умолчанию вернёт промпт."""
    contexts = retrieve(query, index, k)
    prompt = build_prompt(query, contexts)
    if generate is None:
        return prompt                      # без модели показываем готовый промпт
    return generate(prompt)


if __name__ == "__main__":
    corpus = [
        "Возврат товара возможен в течение 14 дней с момента покупки.",
        "Гарантия на электронику составляет 12 месяцев.",
        "Доставка по городу занимает 1-2 рабочих дня.",
    ]
    index = build_index(corpus)
    print(answer("сколько дней на возврат?", index))
    # с моделью:  answer(q, index, generate=lambda p: client.messages.create(...))
