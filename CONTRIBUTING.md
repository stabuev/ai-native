# Как вносить вклад в AI Native

Спасибо за интерес! Курс использует **Build It / Use It / Ship It** как конструктор, а
не обязательное оглавление.
Перед добавлением урока загляни в [`LESSON_CONTRACT.md`](LESSON_CONTRACT.md)
(педагогический результат), [`CLAUDE.md`](CLAUDE.md) (правила проекта) и
[`LESSON_TEMPLATE.md`](LESSON_TEMPLATE.md) (шаблон).

## Физическая структура

```
phases/<PP>-<phase-slug>/<LL>-<lesson-slug>/
├── docs/ru.md   обязательный связный материал
├── code/        опционально: исполняемая практика + тесты
├── outputs/     опционально: переиспользуемый артефакт
└── quiz.json    опционально: диагностический квиз
```

`PP` — номер фазы, `LL` — порядковый номер урока внутри фазы. Например, id `2.4`
соответствует пути `phases/02-<phase>/04-<lesson>/`.

Слаги и номера зафиксированы в `README.md` (колонка «Папка») — не меняй их без
согласованного обновления программы и ссылок.
Секции `MOTTO / PROBLEM / CONCEPT / BUILD IT / USE IT / SHIP IT`, код, артефакт, квиз и
материалы добавляй только после проверки их целесообразности.

## Definition of Done

Полный критерий готовности задан в [`LESSON_CONTRACT.md`](LESSON_CONTRACT.md). Перед
отметкой `[x]` проверь оба слоя:

- [ ] **Педагогический:** наблюдаемый результат, оправданное место, готовые пререквизиты,
  самостоятельное действие, реалистичная ошибка, проверка и перенос.
- [ ] **Целесообразность:** каждый оставленный блок поддерживает outcome; ненужные
  секции, код, квиз, материалы и артефакты опущены или объединены.
- [ ] **Перенос:** результат можно применить в новой задаче; если нужен отдельный
  артефакт, его можно переиспользовать и принять по явным критериям.
- [ ] **Структурный:** существует заполненный `docs/ru.md` без следов стаба.
- [ ] **Технический:** если код входит в результат, он запускается, а тесты проверяют
  ключевое поведение; соразмерный `pytest` зелёный.
- [ ] **Публикация:** сайт пересобран, ограничения доступа отмечены, изменчивые источники
  проверены.

Проверить структуру автоматически:

```bash
python course_validation.py phases/<PP-phase>/<LL-lesson>
```

Валидатор подтверждает только наличие заполненного материала. Он намеренно не требует
именованные секции и артефакт; `OK` не заменяет проверку их целесообразности.

## Стиль

- Объяснения по-русски, код и идентификаторы — по-английски.
- Минимум зависимостей; стандартная библиотека по умолчанию.
- Никаких заглушек — только рабочие примеры.
- Use It и мульти-провайдерность добавляй только когда это помогает outcome.
- Если есть «Материалы», **проверяй** ссылки и не добивай количество слабыми источниками.
- Образец: `phases/01-how-llms-work/01-tokens-and-tokenization/`.

## Перед коммитом

```bash
pip install -r requirements.txt
pytest -q                 # все тесты зелёные
python course_validation.py
node site/build.js        # пересобрать сайт (data.js), если менял уроки
```

Коммит — один на завершённую фазу (`Phase <N> (<slug>): lessons complete`); пуш — за мейнтейнером.
