# Конструктор урока

Обязательного набора секций нет. Сначала сформулируй паспорт урока, затем выбери только
те элементы, которые поддерживают его результат. Полный критерий — в
[`LESSON_CONTRACT.md`](LESSON_CONTRACT.md).

## Обязательная основа

```markdown
# Урок <id> · <название>

**Фаза <N> — <фаза>** · **Результат фазы:** <outcome>
**Результат урока:** после урока студент сможет <наблюдаемое действие + критерий качества>.
**Опора:** <что уже освоено> · **Дальше:** <где результат понадобится>
<!-- **Requires:** внешний доступ или платный ключ — только если применимо -->

<Связный учебный переход в целесообразной структуре>

---
**Часы:** ~<n> · **DoD:** результат доказан; структура целесообразна; соразмерные проверки зелёные.
```

## Возможные элементы

Не копируй весь список автоматически. Заголовки можно переименовывать и объединять.

| Элемент | Добавляй, когда |
|---|---|
| `MOTTO` | одна фраза помогает удержать центральную модель |
| `PROBLEM` | новую рабочую ситуацию нужно поставить отдельно |
| `CONCEPT` | появляется новый механизм, который нельзя объяснить внутри примера |
| `BUILD IT` | сборка или вычисление делает механизм наблюдаемым и входит в outcome |
| `USE IT` | перенос в реальный инструмент отличается от учебной практики |
| `SHIP IT` | отдельный артефакт будет переиспользован за пределами урока |
| `ЧАСТЫЕ ОШИБКИ` | есть существенный failure mode, которому нужен отдельный разбор |
| `ПРОВЕРЬ СЕБЯ` | квиз диагностирует неверную модель лучше основной практики |
| `Материалы` | конкретный источник закрывает границу урока или полезное углубление |

Для исполняемой практики используй `code/` и behavioral tests. Для решения, плана или
спецификации — самостоятельный артефакт и содержательную рубрику. Для процедуры или роли
— сценарий, наблюдаемые критерии и повторную попытку. Отдельный файл в `outputs/` не
нужен, если переносимый результат естественно остаётся в самом действии студента.

## Чек-лист

- [ ] Результат сформулирован как наблюдаемое действие и нужен следующему шагу.
- [ ] Существенные пререквизиты освоены раньше или получают короткий мост.
- [ ] Каждый оставленный блок выполняет понятную учебную функцию.
- [ ] Ненужные блоки объединены или опущены, а не заполнены ради шаблона.
- [ ] Студент выполняет самостоятельное действие, а не копирует демонстрацию.
- [ ] Способ проверки соответствует outcome: тест, рубрика, сценарий или объяснение.
- [ ] Если ошибка существенна, студент умеет её распознать и исправить.
- [ ] Понятно, как перенести результат в новую задачу; артефакт создаётся только при
  реальной пользе.
- [ ] Код, квиз, материалы и внешние инструменты добавлены только по необходимости.
- [ ] Выполнены соразмерные проверки; сайт пересобран после изменения урока.
